# 결제모듈 보존형 비활성화 배포 문서화 요청 20260723

## 사용자 요청

사용자는 이전 “Paddle 제외” 표현이 결제 코드를 제거한다는 뜻이 아니라, 나중에 결제 기능을 추가할 것을 대비해 결제 부분은 구조적으로 남겨두고 이번 배포에서는 비활성화한다는 뜻이었다고 정정했다.

이후 지금까지 나온 내용을 개발자에게 전달할 문서로 작성해 달라고 요청했다.

## 정리 방향

- “Paddle 제외”가 아니라 “결제 모듈 보존형 비활성화 배포”로 표현을 바꿨다.
- 결제 코드는 삭제하지 않고 feature flag로 production 유저 진입만 막는 정책으로 정리했다.
- 무료 상담형 베타 배포 범위와 결제 재오픈 전 체크리스트를 분리했다.

## 주요 전달 내용

- 결제 정책/타입/테스트는 보존
- `PAYMENTS_ENABLED=false` 같은 feature flag로 checkout, 코인샵, 멤버십, 유료 unlock 차단
- Paddle env gate는 이번 배포 제외
- Supabase, AI, origin, rate limit, 실제 API QA는 P0 유지
- 결제 재오픈 시 Paddle env, QA, webhook, 환불/CS 플로우를 별도 스프린트로 처리
