# 배포보안 문제점파악 문서화 메모리저장

## 사용자 프롬프트
배포를 할려고 하는데 보안이나 배포할시 문제점이 있는지 파악. 코딩은 하지말고 파악만 해줘

## 추가 사용자 프롬프트
지금까지 나온 내용을 문서화 하고 메모리에 저장. 그리고 개발자가 수정할수 있도록 문서도 정확히 작성

## 처리 내용
- 보안 리뷰와 배포 관점으로 현재 코드와 설정을 읽기 전용으로 점검했어.
- 소스코드는 수정하지 않았어.
- 배포 전 남은 P0/P1/P2 이슈를 개발자 수정 가이드로 정리했어.
- 개발일지에 배포보안 릴리즈게이트 점검 내용을 남겼어.
- `.codex/memory`에 배포 전 보안 게이트와 재발 방지 내용을 저장했어.
