# 하나 궁합상담 상대방정보 미반영 수정요청

## 사용자 요청

하나랑 상담하는데 상대방 정보를 같이 입력했는데도 내 정보로만 상담하고 상대방 정보가 전달이 안되는 것 같다.

## 처리 요약

- 체계적 디버깅으로 입력 폼, 궁합 생성 액션, 채팅 페이지, 채팅 API, 첫 상담 프롬프트 흐름을 추적했다.
- 상대방 정보 저장 흐름은 존재했지만 첫 자동 상담 문구와 fallback이 사용자 단독 분석으로 흐를 수 있는 점을 확인했다.
- 하나 첫 상담 문구를 두 사람의 관계와 궁합 흐름으로 바꿨다.
- 채팅 API에서 상대방 이름을 시스템 프롬프트와 fallback에 명확히 전달하게 수정했다.
- 최신 궁합 row를 읽도록 조회 방식을 보강했다.
