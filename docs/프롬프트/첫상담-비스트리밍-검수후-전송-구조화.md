# 첫상담 비스트리밍 검수 후 전송 구조화 요청

## 사용자 입력

```text
운영 채팅은 아직 스트리밍으로 먼저 사용자에게 보여준 뒤 저장 전 게이트를 거르는 구조라서, 아주 엄격하게 하려면 첫 상담만 비스트리밍으로 생성, 검수, 필요 시 재작성 후 통과본을 보여주는 구조가 다음 단계로 좋아.
```

## 해석

- 첫 상담 응답은 사용자에게 보이기 전에 먼저 생성하고 품질 게이트를 통과해야 한다.
- 품질 미달 응답은 브라우저로 스트리밍하지 않고, 서버에서 재작성한 뒤 통과본만 내려보낸다.
- 후속 상담은 기존 스트리밍 UX를 유지하되, 저장과 별 차감은 기존 성공 게이트를 유지한다.
