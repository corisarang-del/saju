## 사용자 요청

사용자는 테스터 피드백을 개발자에게 전달할 수 있게 정리하고, 지금까지 나온 내용을 문서화하고 메모리에 저장해 달라고 요청했다.

## 원 피드백 요지

- redirect 정상
- Playwright 렌더링 정상, 콘솔 에러 없음
- `env QA_SCENARIO=free-only node scripts/qa-live-api-check.mjs` 실패
- `node scripts/qa-live-api-check.mjs` 실패
- 실패 원인은 Supabase DNS lookup failed
- `sfpwgywcmhgilrqearsz.supabase.co`가 `ENOTFOUND`
- 실제 인증 유저 생성까지 못 감
- 무료/유료 첫 상담, 별 차감, 거래 로그, 저장된 assistant 품질은 실제 DB 기준으로 끝까지 확인 못 함
- QA 스크립트가 DNS 실패를 구조화해서 알려주는 개선은 확인
- Gemini QA는 통과했지만 관계 고민에 “몸 상태 기록” 같은 어색한 행동 제안은 보완 여지 있음

## 정리 방향

- 코드 게이트/화면 렌더링 통과와 실제 DB 기반 API QA 미완료를 분리했다.
- Supabase DNS 정상화 후 재실행해야 할 명령과 통과 기준을 개발자 액션으로 정리했다.
- 결제모듈 보존형 비활성화 배포에서도 Supabase 무료 상담 API QA는 배포 blocker로 유지한다고 명확히 했다.
