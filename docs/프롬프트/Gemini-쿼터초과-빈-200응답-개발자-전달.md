# Gemini 쿼터초과 빈 200응답 개발자 전달

사용자는 Gemini 쿼터 초과 상태에서 `/api/saju/chat`이 `200 OK`를 반환하지만 body가 빈 문자열이고, DB에도 사용자/AI 메시지가 저장되지 않으며, 서버 로그에는 `[saju/chat] stream error {}`만 남는 문제를 공유했다. 이미 `docs/개발일지/실사용-응답흐름-qa-점검.md`, `docs/pm/실사용-qa-빈-ai응답-개발자-전달.md`, `docs/프롬프트/실사용-응답흐름-qa-요청.md`에 문서화돼 있고, 개발자에게는 “Gemini 실패/쿼터 초과 시 빈 200 응답 말고 명확한 실패 UI와 로그를 남기게 고쳐라”로 전달하면 된다고 했다.
