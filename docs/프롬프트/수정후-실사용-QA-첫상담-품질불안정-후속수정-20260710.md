# 수정후 실사용 QA 첫상담 품질불안정 후속수정

## 사용자 요청
- `docs/개발일지/수정후-실사용-QA-첫상담-품질불안정-20260710.md`를 확인했다.
- 해당 QA 문서를 보고 수정할 부분이 있으면 수정해달라고 요청했다.

## 해석
- 첫 상담 실제 API에서 503 반복, 내부 저장 전 게이트와 외부 QA 기준 불일치, `오늘 할 구체 행동 없음` 저장, 무료 채팅 거래 로그 확인 문제가 핵심이다.
- 코드로 바로 닫을 수 있는 범위는 첫 상담 품질 게이트 정렬, 안전 fallback, 구조화 로그, QA 스크립트 기준 정렬이다.
- 채팅 거래 로그 누락은 최신 `decrement_star` RPC 마이그레이션 적용 여부와 운영 DB 함수 정의 확인이 필요해서 중복 insert를 코드에 넣지 않고 원격 SQL 적용으로 해결한다.
- 실사용 API QA에서 최종 잔액 기대값이 잘못된 것도 확인해 QA 스크립트 회귀 테스트와 함께 수정한다.
