# 결제모듈 보존형 비활성화 배포 개발자전달 반영

## 사용자 요청
- 사용자가 `docs/pm/결제모듈-보존형-비활성화-배포-개발자-전달-20260723.md` 문서를 전달했다.
- 목표는 결제 코드를 삭제하지 않고 보존하되, production 초기 배포에서는 결제 진입점만 feature flag로 닫는 것이다.

## 핵심 요구
- `PAYMENTS_ENABLED` 또는 동등한 flag 기본값 false
- Paddle env는 비결제 베타 배포의 release blocker에서 제외
- 코인샵, checkout, 채팅 paywall, 월간 리포트 상세판, 종합 백서 CTA에서 결제 진입 차단
- 무료 상담형 베타 플로우는 유지
- 결제 재오픈 때는 Paddle env gate와 실제 결제 QA를 별도로 통과해야 함
