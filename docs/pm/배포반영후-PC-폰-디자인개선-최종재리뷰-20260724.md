## 해결 확인

- 모바일 홈 H1에서 `고르면`이 `고르/면`으로 찢어지던 문제가 재현되지 않는다.
- 모바일 홈 첫 카드의 `대화하기` CTA가 하단 탭 위에서 온전히 보인다.
- PC 홈 H1은 의미 단위로 안정적으로 줄바꿈된다.
- PC 홈 캐러셀은 잘린 오류보다는 의도된 peek 패턴으로 읽힌다.
- `/ko/reading` 날짜 placeholder와 예시 텍스트 대비가 이전보다 선명하다.
- `/ko/reading` 하단 `다음` CTA가 하단 탭에 가려지지 않는다.
- `/ko`, `/ko/reading` 모두 콘솔 error/warning 0건이다.
- 1280x720, 390x844 모두 가로 오버플로우가 없다.

## 남은 개선점

### P2. 시간 선택 placeholder 대비를 날짜 입력과 더 맞춰줘

- 위치: `/ko/reading` 모바일, `태어난 시간` select
- 현상: `시간을 선택하세요 (선택)` placeholder가 날짜 placeholder보다 한 단계 약하게 보인다.
- 영향: 사용성 차단은 아니지만, 입력 폼 전체의 완성감과 선택 가능 상태의 명료함이 조금 떨어진다.
- 권장: select placeholder 색상을 날짜 placeholder와 같은 톤으로 맞추거나, trigger 내부 텍스트 대비를 한 단계 올려줘.

### P3. PC 캐러셀 왼쪽 peek 존재감을 조금만 더 낮춰줘

- 위치: `/ko` PC 1280x720
- 현상: 첫 카드 왼쪽의 이전 카드 조각/edge fade가 기능 오류처럼 보이진 않지만, 사이드바 바로 옆이라 시선이 살짝 걸린다.
- 영향: 릴리즈 차단은 아니고 고급감 마감 이슈에 가깝다.
- 권장: 1280px 전후에서 캐러셀 시작 gutter를 조금 늘리거나 fade 시작 구간을 소폭 넓혀줘.
