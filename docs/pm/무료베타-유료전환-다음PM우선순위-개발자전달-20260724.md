# 무료베타 유료전환 다음 PM 우선순위 개발자 전달

## 배경
- 현재 production 무료 베타는 제품/디자인/보안/운영 기준에서 공개 운영 가능한 수준까지 올라왔다.
- 다만 유료 코칭형 사주앱으로 완성됐다고 보려면 결제 운영 안정성, 상담 품질 반복성, 구독 가치 증명이 더 필요하다.
- 이번 문서는 Paddle 결제를 당장 열라는 문서가 아니다. 결제는 계속 닫아둔 상태에서 무료 베타 운영 데이터를 모으고, 결제 재오픈 준비물을 닫는 개발 우선순위다.

## 총괄 PM 판정
- 무료 베타 사이트는 제대로 설계되고 구현된 상태로 본다.
- 유료 코칭형 사주앱 방향도 맞지만, 아직 결제 운영 안정성, 상담 품질 반복성, 구독 가치 증명이 남아 있다.
- 무료 베타는 계속 운영해도 된다.
- 결제 버튼은 `release:gate:payments:live`와 Paddle signed webhook QA가 통과하기 전까지 열면 안 된다.

## P0. 무료 베타 운영 데이터 수집

### 요구사항
- Paddle 제외 무료 베타 운영을 유지하면서 실제 상담 품질 데이터를 수집한다.
- 결제 기능은 production에서 계속 비활성화한다.
- 무료 베타 핵심 퍼널 데이터를 이벤트로 남긴다.

### 필수 이벤트
- 랜딩 진입
- `/ko/reading` 진입
- 생년월일 입력 완료
- 성별 입력 완료
- 상담사 선택
- 로그인 시작
- 로그인 성공
- 무료 상담 시작
- 첫 assistant 응답 성공
- 첫 assistant 응답 실패
- 후속 질문 전송
- 무료 3회 소진
- 코인샵 또는 가격 패널 노출
- 결제 비활성화 안내 노출

### 수용 기준
- 관리자 또는 로그에서 하루 단위로 가입 전환, 첫 상담 완료율, 무료 3회 소진율을 확인할 수 있다.
- 결제 off 상태에서도 가격/코인샵/푸터/JSON-LD가 결제 가능처럼 오해되지 않는다.
- 무료 상담 실패율과 평균 응답 시간을 운영자가 확인할 수 있다.

## P0. 첫 상담 응답 속도와 실패 복구 UX 안정화

### 문제
- 이전 QA 기록에서 첫 상담 응답이 55초 이상 걸린 사례가 있었고, paid 경로에서는 110초 사례도 있었다.
- 20대 여성 유저는 30초를 넘으면 앱이 멈췄다고 느낄 수 있다.

### 요구사항
- 첫 상담 생성 중 상태를 단계별로 보여준다.
- 빈 응답, provider quota 초과, stream 오류, timeout은 성공처럼 보이면 안 된다.
- 실패 시 사용자 메시지와 별/무료 횟수/거래 로그 상태가 일관되게 복구되어야 한다.

### 권장 UX
- 0-10초: `사주 흐름을 정리하고 있어`
- 10-30초: `고민에 맞는 상담 방향을 잡고 있어`
- 30초 이후: `답변이 조금 길어지고 있어. 완료되면 바로 보여줄게`
- 실패 시: `답변을 완성하지 못했어. 무료 횟수는 차감하지 않았어. 다시 시도해줘`

### 수용 기준
- 첫 상담 실패/빈 응답/부분 응답에서는 assistant 메시지 저장, 별 차감, 무료 사용량 증가가 발생하지 않는다.
- 클라이언트는 빈 200 응답을 성공으로 처리하지 않는다.
- 서버 로그에는 provider error type, request id, reading id, user id hash 또는 안전한 식별자가 남는다.
- p95 첫 상담 응답 시간을 주기적으로 확인할 수 있다.

## P0. 5대 고민별 첫 답변 품질 캡처와 캐릭터 차별화 검수

### 대상 고민
- 썸/재회
- 이직/퇴사
- 돈 모으기
- 번아웃
- 친구/가족관계

### 검수 기준
- 답변 첫 문단이 사용자의 고민을 생활 언어로 다시 잡아준다.
- 사주 요소가 조언의 근거로 자연스럽게 들어간다.
- 무섭게 압박하거나 결정을 강요하지 않는다.
- 캐릭터별 답변 구조와 마지막 질문이 구분된다.
- 같은 고민을 여러 상담사에게 물었을 때 말투만 다른 것이 아니라 관점이 달라야 한다.

### 캐릭터별 기대
- 현우: 핵심 판단, 오늘 할 행동, 위험을 겁주지 않는 현실 조언
- 하나: 감정 속도, 표현 방식, 관계 대화 톤
- 민준: 돈 구조, 새는 돈, 이번 달 기준
- 인영: 좋은 시기, 피할 시기, 준비 행동
- 지안: 재회 가능성보다 연락 기준과 감정 소모 관리
- 서준: 버틸 조건, 움직일 조건, 준비 항목
- 도윤: 사람, 돈, 시기 리스크를 차분히 정리

### 산출물
- 5대 고민별 첫 답변 캡처
- 상담사별 톤 차이 표
- 수정 필요한 문장 목록
- 결제 전환을 막는 어색한 조언 목록

## P1. 스타터 2,900원 첫 결제 1회 한정 정책 확정

### 문제
- `10별 2,900원`은 첫 결제 장벽을 낮추는 데 좋다.
- 하지만 30별 9,900원보다 별 단가가 낮아 반복 구매가 열리면 30별 상품을 잠식한다.

### 요구사항
- 스타터는 첫 결제 1회 한정 상품으로 정책을 확정한다.
- 서버 정책, 관리자 화면, 코인샵 문구, 결제 webhook 지급 정책이 같은 기준을 사용해야 한다.
- 이미 스타터를 구매한 유저에게는 코인샵에서 재구매 불가 또는 일반 패키지 유도 상태가 보여야 한다.

### 수용 기준
- 스타터 미구매 유저는 `10별 2,900원`을 볼 수 있다.
- 스타터 구매 이력이 있는 유저는 같은 상품을 다시 구매할 수 없다.
- 클라이언트 조작으로 starter 재구매를 요청해도 서버에서 거부된다.
- 운영자는 관리자 화면에서 스타터 구매 여부를 확인할 수 있다.

## P1. 월구독 50별 사용 예시를 화면에 구체화

### 문제
- `월 9,900원 / 50별`은 숫자만으로는 구독 가치가 바로 느껴지지 않는다.

### 요구사항
- 월구독 카드, 코인샵, 가격 FAQ 중 최소 2곳에 50별 사용 예시를 보여준다.
- 예시는 기능 설명이 아니라 실제 사용 장면 중심이어야 한다.

### 권장 카피
- `매일 1회 상담 30별 + 월간 리포트 3별 + 후속 질문 17회`
- `한 달 동안 매일 가볍게 묻고, 중요한 달 흐름은 리포트로 정리해`
- `반복 고민을 이어서 물어보는 유저에게 맞는 플랜`

### 수용 기준
- 유저가 50별로 무엇을 할 수 있는지 가격 카드 안에서 바로 이해할 수 있다.
- 무료 베타 상태에서는 결제 가능처럼 보이지 않도록 `결제 준비 중` 또는 비활성 상태와 충돌하지 않는다.
- JSON-LD/FAQ/코인샵/랜딩의 월구독 설명이 서로 모순되지 않는다.

## P1. 결제 재오픈 전 release gate 고정

### 요구사항
- 결제 재오픈 전 `release:gate:payments:live`와 signed webhook QA를 필수 gate로 둔다.
- Paddle env가 없거나 webhook 정상 서명 payload가 실패하면 결제 버튼을 열지 않는다.

### 필수 검증
- Vercel production Paddle env 등록
- Paddle Dashboard product/price id가 실제 가격 정책과 일치
- `10별 2,900원` product/price id 일치
- `월 9,900원 / 50별` product/price id 일치
- 무서명 webhook 401
- 정상 서명 `transaction.completed` 200
- 정상 서명 중복 transaction 중복 지급 없음
- 실패/환불/구독 취소 이벤트 상태 전이 확인

### 수용 기준
- `release:gate:payments:live`가 통과하지 않으면 결제 UI를 활성화할 수 없다.
- payment off 무료 베타 gate와 payment on 유료 gate가 문서와 스크립트에서 분리되어 있다.

## P1. 무료 베타 지표 기반 유료 전환 검증

### 핵심 질문
- 유저가 무료 3회를 실제로 다 쓰는가?
- 첫 답변 뒤 후속 질문을 하는가?
- 같은 유저가 다음 날 다시 오는가?
- 월간 리포트 미리보기를 보고 더 보고 싶어 하는가?
- 결제 버튼이 열렸을 때 2,900원 스타터를 누를 만큼 신뢰가 쌓였는가?

### 핵심 지표
- 랜딩 방문 대비 `/ko/reading` 진입률
- `/ko/reading` 진입 대비 생년월일 입력 완료율
- 로그인 성공률
- 첫 상담 시작률
- 첫 상담 완료율
- 첫 assistant 응답 p50/p95 시간
- 첫 답변 후 후속 질문률
- 무료 3회 소진율
- D1 재방문율
- D7 재방문율
- 월간 리포트 미리보기 노출률
- 스타터 결제 클릭률
- 멤버십 관심 클릭률

### 수용 기준
- 무료 베타 기간에도 유료 전환 전조 지표를 볼 수 있다.
- 유료 결제 오픈 전 최소 7일치 실제 사용 데이터를 기준으로 우선순위를 다시 판단한다.

## 개발 순서 제안
1. 첫 상담 실패 복구와 응답 시간 계측을 먼저 닫는다.
2. 5대 고민별 첫 답변 QA 산출물을 만든다.
3. 무료 베타 퍼널 이벤트를 정리한다.
4. 월구독 50별 사용 예시를 가격 화면에 반영한다.
5. 스타터 첫 결제 1회 한정 정책을 서버 기준으로 확정한다.
6. Paddle env와 signed webhook QA는 결제 재오픈 스프린트에서 별도 gate로 닫는다.

## 출시 판단
- 무료 베타: 운영 유지 가능
- 유료 결제: 아직 오픈 불가
- 다음 목표: 무료 사용 데이터로 상담 품질과 반복 사용 의향을 확인한 뒤, Paddle live gate 통과 후 제한적으로 결제 재오픈
