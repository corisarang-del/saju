# 20대 여성고객 랜딩 입력흐름 피로도 개발자 전달

## 배경
- 20대 여성 유저 관점에서 현재 서비스는 긍정적이다.
- 전체적으로 “광고 같은 사주앱” 느낌이 줄고, “내 고민을 차분히 정리해주는 상담형 앱”에 가까워졌다.
- 이번 피드백은 큰 구조 문제가 아니라, 9점대 완성도를 위한 미세 UX 조정이다.
- 핵심은 무료 혜택 표현 통일, 가격표 분리감, 태어난 시간 안내, 현우 톤, 컬러 피로도 완화다.

## 우선순위 요약
1. 캐릭터 카드 무료 혜택 문구를 `3별`에서 `3회 무료`로 통일
2. 태어난 시간 입력에 `몰라도 분석 가능` 안내 추가
3. 가격표 배지의 시각적 분리감 보강
4. 현우 카드/프롬프트 톤을 덜 위협적으로 조정
5. 랜딩에서 캐릭터 카드와 가격표 인지 순서 점검
6. 장기 사용감을 위해 일부 포인트 컬러에 따뜻한 중립색 검토

## 티켓 1. 캐릭터 카드 무료 혜택 문구 통일

### 문제
- 캐릭터 카드에는 아직 `가입시 3별 무료 지급`이라고 보인다.
- 다른 영역에서는 `3회 무료`로 설명하고 있어 표현이 나뉜다.
- 일반 유저에게는 `3별`보다 `3회 무료`가 더 직관적이다.

### 요구사항
- 캐릭터 카드의 무료 혜택 문구를 `가입시 3회 무료` 또는 `가입하면 3회 무료`로 바꾼다.
- 별 정책은 내부적으로 유지하되, 가입 전 마케팅 문구는 메시지 횟수 기준으로 설명한다.

### 권장 카피
- `가입하면 3회 무료`
- `가입 후 3회 무료 상담`
- `무료 3회로 먼저 확인`

### 수용 기준
- 랜딩 캐릭터 카드에서 `3별 무료 지급` 표현이 보이지 않는다.
- 랜딩, FAQ, 가격표, 로그인 모달의 무료 혜택 표현이 충돌하지 않는다.
- `1별 = 메시지 1회` 설명은 가격표나 FAQ에서 유지된다.

### QA 체크리스트
- 로그인 전 캐릭터 카드
- 로그인 모달
- 가격표
- FAQ
- 모바일 375px에서 문구 줄바꿈

## 티켓 2. 태어난 시간 입력 안내 추가

### 문제
- 태어난 시간 입력은 아직 `선택`만 있고, `몰라도 분석 가능해`가 바로 보이지 않는다.
- 20대 유저 중 태어난 시간을 모르는 경우가 많아, 이 지점에서 불필요한 부담을 느낄 수 있다.

### 요구사항
- 태어난 시간 선택 영역 근처에 짧은 안내 문구를 추가한다.
- 시간 미입력 시에도 분석 가능하다는 점을 분명히 알려준다.
- 단, 시간이 있으면 더 정밀해진다는 정보도 부드럽게 전달한다.

### 권장 카피
- `태어난 시간을 몰라도 분석 가능해. 알면 더 정밀하게 볼 수 있어.`
- `시간은 선택사항이야. 모르면 비워둬도 돼.`
- `태어난 시간을 모르면 일단 비워두고 시작해도 괜찮아.`

### 수용 기준
- 태어난 시간 입력 영역에서 안내 문구가 바로 보인다.
- 시간 미입력 시 사용자가 막혔다고 느끼지 않는다.
- 기존 분석 플로우와 유효성 검사가 깨지지 않는다.

### QA 체크리스트
- 태어난 시간 입력값 있음
- 태어난 시간 미입력
- 모바일 키보드/선택 UI
- 오류 상태
- 안내 문구 줄바꿈

## 티켓 3. 가격표 배지 시각 분리감 보강

### 문제
- 접근성은 개선됐지만 화면상으로는 `별 70개인기`, `별 250개최고 가성비`처럼 붙어 보일 여지가 있다.
- 세심한 UX 관점에서는 별 개수와 배지 사이에 더 명확한 간격/구분이 필요하다.

### 요구사항
- 별 개수와 배지 사이의 시각적 간격을 더 확보한다.
- 배지는 별도 pill로 보이게 하고, 줄바꿈이 생겨도 붙어 읽히지 않게 한다.
- 접근성 라벨도 계속 자연스럽게 유지한다.

### 권장 UI 방향
- `별 70개`와 `인기` 배지 사이 `gap` 명확화
- 모바일에서는 배지를 다음 줄로 내려도 자연스럽게 보이게 처리
- 배지 앞에 숨김 텍스트로 `추천 태그:` 같은 구분을 넣는 방식도 검토

### 수용 기준
- 화면상 `별 70개인기`처럼 붙어 보이지 않는다.
- `별 250개최고 가성비`처럼 붙어 보이지 않는다.
- 스크린리더 읽기와 시각 디자인이 모두 자연스럽다.

### QA 체크리스트
- 375px 모바일
- 1280px 데스크톱
- 브라우저 확대 125%, 150%
- VoiceOver 또는 접근성 트리

## 티켓 4. 현우 캐릭터 위협감 추가 완화

### 문제
- 현우의 `위험 경고`, `미리 알아야 해`는 후킹은 되지만 일부 유저에게 겁주는 사주앱처럼 느껴질 수 있다.
- 전체 톤이 차분해진 만큼 현우도 덜 위협적인 직설 캐릭터로 맞추는 것이 좋다.

### 요구사항
- 현우의 카드 태그, 설명, quote, 원본 프롬프트에서 위협적인 표현을 더 줄인다.
- `경고`보다 `주의 포인트`, `체크 포인트`, `미리 정리` 같은 표현을 우선한다.

### 권장 카피
- `조심할 포인트를 먼저 짚어줄게`
- `미리 알면 덜 흔들리는 흐름이 있어`
- `좋은 흐름과 주의할 흐름을 같이 정리해줄게`
- `위험 경고` 대신 `주의 포인트`

### 수용 기준
- 현우 카드가 겁주기보다 차분한 분석가로 느껴진다.
- 금지/주의 표현 검색에서 `위험 경고`, `겁`, `당하는`, `니 사주` 같은 톤이 남아 있지 않다.
- AI 첫 답변 샘플도 위협감 없이 직설적이다.

### QA 체크리스트
- 랜딩 캐릭터 카드
- 로그인 전 카드 클릭 모달
- 채팅 첫 안내
- 무료/유료 답변 샘플
- 금지 표현 검색 테스트

## 티켓 5. 랜딩 인지 순서 점검

### 문제
- 랜딩에서 캐릭터 카드가 무료 맛보기/가격보다 먼저 나오면, 일부 유저는 가격 확인 전에 카드를 누르고 로그인 모달을 볼 수 있다.
- 다만 가격이 바로 아래에 있어 큰 문제는 아니다.

### 요구사항
- 실제 화면 순서에서 무료 맛보기/가격 정보가 충분히 빨리 인지되는지 점검한다.
- 캐릭터 카드가 먼저 보이는 구조를 유지한다면, 카드 자체에 `3회 무료`, `가격은 아래에서 확인` 같은 부담 완화 힌트를 넣는 방식을 검토한다.
- 무조건 순서 변경이 아니라, 스크롤/첫 viewport/모달 경험을 기준으로 판단한다.

### 수용 기준
- 카드를 누르기 전 또는 누른 직후 사용자가 무료 혜택을 이해할 수 있다.
- 로그인 모달이 가격 정보를 숨기는 느낌을 주지 않는다.
- 가격표는 첫 화면 또는 아주 짧은 스크롤 안에서 확인 가능하다.

### QA 체크리스트
- 375px 모바일 첫 화면
- 390px iPhone 계열
- 1280px 데스크톱
- 캐릭터 카드 클릭 후 로그인 모달
- 가격표 도달 스크롤 거리

## 티켓 6. 장기 사용감을 위한 컬러 피로도 완화 검토

### 문제
- 전체 컬러가 보라 계열에서 많이 정리되어 현재는 괜찮다.
- 다만 오래 쓰는 앱 느낌을 내려면 일부 포인트에 따뜻한 중립색이 섞이면 덜 피곤할 수 있다.

### 요구사항
- 전면 리디자인이 아니라 일부 포인트 컬러만 검토한다.
- 상담형 앱의 안정감, 따뜻함, 신뢰감을 해치지 않는 범위에서 중립색을 섞는다.
- 기존 브랜드 톤과 충돌하지 않아야 한다.

### 권장 방향
- 배경/구분선/보조 배지에 stone, warm gray, soft cream 계열을 부분 사용
- 주요 CTA는 유지하되 보조 정보 영역만 따뜻하게 완화
- 보라/강한 대비를 반복적으로 쓰는 영역을 줄인다.

### 수용 기준
- 전체 브랜드가 흐려지지 않는다.
- 장시간 봐도 피로감이 줄어든다.
- 모바일에서 명도 대비가 충분하다.

### QA 체크리스트
- 랜딩 첫 화면
- 캐릭터 카드 영역
- 가격표
- 입력폼
- 다크/라이트 톤 충돌 여부

## 최종 QA 시나리오
- 신규 유저가 랜딩에서 캐릭터 카드와 무료 혜택을 본다.
- 카드에 `3회 무료`가 직관적으로 보인다.
- 가격표에서 배지와 별 개수가 붙어 보이지 않는다.
- 태어난 시간을 몰라도 입력을 계속 진행할 수 있음을 바로 이해한다.
- 현우 캐릭터가 직설적이지만 겁주지 않는다.
- 전체 컬러가 차분하고 오래 봐도 피곤하지 않다.

## 개발 원칙
- Codex는 PM 역할만 수행한다.
- 소스코드 수정은 개발자가 진행한다.
- 개발자는 TDD 원칙에 따라 검색 기반 회귀 테스트 또는 UI 검증 기준을 먼저 추가한다.
- 문구 변경은 금지 표현 검색 테스트를 포함한다.
