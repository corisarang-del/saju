# 배포후 PC 폰 디자인개선 수정후재리뷰

- 일자: 2026-07-24
- 대상: `https://monthlysaju.vercel.app/`
- 확인 경로: `/ko`, `/ko/reading`
- 확인 환경: PC 1280x720, 모바일 390x844
- 원칙: 코딩 없이 배포된 production 화면만 디자인 리뷰

## 결론

릴리즈를 막을 정도의 디자인 깨짐은 없지만, 이전에 전달했던 주요 디자인 개선점 중 일부는 production 기준으로 아직 해결되지 않았다.

특히 PC 캐러셀 시작 위치, PC H1 줄바꿈, 모바일 첫 CTA 가시성, 입력폼 보조 텍스트 대비, analytics 404가 계속 재현된다. 개발자가 로컬에서 수정했다면 production 재배포 또는 캐시/배포 반영 상태를 먼저 확인해야 한다.

## 재리뷰 Findings

### P1. PC 캐릭터 캐러셀 첫 카드 잘림이 아직 남아 있음

#### 재현

1. PC 1280x720에서 `https://monthlysaju.vercel.app/ko` 접속
2. 첫 화면 캐릭터 캐러셀의 왼쪽 시작 지점 확인

#### 현재 상태

- 왼쪽 로그인 사이드바 바로 옆에서 캐러셀 첫 카드가 반쯤 잘려 보인다.
- `하나` 카드가 메인 카드처럼 보이지만, 왼쪽에 이전 카드 조각이 크게 노출되어 정렬이 어긋난 인상을 준다.

#### 기대 상태

- 첫 활성 카드가 사이드바에 걸리지 않는다.
- peek 효과가 의도라면 양쪽 edge fade 또는 충분한 gutter로 의도가 명확해야 한다.

#### 수용 기준

- PC 첫 화면에서 캐릭터 카드가 사이드바와 시각적으로 충돌하지 않는다.
- 첫 카드/활성 카드/다음 카드의 관계가 한눈에 이해된다.

### P1. 모바일 첫 화면 `대화하기` CTA가 아직 첫 viewport 밖에 있음

#### 재현

1. 모바일 390x844에서 `https://monthlysaju.vercel.app/ko` 접속
2. 쿠키 미동의 첫 진입 상태에서 첫 viewport 확인

#### 현재 상태

- 쿠키 안내, 내비게이션, H1, 설명문, 캐릭터 이미지, 하단 탭은 보인다.
- 하지만 핵심 행동인 `대화하기` CTA는 첫 화면에 보이지 않는다.

#### 기대 상태

- 모바일 첫 진입에서 첫 캐릭터 카드의 CTA 또는 명확한 행동 유도 영역이 viewport 안에 들어와야 한다.

#### 수용 기준

- 쿠키 미동의 상태에서도 390x844 첫 화면에서 `대화하기` 또는 CTA 바로 위 행동 영역이 보인다.
- 하단 탭과 카드 CTA가 서로 밀어내지 않는다.

### P2. PC H1 마지막 단어 외톨이 줄바꿈이 아직 남아 있음

#### 재현

1. PC 1280x720에서 `/ko` 접속
2. H1 `캐릭터를 고르면 오늘의 흐름부터 먼저 정리해줄게` 확인

#### 현재 상태

- 마지막 `게` 또는 `줄게` 단위가 아래 줄로 떨어져 타이포그래피 마감이 어색하다.

#### 기대 상태

- PC 주요 폭에서 마지막 1~2글자만 외톨이처럼 떨어지지 않는다.
- 문장 의미 단위로 자연스럽게 줄바꿈된다.

### P2. `/ko/reading` 입력폼 placeholder/helper text 대비가 아직 약함

#### 재현

1. 모바일 390x844에서 `https://monthlysaju.vercel.app/ko/reading` 접속
2. 생년월일 placeholder, 예시 텍스트, 태어난 시간 placeholder 확인

#### 현재 상태

- 입력폼 구조와 신뢰 문구는 좋다.
- 다만 `1990`, `03`, `21`, `시간을 선택하세요 (선택)` 같은 placeholder가 여전히 흐릿하다.

#### 기대 상태

- placeholder와 helper text가 차분한 톤을 유지하면서도 실제 폰에서 읽기 쉬워야 한다.

#### 수용 기준

- 모바일 화면에서 생년월일 예시와 선택 안내가 눈을 찡그리지 않아도 읽힌다.
- 입력값, placeholder, 보조 문구의 시각 위계가 명확하다.

### P2. production analytics 404가 계속 남아 있음

#### 재현

1. `https://monthlysaju.vercel.app/ko` 또는 `/ko/reading` 접속
2. 브라우저 console 확인

#### 현재 상태

- `https://monthlysaju.vercel.app/api/analytics/track` 요청이 404를 반환한다.

#### 기대 상태

- production 첫 진입에서 console error가 없어야 한다.
- analytics를 꺼둔 상태라면 클라이언트 호출도 비활성화하거나 no-op 성공 응답으로 처리해야 한다.

#### 수용 기준

- `/ko`, `/ko/reading` production console error 0건.
- analytics 수집 또는 비활성화 정책이 일관된다.

## 좋아진 점

- 쿠키 안내는 여전히 콘텐츠를 덮지 않는다.
- 개인정보 링크는 `/ko/privacy-policy`로 locale을 유지한다.
- 입력폼의 개인정보/성별/태어난 시간 안내 문구는 신뢰감을 준다.
- 캐릭터 이미지 품질과 전체 브랜드 톤은 안정적이다.

## 개발자 액션

1. production 배포가 최신 수정본인지 먼저 확인한다.
2. PC 캐러셀 시작 위치와 H1 줄바꿈을 1280x720 기준으로 다시 조정한다.
3. 모바일 첫 화면에서 CTA가 보이도록 카드 높이, H1, 여백, 쿠키 안내 높이를 함께 조정한다.
4. 입력폼 placeholder/helper text 대비를 한 단계 올린다.
5. `/api/analytics/track` production 404를 제거한다.

## QA 체크리스트

- PC 1280x720 `/ko`: H1 마지막 단어가 외톨이 줄로 떨어지지 않음
- 모바일 390x844 `/ko`: 쿠키 미동의 상태에서도 첫 CTA 접근성이 개선됨
- 모바일 390x844 `/ko/reading`: placeholder/helper text가 읽기 쉬움
- production console: `/api/analytics/track` 404 없음
