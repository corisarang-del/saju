# 배포웹사이트 디자인개선 수정후리뷰

## 기준

- 일시: 2026-07-24 KST
- 대상: `https://monthlysaju.vercel.app/ko`, `https://monthlysaju.vercel.app/ko/reading`
- 검증 뷰포트: PC 1280x720, PC 1366x768, 모바일 390x844
- 방식: production 화면 캡처, DOM 좌표, 콘솔, 가로 오버플로우 확인
- 소스코드 변경: 없음

## 결론

릴리즈 차단급 디자인 이슈는 없다. 직전 핵심 문제였던 PC 1280x720 `현우` 카드 이름 클리핑은 해결됐다.

## 해결 확인

- PC 1280x720 `/ko`: 가시 카드 이름 라인이 viewport 안에 들어온다.
- PC 1280x720 `/ko`: `현우` 이름 영역은 이전처럼 `bottom=737`로 fold를 넘지 않고, 최신 측정 기준 `y=509`, `bottom=537` 위치까지 올라왔다.
- PC 1366x768 `/ko`: `현우`, `하나`, `민준`, `인영` 이름이 모두 온전히 보인다.
- 모바일 390x844 `/ko`: H1 단어 중간 줄바꿈이 없다.
- 모바일 390x844 `/ko`: 첫 가시 카드와 `대화하기` CTA가 하단 탭 위에 안정적으로 보인다.
- 모바일 390x844 `/ko/reading`: 시간 선택 placeholder 대비가 날짜 placeholder와 거의 같은 톤으로 정리됐다.
- 모바일 390x844 `/ko/reading`: `다음` CTA는 `y=752`, `bottom=812`로 viewport 안에 안정적으로 보인다.
- `/ko`, `/ko/reading`: 콘솔 error/warning 0건.
- PC/모바일: 가로 오버플로우 없음.

## 남은 개선점

### P3. PC 1280x720에서 카드 하단 액션 영역이 fold에 거의 맞닿는다

- 위치: `/ko` PC 1280x720 캐릭터 캐러셀
- 현상: 이름 클리핑은 해결됐지만 카드 전체 bottom이 viewport 하단과 거의 맞닿아, 카드 하단의 설명/액션 영역은 첫 화면에서 여유 있게 보이지 않는다.
- 영향: 카드 전체가 클릭 가능한 구조라 기능 차단은 아니다. 다만 사용자가 명시적 `대화하기` 버튼을 첫 화면에서 바로 발견하는 힘은 모바일보다 약하다.
- 권장: `max-height: 760px` 전후 PC에서 카드 높이, 히어로 상단 여백, 본문 간격 중 하나를 소폭 줄여 카드 하단 액션 영역에 16~24px 정도의 breathing room을 확보한다.

### P3. 캐러셀 시작 위치가 뷰포트마다 다르게 느껴진다

- 위치: `/ko` PC/모바일 캐릭터 캐러셀
- 현상: 새로고침/스냅 상태에 따라 첫 가시 카드가 `현우`, `인영` 등으로 달라져 보일 수 있다.
- 영향: 이름 잘림 문제는 아니지만, 사용자가 “첫 카드가 무엇인지”를 예측하기 어렵게 느낄 수 있다.
- 권장: 첫 진입 기준 시작 카드를 고정하거나, 의도된 자동 위치라면 pagination active state와 카드 위치가 항상 일치하는지 QA한다.
