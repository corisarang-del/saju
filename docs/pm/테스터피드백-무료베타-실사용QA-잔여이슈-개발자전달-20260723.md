# 테스터 피드백 무료베타 실사용 QA 잔여이슈 개발자전달 20260723

## 결론

이번 테스터 피드백 기준으로 아직 무료 베타 배포 최종 승인은 어렵다.

이유는 결제 때문이 아니다. 결제 기능은 이번 배포에서 비활성화해도 된다. 하지만 무료 베타의 핵심인 실제 Supabase 기반 무료 상담 API QA가 아직 끝나지 않았다. `sfpwgywcmhgilrqearsz.supabase.co` DNS가 `ENOTFOUND`로 잡히지 않아 임시 인증 사용자 생성 단계까지 가지 못했고, 실제 DB 기준 무료 첫 상담, 별 지급/차감, 거래 로그, assistant 저장 품질이 미확인이다.

추가로 무인증 채팅 API 401 응답 형식, 무료 베타와 충돌하는 결제 가능 암시 문구, 사이드바 접근성 텍스트 붙어 읽힘 문제가 새로 남았다.

## 현재 통과/차단 상태

### 통과 또는 개선 확인

- redirect 정상
- Playwright 렌더링 정상
- 콘솔 에러 없음
- QA 스크립트가 Supabase DNS 실패를 구조화해서 보고함
- Gemini QA는 통과

### 차단

아래 실제 API QA가 모두 Supabase DNS 실패로 중단됐다.

```bash
env QA_SCENARIO=free-only node scripts/qa-live-api-check.mjs
node scripts/qa-live-api-check.mjs
```

실패 원인:

- host: `sfpwgywcmhgilrqearsz.supabase.co`
- error: `ENOTFOUND`
- 단계: Supabase 임시 인증 사용자 생성 전
- 결과: 실제 DB 기반 상담 플로우에 도달하지 못함

## 아직 실제 DB 기준으로 확인 못 한 것

- 무료 첫 상담 실제 응답

- 오늘피드/스냅샷 실제 반영

## 새로 남은 이슈

### P0. 실제 무료/전체 API QA 미완료

무료 베타 배포에서 이건 blocker다.

Paddle 결제 QA는 이번 배포에서 제외 가능하지만, Supabase 기반 무료 상담 QA는 제외하면 안 된다. 무료 베타의 핵심 기능이기 때문이다.

개발자 액션:

1. Supabase project ref `sfpwgywcmhgilrqearsz`가 실제 활성 상태인지 확인한다.
2. `NEXT_PUBLIC_SUPABASE_URL`이 올바른 project URL인지 확인한다.
3. 현재 로컬 네트워크, CI, Vercel production 각각에서 DNS가 정상인지 확인한다.
4. DNS 정상 환경에서 아래 명령을 재실행한다.

```bash
env QA_SCENARIO=free-only node scripts/qa-live-api-check.mjs
node scripts/qa-live-api-check.mjs
```

통과 기준:

- 임시 인증 사용자 생성 성공
- 무료 3회 별 지급 확인
- 무료 첫 상담 저장
- 별 1개 차감 또는 예약 후 확정
- 실패 시 환불
- `chat_message:-1` 거래 로그 저장
- `chat_used` 증가
- 저장된 assistant 품질 통과

### P1. 무인증 채팅 API 401 본문이 JSON이 아님

테스터 관찰:

- 무인증 채팅 API가 HTTP 401로 막히는 건 정상이다.
- 하지만 본문이 JSON이 아니라 `Unauthorized` 텍스트다.
- 클라이언트 오류 처리 일관성 확인이 필요하다.

관련 확인 대상:

- `src/app/api/saju/chat/route.ts`
- 클라이언트 채팅 transport 오류 처리
- 공통 API error parser

개발자 액션:

- `/api/saju/chat` 무인증 응답을 다른 사주 API처럼 JSON 형태로 맞출지 결정한다.
- 권장 응답:

```json
{
  "error": "unauthorized",
  "message": "로그인이 필요해."
}
```

- 클라이언트가 text/plain 401과 JSON 401을 모두 안전하게 처리하는지 확인한다.
- 회귀 테스트는 “무인증 채팅 요청이 401 JSON error를 반환한다”처럼 사용자 관찰 가능한 응답 형식을 검증한다.

### P1. 무료 베타인데 결제 가능 암시 문구가 일부 남아 있음

테스터 관찰:

- 무료 베타 배포인데 일부 화면/메타 문구가 여전히 결제 가능처럼 읽힐 수 있다.

확인 대상:

- 랜딩 가격 패널
- FAQ/JSON-LD FAQ
- 푸터
- 코인샵
- 채팅 paywall
- 채팅 입력 disabled placeholder
- 월간 리포트/종합 백서 CTA
- 약관/개인정보/환불정책의 결제 문구

현재 의심 문구 예:

- “이후에는 별을 충전하여 계속 상담할 수 있습니다”
- “결제는 Paddle의 안전한 결제 시스템으로 처리됩니다”
- “별을 충전해주세요”
- “별 충전하러 가기”

개발자 액션:

- `PAYMENTS_ENABLED=false` 상태에서는 결제 가능/충전 가능처럼 읽히는 문구를 숨기거나 무료 베타 안내로 바꾼다.
- “준비 중” 문구는 필요한 곳에만 쓰고, 핵심 상담 흐름에서는 결제 기대를 만들지 않는다.
- JSON-LD FAQ도 실제 화면 정책과 맞춰야 한다.

권장 문구:

- “지금은 무료 상담 베타로 운영 중이야.”
- “추가 충전은 정식 결제 기능 안정화 후 열릴 예정이야.”
- “가입 후 3회 무료 상담을 먼저 이용할 수 있어.”

### P2. Playwright 스냅샷에서 `기록을저장하고`처럼 붙어 보임

테스터 관찰:

- Playwright 스냅샷에서 `기록을저장하고`처럼 띄어쓰기가 붙어 보인다.
- 코드상 aria-label이 있어도 실제 화면/접근성 트리에서 붙어 읽히면 신뢰와 접근성이 떨어진다.

확인 대상:

- `src/components/saju/landing/LoginSidebar.tsx`
- 사이드바 로그인 안내 문구
- 모바일/데스크톱 스냅샷
- 접근성 이름

개발자 액션:

- 시각 텍스트가 줄바꿈되더라도 스냅샷/접근성 트리에서 단어가 붙지 않게 처리한다.
- 필요하면 문장을 한 줄 텍스트로 유지하거나, 줄바꿈 사이에 명시적 공백을 둔다.
- 테스트는 `대화 기록을저장` 같은 붙은 문자열이 렌더링 결과에 남지 않는지 확인한다.

4. 무료 베타 결제 암시 문구 제거
5. 사이드바 `기록을저장하고` 붙어 읽힘 수정
6. Gemini QA 통과 후에도 고민별 행동 제안 사람이 한 번 더 샘플링

## PM 판단

현재 제품은 결제 비활성화 방향 자체는 맞다. 하지만 무료 베타로 내보내려면 실제 Supabase 기반 상담/별/저장 흐름이 확인돼야 한다.

이번 피드백의 핵심은 “Paddle 때문에 못 나간다”가 아니라 “무료 베타 핵심 API가 DNS 차단으로 아직 검증되지 않았다”다. 개발자는 코드 수정 전에 Supabase DNS/환경부터 풀고, 그 다음 401 응답 형식과 무료 베타 문구/접근성 이슈를 닫아야 한다.
