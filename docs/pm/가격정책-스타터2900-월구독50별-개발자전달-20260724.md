# 가격정책 스타터2900 월구독50별 개발자전달 20260724

## 결론

가격 정책을 아래처럼 조정한다.

- 스타터: `10별 2,900원`
- 월간 멤버십: `월 9,900원 / 매월 50별 지급`
- 무료: 가입 후 `3회 무료` 유지
- 채팅: `1별 = 메시지 1회` 유지
- 기존 30/70/250별 패키지는 이번 지시에서는 유지한다.

이번 변경의 목표는 원가 절감이 아니라 **첫 결제 진입 장벽을 낮추고, 월구독을 “매일 쓰는 상담 플랜”으로 더 납득되게 만드는 것**이다.

## 중요한 PM 판단

### 스타터는 첫 결제 1회 한정으로 두는 게 좋다

`10별 2,900원`은 `30별 9,900원`보다 별 단가가 낮다.

- 스타터: 290원/별
- 30별: 330원/별

따라서 스타터를 무제한 반복 구매 가능하게 두면 30별 패키지의 존재 이유가 약해진다.

개발자 액션:

- 가능하면 `stars10`은 첫 결제 1회 한정 상품으로 처리한다.
- 첫 결제 1회 한정 구현이 당장 어렵다면, 최소한 UI/카피에서 `처음 써보는 분을 위한 스타터`로 포지셔닝한다.
- 결제 재오픈 시에는 서버에서도 `stars10` 반복 구매 제한을 둘지 확인한다.

권장 카피:

- `처음 써보는 분을 위한 10별 스타터`
- `첫 결제 부담을 낮춘 가벼운 시작`
- `무료 3회 뒤 조금 더 이어보고 싶을 때`

피해야 할 카피:

- `최고 가성비`
- `가장 저렴한 별 단가`
- `계속 이 가격으로 상담`

### 월구독은 50별로 올려도 괜찮다

월 9,900원에 40별은 “조금 할인된 포인트 묶음”처럼 보일 수 있다.

월 9,900원에 50별이면 사용자가 이렇게 이해하기 쉽다.

- 매일 1회 오늘 흐름/상담: 약 30별
- 월간 리포트 상세: 3별
- 남은 후속 질문: 약 17별

즉 `월 50별`은 “매일 한 번 들어와도 부족하지 않은 플랜”으로 설명할 수 있다.

권장 카피:

- `매일 한 번씩 내 흐름을 보고, 남는 별로 고민을 이어가는 플랜`
- `월 50별로 오늘피드, 월간 리포트, 후속 상담까지 여유 있게`
- `자주 들어오는 유저를 위한 월간 상담권`

## Gemini 토큰 비용 기준 판단

현재 모델은 `gemini-2.5-flash-lite` 기준이다.

현재 코드 구조상 추정 비용:

- 첫 상담: 대략 0.5~1원 수준
- 품질 재시도 3회 최악: 대략 1.5~3원 수준
- 일반 후속 질문: 대략 1~3원 수준
- 긴 대화/누적 컨텍스트: 대략 3~8원 수준

따라서 현재 가격 정책에서 AI 원가는 충분히 감당 가능하다. 가격은 토큰 원가보다 다음 요소로 결정해야 한다.

- 첫 결제 심리
- 상담 품질 체감
- 캐릭터별 답변 차이
- 이전 상담 기억 체감
- 매일 들어올 이유
- 결제 실패/별 지급 오류 없는 신뢰

## 사용량 정책

이번 변경에서 별 차감 정책은 바꾸지 않는다.

- 채팅 메시지 1회: 1별
- 월간 리포트 상세판: 3별
- 종합 사주 백서: 5별
- 가입 후 무료 상담: 3회

다만 운영 데이터가 쌓이면 아래는 재검토한다.

- 종합 사주 백서 5별이 너무 싼지
- 긴 답변/고급 리포트가 일반 채팅과 같은 1별이어도 되는지
- 누적 대화가 길어질수록 provider input token이 커지는 문제

## 개발자 수정 대상

우선 확인 대상:

- `src/lib/monthly-saju/pricing.ts`
- Paddle product/price mapping
- landing 가격 패널
- coin shop
- FAQ
- JSON-LD
- terms/refund policy
- admin 표시
- 결제 QA 스크립트
- 가격 정책 테스트

변경값:

```ts
{ type: "stars10", stars: 10, price: 2900, badge: "스타터" }

export const MONTHLY_MEMBERSHIP = {
  type: "monthlyMembership",
  name: "월간 멤버십",
  stars: 50,
  price: 9900,
  description: "매월 50별 지급, 오늘피드/월간 리포트 우선 노출",
} as const;
```

주의:

- Paddle price id는 기존 3,900원 상품과 달라져야 한다.
- 결제 재오픈 전 Paddle Dashboard에도 `stars10 2,900원` price를 새로 만들거나 기존 price를 확실히 교체해야 한다.
- webhook은 customData가 아니라 실제 `items.price.id` 기준 지급을 유지해야 한다.
- 가격 상수와 Paddle 매핑이 어긋나면 결제 금액과 별 지급량이 불일치할 수 있다.

- `keeps_signup_free_three_consultations`
- `uses_same_price_policy_for_landing_faq_jsonld_terms_and_paddle_mapping`

결제 재오픈 전 테스트:

- `stars10` 결제 price id가 실제 2,900원 상품과 매칭된다.
- `monthlyMembership` 결제 price id가 월 9,900원/50별 상품과 매칭된다.
- webhook은 실제 Paddle price id 기준으로 10별 또는 월 50별을 지급한다.
- 싼 상품 결제에 다른 customData가 들어와도 지급량이 바뀌지 않는다.
