# 현재 프로젝트 PM 인수인계

## 역할 원칙
- 앞으로 Codex는 이 프로젝트에서 PM 역할로만 움직인다.
- 소스코드는 직접 수정하지 않는다.
- 구현 요청은 개발자에게 넘길 수 있는 문서로 정리한다.
- 문서 형식은 요구사항, 배경, 우선순위, 수용 기준, QA 체크리스트, 리스크를 포함한다.

## 프로젝트 개요
- 프로젝트명: 월간사주
- 한 줄 설명: 먼저 챙겨주는 사주친구이자 대화형 인생운 코치
- 주요 고객: 20대 여성 고객
- 핵심 경험: 캐릭터 선택, 오늘피드, 사주 기반 채팅, 월간 리포트, 별 기반 결제
- 기술 맥락: Next.js App Router, React 19, TypeScript, Supabase, Vercel AI SDK, Google Gemini, Paddle, pnpm

## 현재까지 확인된 제품 방향
- 랜딩은 가입 전 신뢰 형성이 중요하다.
- 사용자는 가입 전에 결과 품질을 한 문단이라도 미리 보고 싶어한다.
- 가격과 별 사용 기준은 숨기지 말고 랜딩에서 바로 공개해야 한다.
- 캐릭터는 개성이 있어야 하지만, 20대 여성 고객에게 무례하거나 하대처럼 들리면 안 된다.
- 고민 선택지는 사주 카테고리 용어보다 생활 언어가 더 적합하다.

## 20대 여성고객 피드백 정리
- 랜딩에 “무료 맛보기 결과”를 먼저 보여줘야 한다.
- 가입 전에 한 문단 샘플이 필요하다.
- 가격과 별 사용 기준을 공개해야 한다.
- 기준 문구 예: `1별 = 메시지 1회, 가입 후 3회 무료, 추가 가격은 ...`
- 푸터/환불 이메일 placeholder는 실제 정보로 바꿔야 한다.
- 캐릭터 말투는 여성 유저 기준으로 조금 부드럽게 조정해야 한다.
- 직설 캐릭터도 무례하지 않게 만들어야 한다.
- 고민 선택지는 생활 언어로 바꿔야 한다.
- 예시: `썸/재회`, `이직/퇴사`, `돈 모으기`, `번아웃`, `친구/가족관계`

## 현재 반영된 상태
- 랜딩 상단에 가입 전 확인 가능한 무료 맛보기 결과 한 문단이 추가됐다.
- 별 가격과 사용 기준이 랜딩과 FAQ에 노출된다.
- 별 기준은 `1별 = 메시지 1회, 가입 후 3회 무료`다.
- 현재 가격 패키지는 `30별 9,900원`, `70별 19,900원`, `250별 39,900원`이다.
- 문의/환불 이메일은 `corisarang@gmail.com`으로 통일됐다.
- 고민 라벨은 생활 언어 중심으로 바뀌었다.
- 캐릭터 말투는 실제 노출/호출 단계에서 하대감 있는 표현을 줄이도록 정제됐다.

## PM 리뷰에서 확인된 주요 리스크
- 공개 런칭 전 개인정보 접근 경계 확인이 필요하다.
- `readingId` 기반 조회는 서버 코드에서도 `user_id` 소유자 조건을 명시하는 방향이 안전하다.
- 결제/별 차감은 원자성, 중복 웹훅, 거래 이력 정합성 확인이 필요하다.
- 약관/개인정보/환불 문서는 실제 사업자 정보 확정 후 다시 검수해야 한다.
- 전체 ESLint는 기존 React 19 규칙 위반 때문에 아직 실패한다.
- Next 빌드는 Google Fonts fetch 실패 리스크가 있어 로컬 폰트 전환 검토가 필요하다.
- README에는 npm 안내가 남아 있어 pnpm 전용 정책과 충돌한다.

## 개발자에게 넘길 때 기본 티켓 형식
```md
# 티켓 제목

## 배경
- 고객 피드백 또는 PM 판단 근거

## 요구사항
- 구현해야 할 동작

## 수용 기준
- 사용자가 확인할 수 있는 완료 조건

## QA 체크리스트
- 모바일/데스크톱
- 로그인 전/후
- 결제 전/후
- 실패/빈 상태

## 리스크
- 보안, 결제, 법무, 데이터 정합성

## 비고
- 소스코드는 개발자가 수정한다.
```
