# 배포반영후 PC 폰 디자인개선 재리뷰

- 일자: 2026-07-24
- 대상: `https://monthlysaju.vercel.app/`
- 확인 경로: `/ko`, `/ko/reading`
- 확인 환경: PC 1280x720, 모바일 390x844
- 원칙: 코딩 없이 production 화면만 디자인 리뷰

## 결론

배포 반영 후 다시 확인한 결과, 이전에 재현되던 주요 디자인 이슈 대부분은 해결됐다. 릴리즈 차단급 디자인 이슈는 없다.

이전 재리뷰에서 “주요 개선점 일부가 그대로 재현된다”고 판단했던 이유는 최신 수정본이 아직 production에 배포되지 않았기 때문으로 정정한다.

## 해결 확인

### PC H1 외톨이 줄바꿈 해결

- 기존: `줄게` 또는 마지막 글자가 외톨이 줄처럼 떨어졌다.
- 현재: H1이 `캐릭터를 고르면 / 오늘의 흐름부터 먼저 정리해줄게`에 가깝게 의미 단위로 정리되어 보인다.
- 상태: 해결

### analytics 404 해결

- 기존: `/api/analytics/track` production 404가 `/ko`, `/ko/reading`에서 발생했다.
- 현재: Playwright console 기준 errors 0, warnings 0.
- 상태: 해결

### 입력폼 대비 개선

- 기존: 생년월일 placeholder와 예시 텍스트가 너무 흐렸다.
- 현재: `1990`, `03`, `21`, `예: 1998 / 03 / 21`은 폰에서도 읽기 쉬워졌다.
- 남은 점: `시간을 선택하세요 (선택)`은 여전히 조금 연하지만, 차단급은 아니다.
- 상태: 대부분 해결

### 모바일 `/ko/reading` 하단 CTA 안정화

- 기존: 입력 CTA가 하단에서 가림/접근성 우려가 있었다.
- 현재: 390x844 하단에서 `다음` CTA가 넓고 안정적으로 보인다.
- 상태: 해결

## 남은 개선점

### P2. 모바일 H1이 단어 중간에서 끊김

#### 재현

1. 모바일 390x844에서 `/ko` 접속
2. 쿠키 미동의 첫 진입 상태에서 H1 확인

#### 현재 상태

- `캐릭터를 고르면`이 `캐릭터를 고르 / 면`처럼 단어 중간에서 줄바꿈된다.
- PC H1 줄바꿈은 좋아졌지만, 모바일에서는 새 타이포그래피 마감 이슈가 생겼다.

#### 기대 상태

- `고르면`처럼 의미 있는 단어는 중간에서 쪼개지지 않는다.
- 모바일 H1은 3~4줄 안에서 자연스럽게 읽힌다.

#### 수용 기준

- 390x844에서 `고르면`, `오늘의`, `흐름부터`, `먼저 정리해줄게` 같은 의미 단위가 깨지지 않는다.

### P2. 모바일 첫 CTA가 개선됐지만 완전히 안정적이진 않음

#### 현재 상태

- 첫 카드 내부 문구가 줄어들어 `대화하기` CTA가 첫 viewport 하단에 걸쳐 보인다.
- 다만 하단 탭 바로 위에 붙어 있어 버튼 전체가 여유 있게 보인다는 느낌은 아직 약하다.

#### 기대 상태

- 쿠키 미동의 상태에서도 첫 CTA가 하단 탭에 눌리지 않고, 여백을 갖고 보인다.

#### 수용 기준

- 390x844 첫 화면에서 `대화하기` 버튼 전체가 하단 탭 위로 명확히 보인다.

### P2. PC 캐러셀 왼쪽 peek가 아직 조금 무거움

#### 현재 상태

- 이전처럼 정렬 오류처럼 보이진 않고, fade가 생겨 의도는 읽힌다.
- 다만 1280x720에서는 사이드바 바로 옆 이전 카드 조각이 여전히 크게 보인다.

#### 기대 상태

- peek 효과는 유지하되, 사이드바와 메인 카드 사이의 시각적 충돌을 더 줄인다.

#### 수용 기준

- PC 1280x720에서 왼쪽 카드 조각이 사이드바에 붙어 보이지 않고, 캐러셀의 의도된 depth로 읽힌다.

### P3. 시간 선택 placeholder 대비 미세 보강

#### 현재 상태

- 생년월일 placeholder 대비는 개선됐다.
- `시간을 선택하세요 (선택)`은 여전히 살짝 흐리다.

#### 기대 상태

- optional 입력이라는 느낌은 유지하되, 밝은 화면에서도 읽기 쉬워야 한다.

## 좋아진 점

- production console이 깨끗해졌다.
- PC H1의 정보 단위가 좋아졌다.
- 입력폼의 숫자 placeholder와 예시 텍스트 가독성이 좋아졌다.
- 모바일 카드에서 CTA가 첫 화면 가까이 올라와 전환 동선이 좋아졌다.
- 쿠키 안내는 계속 콘텐츠를 덮지 않고 locale 링크도 유지한다.
