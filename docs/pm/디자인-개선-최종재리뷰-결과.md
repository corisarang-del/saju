# 디자인 개선 최종재리뷰 결과

- 일자: 2026-07-01
- 범위: `/ko`, `/ko/reading` 모바일 390x844, 쿠키 미동의 첫 진입, 이전 디자인 감사 항목 재검증
- 원칙: 소스코드 수정 없이 디자인 리뷰와 문서화만 진행

## 결론

현재 디자인 개선안에서 릴리즈를 막을 만한 P0/P1 디자인 이슈는 발견하지 못했다.

이전에 남았던 상단 fixed 쿠키 배너의 H1/제목 가림 문제는 해결됐다. 쿠키 안내가 문서 흐름 안의 `region`으로 배치되어 첫 화면 콘텐츠를 덮지 않고, 모바일에서도 H1과 입력 제목을 아래로 밀어낸다.

## 해결 확인

1. 쿠키 배너 가림
   - 위치: `src/components/ui/cookie-consent.tsx:31`
   - 확인: 쿠키 안내가 `role="region"`이며 `fixed` overlay가 아니다.
   - `/ko` 390x844에서 쿠키 안내, 내비게이션, H1, 캐릭터 카드가 순서대로 보인다.
   - `/ko/reading` 390x844에서 쿠키 안내가 제목을 덮지 않고, 하단 `다음` 버튼도 가리지 않는다.

2. 개인정보 링크 locale 유지
   - 위치: `src/components/ui/cookie-consent.tsx:3`, `src/components/ui/cookie-consent.tsx:45`
   - 확인: i18n `Link`를 사용하고, 스냅샷에서 `/ko/privacy-policy`로 잡힌다.

3. 랜딩 첫 캐릭터 이미지 LCP 우선순위
   - 위치: `src/components/saju/landing/CharacterCards.tsx:135`
   - 확인: 첫 카드에만 `priority`, `loading="eager"`, `fetchPriority="high"`가 적용되어 있다.
   - Playwright console warning 기준 LCP priority 경고는 재현되지 않았다.

4. 모바일 입력 흐름
   - `/ko/reading` 390x844 하단에서 성별, 달력 구분, `다음` CTA가 모두 터치 가능한 위치에 보인다.
   - 쿠키 안내가 하단 CTA와 겹치지 않는다.

## 남은 리스크

- 현재 회귀 테스트는 문자열 기반 검사가 많다. 실제 390x844 시각 겹침을 Playwright 자동화로 고정하는 편이 안전하다.
- `pnpm exec vitest`는 이전 검증에서 `[ERROR] fetch failed`로 실패한 적이 있어, 현재 환경의 pnpm 실행 경로는 별도 점검이 필요하다.
- 하단 모바일 탭과 캐릭터 카드 CTA가 가까운 구간은 지금은 괜찮지만, 카드 높이/문구가 바뀌면 다시 겹침 위험이 생긴다.
