# 20대 여성 피드백 무료베타 배포보류 문구일관성 개발자전달 20260723

## 결론

이번 20대 여성 유저 관점 피드백 기준으로 무료 베타 배포 승인은 아직 보류가 맞다.

핵심 이유는 두 가지다.

1. Supabase DNS 문제 때문에 실제 무료 상담 API QA가 아직 완료되지 않았다.
2. 무료 베타라고 안내하면서 푸터와 JSON-LD에는 결제/별 충전 가능 문구가 남아 있어 메시지 일관성이 깨진다.

Paddle 결제 기능은 이번 배포에서 비활성화해도 된다. 하지만 무료 베타의 핵심인 실제 상담 API, 무료 3회, 별 지급/차감, 거래 로그, assistant 저장 품질은 실제 DB 기준으로 확인돼야 한다.

## 개발자에게 전달할 핵심 판단

### P0. Supabase DNS 차단으로 실제 무료 상담 API QA 미완료

현재 `sfpwgywcmhgilrqearsz.supabase.co`가 DNS `ENOTFOUND`로 잡히지 않아 실제 QA가 임시 인증 사용자 생성 단계까지 가지 못했다.

따라서 아래 항목은 아직 실제 DB 기준 통과로 볼 수 없다.

- 신규 가입 후 3회 무료 지급
- 무료 첫 상담 응답 생성
- user/assistant 메시지 저장
- 별 예약/차감/환불
- 거래 로그 기록
- `chat_used` 증가
- 저장된 assistant 답변 품질
- 오늘피드/스냅샷 반영

개발자 액션:

1. production/staging/local에서 `NEXT_PUBLIC_SUPABASE_URL`이 실제 활성 Supabase project ref를 가리키는지 확인한다.
2. `sfpwgywcmhgilrqearsz.supabase.co` DNS가 로컬, CI, Vercel 환경에서 모두 정상 resolution 되는지 확인한다.
3. DNS 정상 환경에서 실제 API QA를 다시 실행한다.

재실행 대상:

```bash
env QA_SCENARIO=free-only node scripts/qa-live-api-check.mjs
node scripts/qa-live-api-check.mjs
```

완료 기준:

- 임시 인증 사용자 생성 성공
- 무료 3회 지급 확인
- 무료 첫 상담 성공
- 별 차감 또는 예약 후 확정
- 실패 시 환불
- 거래 로그 저장
- user/assistant 메시지 저장
- assistant 품질 게이트 통과

이 항목이 닫히기 전에는 “무료 베타 배포 승인”이라고 표현하면 안 된다.

### P1. 무료 베타와 결제 가능 문구가 충돌함

현재 무료 베타로 배포하려는 방향과 일부 문구가 서로 다르게 말하고 있다.

특히 아래 영역을 우선 확인해야 한다.

- 푸터
- JSON-LD FAQ
- 랜딩 FAQ
- 코인샵 진입/차단 화면
- 채팅 입력 disabled placeholder
- 채팅 paywall
- 월간 리포트/종합 백서 CTA
- 약관/개인정보/환불정책

문제 예시:

- “이후에는 별을 충전하여 계속 상담할 수 있습니다”
- “결제는 Paddle의 안전한 결제 시스템을 통해 처리됩니다”
- “별을 충전해주세요”
- “별 충전하러 가기”

무료 베타 상태에서 이런 문구가 보이면 사용자는 “결제가 되는 건지, 막힌 건지, 나중에 돈을 내야 하는 건지”를 헷갈릴 수 있다. 20대 여성 유저 관점에서는 이 혼선이 광고 냄새나 숨은 과금처럼 느껴질 수 있다.

개발자 액션:

1. `PAYMENTS_ENABLED=false` 상태에서 노출되는 모든 문구를 다시 확인한다.
2. 결제/충전 가능처럼 읽히는 문구를 무료 베타 안내로 바꾼다.
3. JSON-LD도 실제 화면과 같은 정책을 말하게 맞춘다.
4. 푸터의 Paddle 결제 문구는 결제 기능이 켜진 상태에서만 노출되게 한다.

권장 문구:

- “지금은 무료 상담 베타로 운영 중이야.”
- “가입 후 3회 무료 상담을 먼저 이용할 수 있어.”
- “추가 충전은 정식 결제 기능 안정화 후 열릴 예정이야.”
- “결제 기능은 아직 열려 있지 않아.”

완료 기준:

- 무료 베타 상태에서 결제 가능/충전 가능처럼 읽히는 문구가 보이지 않는다.
- 화면, FAQ, JSON-LD, 푸터, 정책 문서가 모두 같은 상태를 말한다.
- 사용자가 결제 버튼을 찾거나 결제를 기대하지 않는다.

## 배포 판단

### 배포 승인 보류 사유

- 실제 무료 상담 API QA가 Supabase DNS 문제로 미완료
- 무료 베타 메시지와 푸터/JSON-LD 결제 문구 불일치

### 결제 기능 관련 판단

- Paddle 결제 실제 QA는 이번 무료 베타 배포 범위에서 제외 가능하다.
- 결제 모듈은 보존하되 production에서는 비활성화한다.
- 단, 결제 비활성화 상태와 맞지 않는 문구는 반드시 정리해야 한다.

## 개발 우선순위

1. Supabase DNS/환경 정상화
2. 실제 무료 상담 API QA 재실행
3. 푸터 결제 문구 feature flag 정리
4. JSON-LD FAQ 결제/충전 문구 정리
5. 랜딩/FAQ/placeholder/paywall/정책 문서 문구 일관성 확인

## PM 메모

이번 피드백의 핵심은 “아직 제품이 나쁘다”가 아니다. 오히려 화면 설득력은 많이 올라왔고, 남은 건 출시 전 신뢰 확인이다.

무료 베타는 무료로 보이는 것보다 “실제로 무료 핵심 플로우가 검증됐고, 어디에도 결제 가능한 척하지 않는다”가 더 중요하다. 개발자는 Supabase QA를 먼저 닫고, 그다음 무료 베타 메시지를 푸터/JSON-LD까지 끝까지 맞춰야 한다.
