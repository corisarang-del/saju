# 20대 여성고객 9점대 신뢰도 개선 개발자 전달

## 배경
- 20대 여성 유저 관점에서 현재 서비스는 “광고 같은 사주앱” 느낌이 많이 줄었다.
- 지금은 “내 고민을 차분히 정리해주는 상담형 앱”에 가까워졌다는 긍정 피드백이 들어왔다.
- 다음 개선 목표는 9점대 체감 완성도다.
- 핵심 개선 포인트는 쿠키 문구, 성별 입력 안내, 현우 톤 완화, 가격표 접근성, 로그인 사이드바 우선순위다.

## 우선순위 요약
1. 쿠키 배너 문구와 위치 부담 완화
2. 성별 입력에 사주 계산상 필요한 이유 안내
3. 현우 캐릭터의 위협적 후킹 문구 완화
4. 가격표의 배지 접근성 개선
5. 데스크톱 로그인 사이드바가 서비스 가치보다 먼저 보이는 문제 점검

## 티켓 1. 쿠키 배너 문구와 시선 부담 완화

### 문제
- 현재 쿠키 배너 문구가 `더 나은 서비스를 위해 쿠키를 사용합니다`처럼 일반적이다.
- 입력 화면 하단에서 시선을 뺏어, 사용자가 생년월일/성별 입력에 집중하기 어렵다.
- 현재 서비스 톤에는 “필요한 쿠키만 쓴다”는 신뢰형 문구가 더 잘 맞는다.

### 요구사항
- 쿠키 배너 문구를 필수 쿠키 중심으로 바꾼다.
- 입력 화면에서 배너가 핵심 CTA나 입력 흐름을 방해하지 않도록 위치/크기/대비를 점검한다.
- 쿠키 배너가 과한 마케팅 문구처럼 보이지 않게 한다.

### 권장 카피
- `로그인과 서비스 이용에 필요한 쿠키만 사용해.`
- `서비스 이용에 필요한 최소한의 쿠키만 사용해.`
- `필수 쿠키만 사용하고, 자세한 내용은 개인정보처리방침에서 볼 수 있어.`

### 수용 기준
- 쿠키 문구가 필수 쿠키 중심으로 바뀐다.
- 생년월일 입력 화면에서 입력 필드나 주요 버튼을 가리지 않는다.
- 모바일 375px 폭에서도 배너가 과하게 눈에 띄지 않는다.
- 개인정보처리방침 링크가 있다면 정상 동작한다.

### QA 체크리스트
- 로그인 전 랜딩
- 생년월일 입력 화면
- 모바일 하단 CTA 근처
- 쿠키 동의 전/후 상태
- 스크린리더에서 문구가 자연스럽게 읽히는지

## 티켓 2. 성별 입력에 사주 계산 기준 안내 추가

### 문제
- 성별 입력은 사주 계산상 필요할 수 있지만, 민감하게 느끼는 유저도 있다.
- “왜 성별을 묻는지”가 한 줄만 있어도 납득도가 높아진다.

### 요구사항
- 성별 선택 영역 가까이에 짧은 설명을 추가한다.
- 설명은 정당화하거나 장황하게 말하지 말고, 계산 기준상 필요한 정보라는 점만 부드럽게 안내한다.

### 권장 카피
- `성별은 사주 계산 기준에 필요해서만 사용해.`
- `사주 계산 방식에 따라 성별 기준이 달라져서 필요해.`
- `입력한 성별은 사주 분석 계산에만 사용돼.`

### 수용 기준
- 성별 입력 영역에서 설명 문구가 바로 보인다.
- 설명이 성별 선택을 강압적으로 느끼게 하지 않는다.
- 개인정보 안심 문구와 톤이 충돌하지 않는다.

### QA 체크리스트
- 성별 선택 전/후
- 오류 상태
- 모바일 화면 줄바꿈
- 스크린리더 라벨 연결

## 티켓 3. 현우 캐릭터 톤 완화

### 문제
- 현우 카드의 `위험 경고`, `미리 알아야 해`는 후킹은 되지만 일부 유저에게 겁주는 사주앱처럼 느껴질 수 있다.
- 전체 톤이 차분해진 만큼 현우도 조금 덜 위협적으로 맞추는 것이 좋다.

### 요구사항
- 현우 캐릭터의 카드 문구, 태그, quote, 프롬프트 원본에서 위협적인 단어를 완화한다.
- 직설 캐릭터 개성은 유지하되, 겁주기보다 “미리 정리해주는 분석가” 톤으로 바꾼다.

### 권장 방향
- `위험 경고` → `주의 포인트`
- `미리 알아야 해` → `미리 알면 덜 흔들려`
- `좀 위험한데` → `조심할 포인트가 있어`
- `경고` → `체크 포인트`

### 권장 카피 예시
- `네 흐름에서 조심할 포인트가 보여. 차분히 같이 볼게.`
- `미리 알면 덜 흔들리는 부분부터 짚어줄게.`
- `좋은 흐름과 주의할 흐름을 같이 정리해줄게.`

### 수용 기준
- 현우 카드가 겁주는 느낌보다 차분한 직설 상담 느낌으로 보인다.
- 원본 프롬프트에 위협적 표현이 남아 있지 않다.
- AI 답변 샘플에서도 겁주기/압박 표현이 재발하지 않는다.

### QA 체크리스트
- 랜딩 캐릭터 카드
- 채팅 첫 인사
- 무료 답변 샘플
- 유료 답변 샘플
- 금지 표현 검색

## 티켓 4. 가격표 배지 접근성 개선

### 문제
- 가격표가 화면상으로는 괜찮아 보여도 스크린리더나 텍스트 읽기 흐름에서는 `별 70개인기`, `별 250개최고 가성비`처럼 붙어서 읽힐 수 있다.
- 세심한 UX 관점에서는 별 개수, 배지, 가격 사이 분리감이 더 필요하다.

### 요구사항
- 가격표에서 별 개수, 추천 배지, 가격이 시각적/접근성상 분리되어 읽히게 한다.
- 배지는 스크린리더에서 자연스럽게 읽히거나, 장식이면 `aria-hidden` 처리한다.
- 텍스트가 붙어서 읽히지 않도록 숨김 텍스트 또는 구분 기호를 고려한다.

### 권장 구현 방향
- 시각 텍스트: `별 70개` + 별도 배지 `인기`
- 접근성 라벨: `별 70개, 인기 상품, 19,900원`
- 배지가 장식이면 실제 버튼 `aria-label`에 통합

### 수용 기준
- 스크린리더 기준으로 가격 상품이 자연스럽게 읽힌다.
- 모바일/데스크톱에서 배지와 별 개수가 붙어 보이지 않는다.
- 가격표의 기존 시각 디자인은 크게 흔들리지 않는다.

### QA 체크리스트
- VoiceOver 또는 브라우저 접근성 트리
- 모바일 375px
- 긴 배지 텍스트 줄바꿈
- 키보드 포커스 상태

## 티켓 5. 데스크톱 로그인 사이드바 우선순위 점검

### 문제
- 데스크톱 스냅샷에서 로그인 사이드바가 먼저 잡히면, 일부 화면에서는 서비스 가치보다 로그인 요구가 먼저 보일 수 있다.
- 첫인상에서 “왜 가입해야 하는지”보다 “로그인해라”가 먼저 보이면 전환이 떨어질 수 있다.

### 요구사항
- 데스크톱 첫 화면에서 서비스 가치, 무료 맛보기, 가격 기준이 로그인 요구보다 먼저 인지되는지 점검한다.
- 로그인 사이드바가 필요한 경우에도 시각적 강도를 낮추거나, 가치 설명 이후 자연스럽게 보이게 한다.

### 권장 방향
- 로그인 사이드바는 보조 행동으로 보이게 한다.
- 첫 viewport에서는 무료 맛보기/캐릭터/가격 기준을 우선 노출한다.
- 로그인 CTA 문구는 `가입하고 바로 시작`보다 `무료 3회로 먼저 확인하기`처럼 가치 중심으로 조정한다.

### 수용 기준
- 데스크톱 첫 화면 스냅샷에서 서비스 가치가 로그인 요구보다 먼저 보인다.
- 로그인 유도는 남아 있으나 강압적이지 않다.
- 모바일에서는 기존 주요 흐름을 방해하지 않는다.

### QA 체크리스트
- 1440px 데스크톱 첫 화면
- 1280px 노트북 화면
- 모바일 375px
- 로그인 전/후
- 사이드바 열림/닫힘 상태

## 최종 QA 시나리오
- 신규 유저가 랜딩에서 무료 맛보기와 가격을 본다.
- 쿠키 배너가 필수 쿠키 중심 문구로 조용히 보인다.
- 캐릭터 카드에서 현우가 위협적이지 않게 보인다.
- 생년월일 입력 전 개인정보 안심 문구를 본다.
- 성별 입력 옆에서 사주 계산 기준상 필요한 이유를 확인한다.
- 가격표가 시각적으로도, 접근성상으로도 자연스럽게 읽힌다.
- 데스크톱에서 로그인 요구보다 서비스 가치가 먼저 보인다.

## 개발 원칙
- Codex는 PM 역할만 수행한다.
- 소스코드 수정은 개발자가 진행한다.
- 개발자는 TDD 원칙에 따라 회귀 테스트 또는 검색 기반 검증을 먼저 추가한다.
- 특히 문구/프롬프트 변경은 금지 표현 검색 테스트를 둔다.
