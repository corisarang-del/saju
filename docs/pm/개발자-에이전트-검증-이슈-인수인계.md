# 개발자 에이전트 검증 이슈 인수인계

## 배경

- 2026-06-30 테스터 역할로 프로젝트 검증을 수행했다.
- `verification-before-completion` 스킬 기준으로 성공 주장 전 실제 명령을 실행했다.
- 현재 상태는 유닛 테스트와 타입체크는 통과하지만, 실행 환경, 린트, 빌드, 환경검증 스크립트에 차단점이 있다.

## 검증 증거

| 명령 | 결과 | 비고 |
|---|---:|---|
| `pnpm test` | 실패 | `pnpm` 런처가 `[ERROR] fetch failed`로 종료 |
| `pnpm lint` | 실패 | `pnpm` 런처가 `[ERROR] fetch failed`로 종료 |
| `pnpm --version` | 실패 | `fetch failed` 또는 장시간 응답 없음 |
| `./node_modules/.bin/vitest run` | 통과 | 12 files / 30 tests |
| `./node_modules/.bin/tsc --noEmit` | 통과 | 타입 에러 없음 |
| `./node_modules/.bin/eslint` | 실패 | 43 problems, 14 errors, 29 warnings |
| `./node_modules/.bin/next build` | 실패 | Google Fonts `Noto Sans KR` fetch 실패 |
| `node scripts/check-env.js` | 실패 | `scripts/check-env.js` 파일 없음 |

## 우선순위

1. `pnpm` 실행 환경 복구
2. 빌드가 네트워크 fetch에 의존하지 않도록 폰트 처리 수정
3. ESLint 14개 에러 수정
4. `test:env` 스크립트와 실제 파일 구조 정합성 복구
5. 모든 수정 후 `pnpm test`, `pnpm lint`, `pnpm build`, `pnpm test:env` 재검증

## 티켓 1: pnpm 실행 환경 복구

### 요구사항

- `pnpm test`, `pnpm lint`, `pnpm build`, `pnpm test:env`가 패키지 매니저 fetch 오류 없이 실행돼야 한다.
- 프로젝트 정책상 설치와 스크립트 실행은 pnpm 기준으로 유지한다.

### 수용 기준

- `pnpm --version`이 정상 출력된다.
- `pnpm test`가 Vitest를 실행한다.
- `pnpm lint`가 ESLint를 실행한다.
- 실패가 있더라도 `[ERROR] fetch failed`가 아니라 실제 프로젝트 오류가 출력된다.

### QA 체크리스트

- 네트워크 차단 환경에서도 로컬 pnpm 실행이 가능한지 확인한다.
- Corepack 또는 전역 pnpm 버전이 `packageManager`의 `pnpm@10.0.0`과 충돌하지 않는지 확인한다.
- 패키지 설치가 필요하면 `pnpm`만 사용한다.

### 리스크

- 전역 pnpm/Corepack 설정 수정은 다른 프로젝트에 영향을 줄 수 있다.

## 티켓 2: Google Fonts fetch 없는 빌드 복구

### 요구사항

- `next/font/google`이 빌드 중 외부 네트워크 fetch를 요구하지 않도록 수정한다.
- `Noto Sans KR` 사용 의도가 있다면 로컬 폰트 또는 fetch 없는 대체 방식으로 유지한다.

### 수용 기준

- 네트워크 제한 환경에서 `pnpm build`가 Google Fonts fetch 실패로 중단되지 않는다.
- 한글 폰트 렌더링 품질이 크게 후퇴하지 않는다.

### QA 체크리스트

- `pnpm build` 통과.
- 랜딩, 로그인, 채팅, 리포트 화면에서 한글 폰트가 정상 적용되는지 확인.
- 모바일/데스크톱에서 레이아웃 밀림이 없는지 확인.

### 리스크

- 폰트 교체 시 글자 폭 변화로 UI 줄바꿈과 버튼 높이가 달라질 수 있다.

## 티켓 3: ESLint 14개 에러 수정

### 요구사항

- 전체 `pnpm lint`가 에러 없이 통과해야 한다.
- 경고는 별도 판단 가능하지만, 새 경고를 추가하지 않는다.
- React 19 규칙 위반은 동작을 유지하면서 수정한다.

### 현재 확인된 대표 에러

- `src/app/[locale]/reading/page.tsx`: effect 안 동기 `setState`.
- `src/components/saju/input/BirthDateForm.tsx`: `useMemo` 안 `setState`.
- `src/components/shared/ThemeToggle.tsx`: effect 안 동기 `setState`.
- `src/components/ui/animated/ParallaxBackground.tsx`: effect 안 동기 `setState`.
- `src/components/ui/animated/ScrollAnimation.tsx`: `any` 타입.
- `src/components/ui/cookie-consent.tsx`: effect 안 동기 `setState`, 내부 페이지를 `<a>`로 이동.
- `src/hooks/use-mobile.ts`: effect 안 동기 `setState`.
- `src/lib/saju/natal.ts`: `prefer-const`.
- `src/lib/saju/saju-engine.ts`: `prefer-const`.

### 수용 기준

- `pnpm lint` exit code 0.
- 사용자 관점 동작이 유지된다.
- 상태 초기화 로직 변경 시 hydration 또는 깜빡임이 새로 생기지 않는다.

### QA 체크리스트

- 생년월일 입력 폼에서 월 변경 시 유효하지 않은 일이 안전하게 처리되는지 확인.
- 테마 토글이 SSR/hydration 경고 없이 동작하는지 확인.
- 쿠키 동의 배너가 첫 방문과 동의 후 재방문에서 맞게 동작하는지 확인.
- 모바일 감지 hook이 초기 렌더와 resize에서 맞게 동작하는지 확인.

### 리스크

- React hook 규칙 수정은 hydration, 초기 표시 상태, 사용자 입력 유지에 영향을 줄 수 있다.

## 티켓 4: `test:env` 스크립트 정합성 복구

### 요구사항

- `package.json`의 `test:env`가 실제 존재하는 스크립트를 가리키게 한다.
- 의도한 환경변수 검증 동작이 없다면 `scripts/check-env.js`를 추가한다.

### 수용 기준

- `pnpm test:env`가 실행된다.
- 필수 환경변수 누락 시 명확한 메시지와 non-zero exit code를 낸다.
- 개발용 placeholder 값은 실제 설정으로 오인하지 않는다.

### QA 체크리스트

- `.env.local`이 있는 경우와 없는 경우를 모두 확인한다.
- Supabase, Gemini, Paddle 등 필수 키의 누락 메시지가 구분되는지 확인한다.

### 리스크

- 환경변수 요구사항을 너무 강하게 잡으면 로컬 개발과 문서 작업까지 막을 수 있다.
