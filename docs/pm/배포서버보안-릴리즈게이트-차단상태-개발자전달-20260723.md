# 배포 서버보안 릴리즈게이트 차단상태 개발자전달 20260723

## 결론

현재 production 배포 승인은 보류다.

결제 제외 무료 베타로 가더라도 기본 `release:gate`가 아직 통과하지 못했다. 원인은 `qa:live-api:free` 단계의 Supabase DNS `ENOTFOUND`다. 결제 포함 production 배포라면 여기에 더해 `release:gate:payments`도 반드시 통과해야 하는데, 현재 Paddle 필수 env가 전부 비어 있어 실패한다.

중요한 점은 `release:gate:code`만 통과했다고 배포 가능으로 보면 안 된다는 것이다. `release:gate:code`는 코드/빌드 쪽 게이트이고, 실제 무료 상담 API QA와 결제 env QA를 대체하지 않는다.

## 현재 실패 상태

### P0. `pnpm run release:gate` 실패

상태:

- `release:gate:code`는 통과
- `qa:live-api:free`에서 실패

실패 원인:

- Supabase host: `sfpwgywcmhgilrqearsz.supabase.co`
- error: `ENOTFOUND`
- 결과: 실제 무료 상담 API QA가 임시 인증 사용자 생성 단계까지 가지 못함

아직 실제 DB 기준으로 확인되지 않은 항목:

- 신규 가입 후 3회 무료 지급
- 무료 첫 상담 생성
- user/assistant 메시지 저장
- 별 예약/차감/환불
- 거래 로그 기록
- `chat_used` 증가
- 저장된 assistant 품질

개발자 액션:

1. Supabase project ref `sfpwgywcmhgilrqearsz`가 실제 활성 project인지 확인한다.
2. `NEXT_PUBLIC_SUPABASE_URL`과 service role env가 production/staging/local에서 올바른 project를 가리키는지 확인한다.
3. 로컬, CI, Vercel production에서 Supabase DNS resolution이 되는지 확인한다.
4. DNS 정상 환경에서 `pnpm run release:gate` 또는 최소 `qa:live-api:free`를 재실행한다.

완료 기준:

- `qa:live-api:free` 통과
- 무료 3회, 첫 상담, 별 차감/환불, 거래 로그, 저장 품질이 실제 DB 기준으로 확인됨

### P0. Vercel production env 없음

확인 결과:

```bash
pnpm dlx vercel env ls
```

결과:

- `todocori/monthlysaju`에 `No Environment Variables found`

이 상태에서는 production 배포가 의미 있게 검증될 수 없다.

필수 확인 대상:

- Supabase URL
- Supabase anon key
- Supabase service role key
- AI provider env
- `APP_ORIGIN` 또는 `NEXT_PUBLIC_APP_URL`
- rate limit backend env
- admin allowlist env
- 결제 오픈 시 Paddle env

결제 제외 무료 베타에서도 Supabase, AI provider, canonical origin, rate limit backend는 반드시 production에 있어야 한다.

### P0. `pnpm run release:gate:payments` 실패

상태:

- Paddle 필수 env가 아직 비어 있어 실패

누락된 env:

- `PADDLE_API_KEY`
- `PADDLE_WEBHOOK_SECRET`
- `NEXT_PUBLIC_PADDLE_CLIENT_TOKEN`
- 별 상품/가격 ID
- 월간 멤버십 상품/가격 ID

판단:

- 결제 제외 무료 베타 배포에서는 이 실패를 즉시 blocker로 보지 않을 수 있다.
- 결제 포함 production 배포라면 이 실패는 무조건 P0 blocker다.

개발자 액션:

1. 결제 제외 배포에서는 `PAYMENTS_ENABLED=false`, `NEXT_PUBLIC_PAYMENTS_ENABLED=false`를 유지한다.
2. 결제 UI, checkout, 별 충전, 멤버십 구매, 유료 unlock CTA가 production에 노출되지 않게 확인한다.
3. 결제 포함 배포를 하려면 Paddle env를 전부 등록하고 `pnpm run release:gate:payments`를 통과시킨다.

완료 기준:

- 무료 베타: 결제 진입점이 모두 닫혀 있고 무료 상담 QA가 통과됨
- 결제 포함 배포: `pnpm run release:gate:payments`까지 통과됨

## P1. audit low 1건 추적 필요

확인 결과:

- `pnpm audit --prod` 전체 기준 low 1건 남음
- 패키지: `@ai-sdk/provider-utils`
- 유형: uncontrolled resource consumption

판단:

- high gate는 통과하므로 즉시 배포 차단급은 아니다.
- 하지만 AI 비용/리소스 관점에서는 추적해야 한다.

개발자 액션:

1. 패치 가능한 버전이 있는지 확인한다.
2. 당장 패치가 어렵다면 이슈로 추적한다.
3. AI 호출 경로에 rate limit, quota, timeout, response size guard가 실제로 적용되는지 같이 확인한다.

완료 기준:

- 패치 또는 명시적 추적 이슈 등록
- AI 비용 폭탄 방어가 release gate 문서에 남음

## 게이트 해석 주의

### `release:gate:code`

의미:

- 코드 테스트, 타입, lint, build, high audit 등 코드 품질 중심 게이트

주의:

- 실제 Supabase DB QA를 대체하지 않는다.
- Paddle env 검증을 대체하지 않는다.
- 이것만 보고 production 배포하면 안 된다.

### `release:gate`

의미:

- 결제 제외 무료 베타의 기본 production 승인 게이트
- `release:gate:code`에 더해 실제 무료 상담 API QA가 포함돼야 한다.

현재 상태:

- Supabase DNS `ENOTFOUND` 때문에 실패

### `release:gate:payments`

의미:

- 결제 포함 production 배포 또는 결제 재오픈 전용 게이트
- Paddle 필수 env와 결제 관련 검증을 포함한다.

현재 상태:

- Paddle 필수 env 누락으로 실패

주의:

- 결제 포함 배포라면 반드시 통과해야 한다.
- 결제 제외 무료 베타라면 실패 상태를 문서화하고 결제 진입점을 닫은 채 배포 판단을 분리할 수 있다.

## 개발 우선순위

1. Vercel production env 등록 상태를 먼저 정상화한다.
2. Supabase DNS `ENOTFOUND` 원인을 해결한다.
3. `pnpm run release:gate`를 다시 실행해 무료 상담 실제 API QA를 통과시킨다.
4. 무료 베타 상태에서는 결제 진입점과 결제 암시 문구가 모두 닫혀 있는지 확인한다.
5. 결제 오픈 시점에 Paddle env를 등록하고 `pnpm run release:gate:payments`를 별도로 통과시킨다.
6. `@ai-sdk/provider-utils` low advisory는 비용/리소스 리스크로 추적한다.

## PM 판단

지금 가장 위험한 오해는 `release:gate:code` 통과를 배포 가능으로 해석하는 것이다.

무료 베타 배포의 최소 승인 조건은 `release:gate` 통과다. 현재는 Supabase DNS 때문에 실패하고 있으므로 보류가 맞다. 결제까지 포함하려면 여기에 `release:gate:payments` 통과가 추가로 필요하다.
