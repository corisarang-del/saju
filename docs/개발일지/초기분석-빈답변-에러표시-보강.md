# 초기분석 빈답변 에러표시 보강

## 문제

초기 분석 프롬프트 자체는 `getInitialAnalysisPrompt`에서 정상 생성되고 있었다. 서버도 `toUIMessageStreamResponse`의 `onError`로 Gemini 쿼터/스트림 실패를 사용자용 문구로 변환하고 있었다.

하지만 클라이언트 말풍선은 `text` 파트만 합쳐서 렌더링하고 있었기 때문에, 스트림 실패나 빈 assistant 메시지가 발생하면 사용자는 답변이 멈춘 것처럼 볼 수 있었다.

## 수정

- `src/lib/ai/ui-message-display.ts` 추가
  - 일반 `text` 파트는 그대로 합친다.
  - `error`, `tool-input-error`, `tool-output-error`, `data-*` 형태의 에러 문구도 화면 텍스트로 변환한다.
  - assistant 메시지가 비어 있으면 빈 말풍선 대신 재시도 안내 문구를 반환한다.
- `ChatBubble`이 직접 `text` 파트만 읽지 않고 `getUiMessageDisplayText`를 사용하게 변경했다.
- `getUserFacingChatErrorMessage`가 서버에서 이미 안전하게 변환된 한국어 에러 문구를 보존하게 했다.
- `ChatRoom`의 `handleChatError`가 SDK 에러 메시지를 그대로 노출하지 않고 사용자용 문구 매핑을 사용하게 했다.

## 검증

- 실패 테스트 먼저 추가:
  - 에러 파트만 있는 assistant 메시지가 사용자 문구로 표시되는지
  - assistant 빈 메시지가 재시도 안내로 표시되는지
  - 서버에서 내려온 한국어 에러 문구를 클라이언트 매핑이 보존하는지
- 통과 확인:

  - `./node_modules/.bin/tsc --noEmit`
