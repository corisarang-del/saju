# 무료베타 실사용QA 잔여이슈 응답형식 문구 접근성 수정 20260723

## 배경

테스터 피드백 문서 기준으로 무료 베타 배포 승인은 아직 보류 상태다. 결제 기능 비활성화 자체는 허용 가능하지만, Supabase 실제 API QA가 DNS `ENOTFOUND`로 막혀 무료 상담 핵심 흐름이 실제 DB 기준으로 확인되지 않았다.

추가 잔여 이슈로 무인증 채팅 API 401 응답이 plain text였고, 무료 베타 상태와 충돌하는 결제 가능 암시 문구가 일부 남아 있었으며, 로그인 사이드바 문구가 스냅샷에서 붙어 읽힐 수 있었다.

## 수정 내용

- `src/lib/security/free-beta-qa-regression.test.ts`를 추가해 무료 베타 잔여 이슈를 TDD 회귀 테스트로 고정했다.
- `src/app/api/saju/chat/route.ts`의 무인증 401 응답을 `{ error: "unauthorized", message: "로그인이 필요해." }` JSON으로 변경했다.
- `src/app/[locale]/layout.tsx`의 JSON-LD FAQ를 결제 feature flag 기준으로 분기하고, 결제 off 상태에서는 유료 충전과 멤버십이 정식 결제 기능 안정화 후 열린다고 안내하게 바꿨다.
- `messages/ko.json`, `SajuFooter`, `ChatInput`, `ChatPaywall`, `PaddleCheckout`, 약관, 개인정보처리방침의 무료 베타/결제 문구를 정리했다.
- `LoginSidebar.tsx`의 로그인 안내 문장을 한 줄 텍스트로 유지해 `대화 기록을저장`처럼 붙어 보일 여지를 제거했다.

## 남은 상태

코드 게이트와 무료 베타 문구/응답형식/접근성 회귀는 닫혔다. 하지만 실제 Supabase 기반 무료 상담 API QA는 여전히 DNS 차단으로 미완료다. 무료 첫 상담, 3회 무료 별 지급, 별 차감/환불, 거래 로그, assistant 저장 품질은 DNS 정상 환경에서 다시 검증해야 한다.
