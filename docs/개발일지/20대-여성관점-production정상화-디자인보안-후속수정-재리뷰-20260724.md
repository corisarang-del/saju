# 20대 여성관점 production정상화 디자인보안 후속수정 재리뷰

## 요청
- 개발자가 다시 피드백을 받아 수정한 뒤, 실제 유저 관점에서 다시 리뷰해달라는 요청을 받았어.
- 소스코드는 절대 건드리지 말라는 조건이 있었어.
- 지금까지 나온 내용을 문서화하고 메모리에 저장해달라는 요청도 함께 받았어.

## 진행 범위
- 소스코드는 수정하지 않았어.
- 최근 2026-07-24 개발일지, PM 전달 문서, QA 결과, 프로젝트 메모리를 확인했어.
- `/ko`, `/ko/reading` 화면을 Playwright 스냅샷으로 확인했어.
- 랜딩, 무료 베타 문구, 캐릭터 카드, 입력폼, production 상담 정상화, 보안 하드닝, 결제 재오픈 잔여 리스크를 20대 여성 유저 관점에서 다시 판단했어.
- 문서와 메모리만 업데이트했어.

## 20대 여성 유저 관점 총평
- 무료 상담형 베타 제품 경험 기준으로는 9.7/10이야.
- production 운영 신뢰까지 포함하면 9.6/10이야.
- 결제 재오픈까지 포함한 완성도는 9.3/10이야.
- 종합 점수는 9.6/10으로 보는 게 맞아.
- 이전에 가장 컸던 불안인 Google 로그인, production 상담 실패, Supabase live API 미완료, 무료 베타 문구 충돌, PC/폰 디자인 깨짐은 대부분 해소됐어.
- 이제 남은 이슈는 결제 재오픈 전 검증과 캐릭터 브랜드/말투 마감이야.

## 좋아진 점
- production에서 Google 로그인과 사주 상담이 정상 확인됐어. 무료/전체 live API QA도 실제 user, reading, transaction 기준으로 통과했기 때문에 유저 입장에서는 "상담이 실제로 된다"는 가장 중요한 신뢰가 회복됐어.
- Vertex production 인증을 서비스 계정 키 없이 Vercel OIDC와 Google Workload Identity Federation으로 처리한 점은 운영 신뢰에 좋아. 비밀키를 Vercel에 직접 저장하지 않는 구조라 장기 운영에 안전해 보여.
- 무료 베타 문구 일관성이 많이 좋아졌어. 푸터가 "지금은 무료 상담 베타로 운영 중입니다."로 바뀌었고, JSON-LD도 결제 가능 암시를 줄였어.
- 채팅 disabled placeholder도 `별을 충전해주세요`에서 `무료 베타 상담 횟수를 모두 사용했어`로 바뀌어 결제 off 상태와 충돌하지 않아.
- 사이드바의 `대화 기록을저장하고` 붙어 읽힘은 화면 스냅샷에서 `대화 기록을 저장하고`로 해결됐어.
- 모바일 H1 단어 중간 줄바꿈, 첫 카드 CTA 가시성, `/ko/reading` 입력폼 대비, 하단 CTA 안정성은 문서와 회귀 테스트 기준으로 개선됐어.
- `/ko/reading`은 여전히 유저 진입 장벽이 낮아. 생년월일/성별 사용 이유를 먼저 설명하고, 태어난 시간을 몰라도 된다고 알려줘서 20대 여성 유저가 덜 경계하게 돼.
- 운영 보안 하드닝도 좋아졌어. OAuth origin, PKCE cookie, x-powered-by 제거, CSP `unsafe-eval` 제거, COOP/CORP, RLS/RPC, IDOR QA가 정리됐고, 무료 베타 공개 운영 기준 신뢰도가 높아졌어.

## 아직 아쉬운 점
- `하은`을 `인영`으로 바꾸는 PM 방향은 문서화됐지만 실제 화면과 코드에는 아직 `하은`이 남아 있어. 화면 스냅샷에서도 상담사 카드가 `하은`으로 보이고, `src/lib/saju/characters.ts`, `src/services/analytics/actions.ts`에도 `하은` 표기가 남아 있어. 20대 여성 유저 관점에서는 큰 기능 문제는 아니지만, 브랜드 마감 관점에서는 아직 미반영이야.
- 캐릭터별 말투 차별화도 아직 방향 문서 수준에 가까워 보여. 현재 유저가 보는 카드 문구는 차분하고 안전하지만, 상담사별로 "이 사람에게 물어보고 싶다"는 개성은 더 밀어낼 수 있어.
- 첫 상담 생성 지연은 계속 관찰해야 해. 문서상 무료 live API chat 약 55초, paid live API chat 약 110초까지 걸린 사례가 있어. 20대 유저는 답변이 1분 가까이 늦으면 앱이 멈춘 줄 알 수 있어.
- 결제 재오픈 기준으로는 Paddle signed webhook 운영 정합성, `release:gate:payments:live`, Paddle secret 일치 검증이 아직 남아 있어. 지금 무료 베타에는 큰 문제가 아니지만, 유료 전환 전에는 반드시 닫아야 해.
- CSP `unsafe-inline`은 남아 있어. 무료 베타 유저가 직접 느끼는 이슈는 아니지만, 보안 점수와 장기 운영 신뢰에는 남은 숙제야.
- 로컬 dev에서는 CSP가 React 개발용 eval을 막는 콘솔 에러가 보였어. production 콘솔 0건 기록과는 분리해서 봐야 하지만, 개발자가 로컬 확인할 때는 혼동할 수 있어.
- 랜딩은 여전히 가격표를 보여주면서 결제는 준비 중이라고 말해. 투명성은 좋지만, 일부 유저에게는 "살 수 없는 가격표"처럼 느껴질 수 있어. 무료 베타 기간에는 가격표보다 무료 상담/오늘피드 가치가 더 먼저 보여도 좋아.

## 우선순위
1. `하은` 표시명을 실제 유저 노출 기준으로 `인영`으로 바꾸고, 내부 id `haeun`은 유지하는 방향을 마무리해야 해.
2. 캐릭터별 답변 구조와 마지막 질문 방식을 실제 프롬프트/QA에 반영해 상담사 개성을 더 분리해야 해.
3. 첫 상담 응답 지연이 30초를 넘는 케이스에 대한 로딩 문구, 진행 상태, 재시도 안내를 보강하면 좋아.
4. 결제 재오픈 전 `QA_BASE_URL=https://monthlysaju.vercel.app pnpm qa:paddle-webhook:signed`와 `release:gate:payments:live`를 통과시켜야 해.
5. CSP `unsafe-inline` 제거는 report-only 관측 후 nonce/hash 전환으로 별도 진행하면 돼.

## 최종 판단
- 지금 상태는 무료 상담형 베타로는 유저에게 보여줘도 되는 수준까지 올라왔어.
- 특히 production 상담 정상화와 디자인 마감 개선이 커서, 20대 여성 유저 관점의 체감 신뢰가 많이 좋아졌어.
- 다만 "브랜드가 예쁘게 마감됐다"까지 가려면 `하은→인영` 실제 반영과 캐릭터 말투 차별화가 다음 핵심이야.
- 결제는 아직 열지 않는 판단이 맞고, 유료 전환 전에는 Paddle signed webhook과 결제 live gate를 반드시 통과해야 해.
