# 보안리뷰 P0 핫픽스 티켓화

- 소스코드는 수정하지 않고 PM 관점에서 보안 리뷰 결과를 개발자 전달 문서로 정리했다.
- `docs/pm/보안리뷰-p0-개인정보-결제-핫픽스-개발자-전달.md`에 P0/P1/P2 이슈, 위험도, 개발 요구사항, 수용 기준, 테스트 계획, 릴리즈 게이트를 작성했다.
- P0는 개인정보 IDOR, 별 무단 충전, Paddle 결제 검증 우회다.
- P1은 AI 비용 폭탄과 Server Action 권한 누락이다.
- P2는 관리자 하드닝 부족과 OAuth redirect hardening이다.
- 공개 출시 전 P0 3건은 반드시 막아야 하는 release blocker로 기록했다.
- 누적 보안 리스크와 결정 사항을 `.codex/memory/`에 업데이트했다.
