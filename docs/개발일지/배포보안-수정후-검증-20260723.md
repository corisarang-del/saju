# 배포보안 수정후 검증

## 요청
- 개발자가 보안사항을 수정했으니 제대로 수정됐는지 테스트해달라는 요청을 받았어.
- 이후 개발자에게 전달할 수 있게 문서화하고, 지금까지 나온 내용을 메모리에 저장해달라는 요청을 받았어.
- 소스코드는 수정하지 않고 검증과 문서화만 진행했어.

## 사용한 관점
- 서버 배포 담당자 관점
- 보안 담당자 관점
- Vercel production 배포 관점
- Supabase RLS/RPC 운영 적용 관점
- Paddle 결제/webhook 검증 관점
- AI 비용/rate limit 방어 관점

## 실행한 검증
- 타깃 보안 회귀 테스트
- 전체 `pnpm test`
- `pnpm build`
- `env REQUIRE_PADDLE_ENV=true pnpm test:env`
- `pnpm audit --prod --audit-level high`
- `pnpm dlx vercel env ls`

## 검증 결과
- 타깃 보안 회귀 테스트: 9개 파일 / 45개 테스트 통과
- 전체 테스트: 50개 파일 / 197개 테스트 통과
- production build: 통과
- Paddle env gate: 실패
- production audit high gate: 실패
- Vercel env 조회: `todocori/monthlysaju`에 환경변수 없음

## 확인된 수정 완료 항목
- `user_stars`, `star_transactions`의 authenticated 직접 조작 정책을 제거하는 migration이 추가돼 있어.
- 채팅 별 차감은 AI 호출 전 `reserve_chat_star`로 예약하고 실패 시 `refund_chat_star`로 환불하는 흐름으로 바뀌었어.
- Paddle 멤버십 webhook은 price id, product id, 월간멤버십 상품 타입을 검증하게 바뀌었어.
- rate limit은 production에서 Supabase shared backend가 아니면 fail-closed 하도록 바뀌었어.
- release gate에 Paddle env, test, typecheck, lint, build, prod audit high가 포함돼 있어.

## 남은 배포 차단 이슈
- Vercel `todocori/monthlysaju` production 프로젝트에 환경변수가 하나도 없어.
- `REQUIRE_PADDLE_ENV=true pnpm test:env`가 Paddle 필수값 누락으로 실패해.
- `pnpm audit --prod --audit-level high`가 `next@16.2.9` high 취약점 4개로 실패해. 패치 버전은 `next >=16.2.11`이야.
- Supabase 새 migration은 저장소에는 있지만 production DB 적용 여부를 별도로 확인해야 해.
- OAuth origin은 production env가 비어 있으면 header fallback 위험이 남으므로 `APP_ORIGIN` 또는 `NEXT_PUBLIC_APP_URL`을 운영 도메인으로 고정해야 해.

## 결론
- 핵심 보안 로직 수정은 테스트상 확인됐어.
- 하지만 배포 게이트는 아직 실패 상태야.
- Next.js 패치, Vercel env 등록, Paddle env gate 통과, Supabase production migration 적용 확인 전에는 production 배포를 보류해야 해.

## 후속 개발 수정

## 요청
- `docs/pm/배포보안-수정후-검증-개발자-전달-20260723.md`를 보고 수정할 수 있는 항목을 반영했어.

## 수정 내용
- `next`를 `^16.2.11`로 올려 문서에 적힌 high 취약점 패치 기준을 충족시켰어.
- `eslint-config-next`도 `16.2.11`로 맞췄고, `sharp`는 `^0.35.0`으로 올렸어.
- `pnpm-lock.yaml`을 갱신하고 `pnpm install --force`로 로컬 의존성도 lockfile 기준으로 다시 맞췄어.
- production에서 `APP_ORIGIN` 또는 `NEXT_PUBLIC_APP_URL`이 없으면 OAuth origin 계산이 `PRODUCTION_APP_ORIGIN_REQUIRED`로 fail-closed 하도록 바꿨어.
- OAuth 회귀 테스트에 production canonical origin 누락 시 forwarded header fallback을 쓰지 않는 케이스를 추가했어.
- release gate 회귀 테스트에 `next ^16.2.11` 기준을 추가했어.

- `pnpm test`: 50개 파일 / 199개 테스트 통과.
- `pnpm lint`: 통과.

- `pnpm audit --prod --audit-level high`: 통과. 남은 항목은 `@ai-sdk/provider-utils` low 1개뿐이고 현재 patched version이 없어.

## 남은 배포 차단
- `REQUIRE_PADDLE_ENV=true pnpm test:env`는 Paddle 필수 env 누락으로 실패해.
- `pnpm dlx vercel env ls` 결과 Vercel `todocori/monthlysaju`에는 환경변수가 하나도 없어.
- Supabase production DB에 `202607210010_release_gate_star_reservation_rate_limit.sql`이 실제 적용됐는지 아직 확인해야 해.
