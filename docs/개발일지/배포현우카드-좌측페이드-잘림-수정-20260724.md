# 배포 현우카드 좌측 페이드 잘림 수정

## 문제
- 배포 화면 스크린샷에서 현우 카드의 좌측 일부가 덜 보였다.
- 원인은 데스크톱 캐러셀 edge fade mask가 첫 번째 카드 상태에도 그대로 적용되어, 현우 카드 왼쪽 64px 구간이 투명 처리되는 구조였다.
- 사이드바와의 겹침 문제처럼 보일 수 있지만 실제 핵심은 첫 카드가 active인 상태에서도 `transparent_64px -> black_116px` 왼쪽 mask가 먹는 점이었다.

## 변경 내용
- `src/components/saju/landing/CharacterCards.tsx`
  - `carouselMaskClass`를 추가했다.
  - `activeIndex === 0`일 때는 왼쪽을 `black_0`부터 시작하는 mask로 두어 첫 카드가 온전히 보이게 했다.
  - `activeIndex > 0`일 때만 기존 왼쪽 edge fade를 적용해 두 번째 이후 카드 이동 시 의도된 depth는 유지했다.
- `src/lib/design/design-audit-regression.test.ts`
  - 첫 데스크톱 캐릭터 카드에서는 왼쪽 edge fade가 시작되지 않아야 한다는 회귀 테스트를 추가했다.

## TDD 기록
- 새 테스트 `keeps_first_desktop_character_card_fully_visible_before_left_edge_fade_starts`를 먼저 추가했다.
- 구현 전 해당 테스트가 `activeIndex > 0` 분기 부재로 실패하는 것을 확인했다.
- mask 분기 구현 후 디자인 회귀 테스트가 통과했다.

## Playwright 확인
- 로컬 dev `http://localhost:3000/ko`, 1280x720 기준으로 측정했다.
- 자동 캐러셀을 pointer 이벤트로 멈춘 뒤 첫 위치로 되돌려 측정:
  - `scrollLeft`: 0
  - `sidebarRight`: 240
  - `imageLeft`: 249
  - `hostLeft`: 248
  - mask: `linear-gradient(to right, rgb(0, 0, 0) 0px, rgb(0, 0, 0) calc(100% - 28px), rgba(0, 0, 0, 0) 100%)`
- 현우 이미지가 사이드바 밖에서 시작하고, 첫 카드 왼쪽 mask가 투명으로 시작하지 않는 것을 확인했다.
