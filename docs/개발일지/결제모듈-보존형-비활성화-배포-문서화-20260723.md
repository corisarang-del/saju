# 결제모듈 보존형 비활성화 배포 문서화 20260723

## 작업 내용

- 사용자가 “Paddle을 빼고 배포”라는 의미가 결제 코드를 삭제하는 것이 아니라, 나중에 결제 기능을 추가할 것을 대비해 결제 모듈은 보존하고 production에서는 비활성화한다는 뜻이라고 정정했다.
- 이에 맞춰 개발자 전달 문서를 새로 작성했다.
- 소스코드는 수정하지 않았다.

## 핵심 정리

- 이번 배포는 무료 상담형 베타다.
- Paddle/결제 관련 코드, 가격 정책, 상품 타입, 웹훅 구조, 테스트는 삭제하지 않는다.
- production에서는 `PAYMENTS_ENABLED=false` 또는 동등한 feature flag로 결제 진입점만 막는다.
- Paddle env gate는 이번 배포 blocker에서 제외한다.
- Supabase RLS/RPC, 별 예약/환불, rate limit, production origin, 실제 API QA는 여전히 P0다.
