# 배포웹사이트 프리미엄마감 수정후재리뷰 문서화 메모리저장

## 작업 내용

- 배포된 production 웹사이트를 `design-taste-frontend` 없이 다시 디자인 리뷰했다.
- 사용 기준은 `high-end-visual-design`, `minimalist-ui`, `redesign-existing-projects`, `playwright`, `memory`다.
- `/ko`를 PC 1280x720, PC 1366x768, 모바일 390x844로 확인했다.
- `/ko/reading`을 모바일 390x844 상단/하단으로 확인했다.
- 콘솔 error/warning과 가로 오버플로우를 확인했다.
- 개발자 전달 문서와 메모리를 업데이트했다.

## 확인 결과

- 릴리즈 차단급 디자인 이슈는 없다.
- PC 1280x720 카드 CTA 발견성과 캐러셀 첫 진입 예측성은 해결됐다.
- 모바일 홈과 `/ko/reading`은 회귀 없이 안정적이다.
- PC 1366x768에서는 카드 이름은 보이지만 `대화하기` CTA가 첫 viewport 밖에 있어 P2로 남겼다.
- 보라색 accent hierarchy는 개선됐지만 CTA와 보조 상태의 구분을 더 다듬을 여지가 있어 P3로 남겼다.

## 개발 반영

- PC 1366x768 첫 화면에서 현우 카드의 `대화하기` CTA가 viewport 밖으로 밀리던 문제를 수정했다.
- 원인은 low-height PC 보정 기준이 `max-height:760px`라서 1366x768 같은 일반 노트북 높이에 적용되지 않는 점이었다.
- 홈 hero와 캐릭터 카드의 low-height media query를 `max-height:800px`까지 확장했다.
- low-height 카드 폭은 `236px`로 더 줄이고, hero 상단 여백과 H1 크기를 한 단계 압축했다.
- 서비스 배지와 캐러셀 dot, `/ko/reading` progress fill은 더 muted한 톤으로 낮춰 CTA 중심의 accent hierarchy를 유지했다.
- `src/lib/design/design-audit-regression.test.ts`에 1366x768 CTA 발견성 회귀 테스트를 먼저 추가하고 실패를 확인한 뒤 구현했다.

## 로컬 화면 검증

- 1366x768 `/ko`: 첫 카드 CTA bottom `512.38px`, 하단 여유 `255.63px`, `scrollLeft=0`, active dot `0`, 가로 overflow `0`.
- 1280x720 `/ko`: 첫 카드 CTA bottom `512.38px`, 하단 여유 `207.63px`, `scrollLeft=0`, active dot `0`, 가로 overflow `0`.
- 390x844 `/ko`: 첫 카드 CTA bottom `615.94px`, 하단 여유 `228.06px`, `scrollLeft=0`, active dot `0`, 가로 overflow `0`.
- 7초 후에도 세 viewport 모두 active dot `0`, `scrollLeft=0`으로 자동 캐러셀 이동 회귀가 없었다.
