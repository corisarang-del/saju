## 실패한 항목
### P1: 무료 첫 상담 실제 API가 아직 안정적으로 응답하지 못함
- 전체 실사용 API QA에서 무료 첫 상담 요청이 503으로 실패.
- 응답:
  - status: `503`
  - body: `{"error":"chat_generation_failed","message":"분석 응답을 만들지 못했어. 잠시 후 다시 시도해줘.","requestId":"baf8fbbb-0894-44f7-bb48-5f8158794d5e"}`
- 서버 로그 원인:
  - `AI_RetryError`
  - `Resource exhausted. Please try again later.`
  - Vertex/Gemini 429 자원 소진.

### P1: 무료 첫 상담 단독 재시도도 503
- 무료 첫 상담만 `QA_SCENARIO=free-only`로 재실행했지만 503 실패.
- 응답:
  - status: `503`
  - body: `{"error":"chat_generation_failed","message":"분석 응답을 만들지 못했어. 잠시 후 다시 시도해줘.","requestId":"bd3103c8-ed4f-49c3-b0a7-d9d5a05a9fe8"}`
- 서버 로그 원인:
  - `[saju/chat] Initial analysis failed quality gate { attempts: 3, textLength: 626 }`
  - `Initial analysis failed quality gate`
- 즉, 무료 첫 상담은 모델 quota 문제와 3회 품질 게이트 미통과 문제가 둘 다 실제 경로에서 재현됐다.

## 개발자에게 전달할 내용
- 현재 수정은 유료 첫 상담과 Gemini 단독 QA 기준으로는 좋아졌지만, 무료 첫 상담 실제 API는 아직 릴리즈 통과라고 보기 어렵다.
- 무료 첫 상담은 `isFree: true`일 때 토큰 제한과 품질 기준이 충돌하는지 확인해야 한다.
- 3회 재생성 후 실패하면 사용자에게 503만 보여주므로, 운영 UX에서는 실패 안내는 가능해도 “제대로 응답”은 아니다.
- 해결 방향 후보:
  - 무료 첫 상담 전용 `maxOutputTokens`를 품질 게이트를 통과할 수 있을 만큼 조정.
  - 3회 실패 시 저장/차감 없이 안전한 고정 fallback 첫 상담을 반환할지 정책 결정.
  - 품질 게이트 실패 사유를 로그에 구조화해서 어떤 조건 때문에 탈락했는지 남김.
  - 무료/유료 첫 상담을 같은 실사용 API QA 스크립트에서 계속 회귀 검증.

## 결론
- “전체 정상”은 아님.
- 개발자 수정 후 많은 게이트는 통과했지만, 실제 사용자 무료 첫 상담 경로가 503으로 실패하므로 QA 미통과다.
