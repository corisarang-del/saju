# 프로젝트 디자인 문제점 감사

- 일시: 2026-06-30 23:36 KST
- 범위: 랜딩, 모바일 랜딩, 사주 입력 화면, 채팅/결제/리포트 관련 컴포넌트 구조 확인
- 작업: 소스코드 수정 없이 디자인 시스템, 전환 흐름, 접근성, 모바일 가림 문제를 점검함
- 검증: 기존 Next dev 서버(`http://localhost:3000`)에서 Playwright로 데스크톱/모바일 화면을 확인함

## 확인한 핵심 문제

- 쿠키 배너가 모바일 입력 화면의 다음 버튼과 랜딩 핵심 가격 카드/캐릭터 CTA를 가림.
- 전역 디자인 토큰은 다크 퍼플인데 랜딩은 베이지/티얼, 입력 플로우는 토스 블루로 갈라져 브랜드 일관성이 약함.
- 같은 생년월일 입력이 화면마다 다른 UI와 이름 입력 방식을 사용해 신뢰도가 떨어질 수 있음.
- 생년월일 숫자 입력 칸에 개별 접근성 이름이 없어 보조기술에서 년/월/일 구분이 약함.
- 캐릭터 캐러셀 페이지 점 버튼에 접근성 이름이 없음.
- 후기 섹션의 카카오톡 재현과 과한 구어체가 신뢰 근거보다 조작된 느낌을 줄 수 있음.

## 코드 변경

- 소스코드는 수정하지 않음.
