# 연속 빈응답 완료메시지 오판 수정

## 문제

사용자가 후속 질문을 4번 재시도했지만 `응답을 받지 못했어. 별은 차감하지 않았으니 다시 시도해줘.` 안내가 계속 표시됐다.

이 문구는 서버가 직접 내려주는 Gemini 오류 문구가 아니라, 클라이언트 완료 콜백에서 assistant 텍스트가 비었다고 판단했을 때 띄우는 방어 문구다.

## 원인

`ChatRoom`의 `onFinish`에서 AI SDK가 넘겨주는 `message` 하나만 읽어 assistant 텍스트를 판단하고 있었다.

AI SDK 구현을 확인하면 `onFinish`는 다음 값을 함께 넘긴다.

- `message: this.activeResponse.state.message`
- `messages: this.state.messages`

스트림 처리 타이밍이나 내부 상태 반영 방식에 따라 `message.parts`가 비어 있어도 최종 `messages` 배열의 마지막 assistant에는 텍스트가 들어 있을 수 있다. 기존 코드는 이 fallback을 보지 않아 정상 응답도 빈 응답으로 오판할 수 있었다.

## 수정

- `src/lib/ai/chat-finished-message.ts` 추가
  - 완료 콜백의 `message`에서 텍스트를 먼저 읽는다.
  - 비어 있으면 `messages` 배열의 마지막 assistant 메시지를 fallback으로 읽는다.
  - legacy `content` 문자열도 지원한다.
- `ChatRoom`의 `onFinish`에서 `getFinishedAssistantText`를 사용하게 변경했다.
- 회귀 테스트에 완료 메시지 fallback 요구사항을 추가했다.

## 검증

- `chat-finished-message` 테스트 추가:
  - 기본 완료 메시지 텍스트 읽기
  - 빈 완료 메시지일 때 마지막 assistant fallback 읽기
  - legacy content 지원
- 관련 채팅 회귀 테스트 통과 확인.
