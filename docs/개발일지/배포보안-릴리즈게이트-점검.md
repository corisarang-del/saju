# 배포보안 릴리즈게이트 점검

## 요청
- 배포 전에 보안이나 배포 시 문제가 있는지 파악해달라는 요청을 받았어.
- 이후 지금까지 나온 내용을 문서화하고 메모리에 저장해달라는 요청을 받았어.
- 소스코드는 수정하지 않았고, 문서와 메모리만 갱신했어.

## 사용한 관점
- 서버 배포 담당자 관점
- 보안 담당자 관점
- 결제/별 재화 정합성 관점
- Vercel/Next.js 운영 환경변수 관점
- Supabase RLS/RPC 권한 관점

## 확인한 현재 상태
- `pnpm-workspace.yaml`의 `packages` 문제는 해결돼 있어.
- `pnpm test`는 37개 파일 / 122개 테스트 통과 상태야.
- `pnpm lint`는 통과 상태야.
- `pnpm build`는 통과 상태야.
- 기본 `pnpm test:env`는 통과 상태야.
- `REQUIRE_PADDLE_ENV=true pnpm test:env`는 현재 로컬 환경 기준 Paddle 필수값 누락으로 실패했어.
- `pnpm audit --prod`는 `postcss` moderate, `@ai-sdk/provider-utils` low 취약점으로 실패했어.

## 확인한 개선
- 이전 P0였던 RLS/IDOR, 클라이언트 amount 신뢰, Paddle price 검증 문제는 상당 부분 개선된 것으로 보여.
- 민감 RPC 권한 회수 마이그레이션이 추가돼 있어.
- `/api/saju/suggestions`는 인증, rate limit, daily quota, 입력 길이 제한, provider 실패 처리가 들어갔어.
- 소스에서 명백한 실제 API 키/시크릿 노출은 찾지 못했어.

## 남은 배포 차단 이슈

### P0. 결제 상태 우회
- `src/app/api/saju/update-status/route.ts`가 클라이언트 `status`를 그대로 저장할 수 있어.
- 사용자가 자기 reading을 `paid`로 바꾸면 `src/app/api/saju/analyze/route.ts`의 유료 분석 조건을 통과할 수 있어.
- 결제/별 차감 성공 없이 유료 분석이 열릴 수 있으므로 배포 차단급이야.

### P1. 운영 Supabase 마이그레이션 적용 확인
- `supabase/migrations/202607010020_rpc_suggestions_security.sql`은 민감 RPC를 `service_role` 중심으로 제한해.
- 하지만 운영 DB에 실제 적용됐는지는 코드만으로 보장할 수 없어.
- 앱 배포 전 운영 Supabase에 먼저 적용해야 해.

### P1. 채팅 별 차감 동시성
- `src/app/api/saju/chat/route.ts`는 별 잔액 확인 후 AI를 생성하고, 완료 후 별을 차감해.
- 동시 요청이면 별 1개로 여러 AI 생성이 통과할 수 있어.
- 비용 발생과 별 차감 정합성이 어긋날 수 있어.

### P1. 운영 Paddle 환경변수
- 결제 포함 배포라면 Paddle production 환경변수가 반드시 필요해.
- 현재 로컬 `REQUIRE_PADDLE_ENV=true pnpm test:env`는 Paddle 값 누락으로 실패해.
- 운영 배포에서는 이 검증을 게이트로 올려야 해.

## 남은 운영/보안 경고
- `pnpm audit --prod`에서 production 취약점이 남아 있어.
- Vertex/Gemini 운영 인증은 빌드만으로 검증되지 않아 실제 운영 런타임 호출 확인이 필요해.
- CSP는 설정돼 있지만 `unsafe-inline`, `unsafe-eval`이 남아 있어 XSS 방어력은 제한돼.
- OAuth redirect 생성에서 origin 신뢰 경로는 운영 allowlist와 함께 다시 확인하는 편이 안전해.

## 개발자 전달 문서
- 상세 수정 가이드는 `docs/pm/배포보안-릴리즈게이트-개발자-수정가이드.md`에 정리했어.

## 결론
- 기능 빌드 관점에서는 배포 가능한 상태에 가까워졌어.
- 하지만 결제 포함 공개 배포 관점에서는 `update-status` 결제 우회와 채팅 별 차감 레이스가 남아 있어.
- 최소 P0 해결, 운영 DB 마이그레이션 적용 확인, Paddle/AI 운영 환경변수 검증 후 배포해야 해.
