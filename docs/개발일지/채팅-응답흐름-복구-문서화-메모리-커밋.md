# 채팅 응답흐름 복구 문서화 메모리 커밋

## 요약

사주 채팅에서 이어진 응답 안정성 문제를 문서화하고 메모리에 저장했다.

다룬 문제:
- 초기 분석이 아예 보이지 않음
- `현철 씨, 사`처럼 부분 답변만 남음
- 첫 답변 이후 `응 알려줘` 후속 질문이 무응답처럼 보임
- `응답을 받지 못했어` 안내가 4번 연속 발생

## 정리한 원인

- AI SDK UI stream error가 일반 text part가 아닐 수 있는데, 말풍선이 text part만 렌더링했다.
- `finishReason: error` 또는 너무 짧은 초기 분석 partial 응답이 성공처럼 저장/차감될 수 있었다.
- 후속 빈 assistant 응답을 클라이언트가 실패로 보지 않아 별 UI를 차감할 수 있었다.
- 성공한 첫 답변 뒤 `saju_readings.chat_used`가 증가하지 않아 후속 질문도 계속 첫 상담 규칙으로 취급됐다.
- AI SDK `onFinish.message`만 읽어 완료 텍스트를 판단해, 최종 `messages` 배열에 들어온 assistant 텍스트를 놓칠 수 있었다.
