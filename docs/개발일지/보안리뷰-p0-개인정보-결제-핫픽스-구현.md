# 보안리뷰 P0 개인정보 결제 핫픽스 구현

- 일시: 2026-07-01 00:42:40 KST

## 작업 내용

- `security-review` 스킬과 Supabase/API/결제 하드닝 레퍼런스를 읽고 P0 범위부터 처리했다.
- 보안 회귀 테스트를 먼저 추가해 RLS 복구, owner 필터, 별 차감 정책, Paddle price id 지급, idempotency 기준을 고정했다.
- 핵심 사주 테이블 RLS를 다시 enable하는 `202607010010_p0_security_hotfix.sql` migration을 추가했다.
- 기존 core schema와 RLS 복구 스크립트에서 `disable row level security`를 제거하고 enable 기준으로 뒤집었다.
- `saju_readings`, `saju_chat_messages`, `saju_compatibilities`에 사용자 owner 기반 RLS policy를 추가했다.
- `/api/saju/preview`, `/api/saju/analyze`, `/api/saju/chat`, `/api/saju/pdf/[id]`, `/api/saju/compatibility`, `/reading/[id]`에 로그인 사용자 owner 필터를 추가했다.
- `getReading`, `updateReadingStatus`, `linkReadingToUser`, `createCompatibility`, `getCompatibility`, `getUserReadings`에 세션 사용자 검증과 owner 필터를 추가했다.
- `chat-actions`의 메타 업데이트, 메시지 조회/저장, 크레딧 조회/추가, 삭제에 owner 검증을 추가했다.
- `/api/saju/deduct-stars`는 클라이언트 `amount`를 신뢰하지 않고 서버 고정 `REPORT_STAR_COST`와 원자적 RPC `deduct_stars_for_report`만 사용하게 바꿨다.
- Paddle 웹훅은 `customData.productType` 대신 실제 결제 item의 `price.id`로 지급량을 결정하게 바꿨고, `paddle_transaction_id` 중복 처리와 unique partial index를 추가했다.
- Paddle 구매 적립도 `credit_stars_for_paddle_purchase` RPC로 묶어 잔액 증가와 거래 기록, 중복 거래 처리를 한 트랜잭션 안에서 처리하게 보강했다.
