# 운영사이트 공격형 보안점검 문서화와 메모리 저장

## 요청
- 개발자가 수정 후 `https://monthlysaju.vercel.app/`에 배포했다고 알려줬다.
- 보안상 문제가 있는지 직접 사이트를 공격해서 찾아달라는 요청을 받았다.
- 이어서 개발자에게 전달할 수 있게 문서화하고, 지금까지 나온 내용을 문서화하고 메모리에 저장해달라는 요청을 받았다.

## 수행 범위
- 소스코드는 수정하지 않았다.
- 운영 사이트에 비파괴 공격성 payload를 보냈다.
- 대량 요청, 실제 사용자 데이터 접근, 서비스 장애 유발 테스트는 하지 않았다.
- 임시 QA 계정/테스트 리딩/payload를 사용하는 authenticated IDOR 및 webhook QA를 수행했다.

## 수행한 공격형 점검
- 비인증 `deduct-stars` 음수 amount payload.
- 비인증 `update-status paid` 권한 상승 payload.
- 비인증 `chat` prompt-injection payload.
- 비인증 `suggestions` XSS/prompt payload.
- 악성 origin CORS preflight.
- OAuth forwarded-host spoofing.
- OAuth callback open redirect payload.
- 공개 파일 `.env`, `.git/config`, `package.json`.
- 공개 HTML secret 패턴.
- Next.js JS sourcemap 공개 여부.
- query reflected XSS payload.
- invalid JSON/body payload.
- Paddle 무서명/오서명/정상서명 webhook payload.
- 실제 두 계정 authenticated IDOR QA.
- 운영 live API free/full QA.
- 운영 Supabase RLS/RPC 메타데이터 확인.
- MDN Observatory 최신 스캔.

## 통과한 항목
- 비인증 민감 API 악성 payload는 대부분 401로 차단됐다.
- CORS는 악성 origin에 열려 있지 않았다.
- OAuth forwarded-host spoofing과 open redirect는 재현되지 않았다.
- OAuth PKCE cookie는 `Secure`, `HttpOnly`, `Max-Age=600`, `SameSite=Lax`를 유지했다.
- 공개 파일과 명백한 secret 패턴은 발견하지 못했다.
- JS sourcemap은 403으로 막혔다.
- 실행형 reflected XSS는 재현하지 못했다.
- authenticated IDOR는 `preview`, `analyze`, `deduct-stars`, `update-status`, `chat`, `pdf` 모두 404로 막혔고 owner row는 변하지 않았다.
- 운영 `pnpm qa:live-api:free`, `pnpm qa:live-api`는 통과했다.
- 운영 RLS 대상 10개 테이블은 모두 RLS enabled였다.
- 민감 RPC는 `anon`/`authenticated`에 열려 있지 않았다.

## 발견한 이슈
- P1: Paddle signed webhook 정상 서명 payload가 운영에서 계속 401로 실패한다.
- P1: `REQUIRE_PRODUCTION_ENV=true pnpm test:env`가 `GOOGLE_VERTEX_RUNTIME_AUTH` 누락/placeholder로 실패한다.
- P2: `/api/saju/preview`, `/api/saju/analyze`, `/api/saju/compatibility`, `/api/saju/chat`이 invalid JSON/body에 500을 반환한다.
- P2: CSP `unsafe-inline`이 남아 MDN Observatory가 B+ 80점, 실패 1개다.

## 결론
- 무료 상담형 운영은 직접 공격 payload 기준으로 큰 구멍은 발견하지 못했다.
- 결제 포함 운영은 Paddle signed webhook 정상 경로와 production env gate가 닫히기 전까지 보류한다.
- 개발자 우선순위는 Paddle secret 정합성, `GOOGLE_VERTEX_RUNTIME_AUTH`, invalid JSON/body 500 처리, CSP 강화다.
