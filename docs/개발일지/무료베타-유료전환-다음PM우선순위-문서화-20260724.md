# 무료베타 유료전환 다음 PM 우선순위 반영

## 요청
- `docs/pm/무료베타-유료전환-다음PM우선순위-개발자전달-20260724.md`를 기준으로 무료 베타에서 유료 전환 전 필요한 작업을 반영해달라는 요청을 받았다.
- 작업 종료 후 커밋/푸시, 문서화, 메모리 저장까지 요청받았다.

## 반영한 핵심 결론
- 무료 베타 사이트는 제대로 설계되고 구현된 상태로 본다.
- 유료 코칭형 사주앱 방향은 맞지만, 결제 운영 안정성, 상담 품질 반복성, 구독 가치 증명이 아직 남아 있다.
- Paddle 제외 무료 베타 운영은 유지한다.
- 결제 버튼은 `release:gate:payments:live`와 Paddle signed webhook QA가 통과하기 전까지 열지 않는다.

## 구현 내용
- 무료 베타 퍼널 이벤트 타입을 `src/types/analytics.ts`에 추가하고 `/api/analytics/track` 허용 목록에 연결했다.
- 클라이언트 계측 helper `src/lib/analytics/client.ts`와 화면 진입 이벤트 컴포넌트 `src/components/analytics/ClientAnalyticsEvent.tsx`를 추가했다.
- 랜딩, 입력 페이지, 캐릭터 선택, Google 로그인 시작, 코인샵 노출, 결제 비활성 안내, 첫 상담 시작/성공/실패, 후속 질문, 무료 할당량 소진 이벤트를 기록하도록 붙였다.
- 첫 상담 대기 UX는 0초, 10초, 30초 기준으로 `사주 흐름을 정리하고 있어`, `고민에 맞는 상담 방향을 잡고 있어`, `답변이 조금 길어지고 있어. 완료되면 바로 보여줄게` 문구가 바뀌도록 했다.
- 월구독 50별 사용 예시는 랜딩 가격 패널, FAQ, 코인샵에 `매일 1회 상담 30별 + 월간 리포트 3별 + 후속 질문 17회`로 노출했다.
- 결제 기능은 계속 `paymentsEnabled` 기준으로 닫아두고, 준비 중 문구를 유지했다.
