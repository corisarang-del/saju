# 월간사주 원본 계승 업그레이드

## 작업 내용
- 원본 `imgompanda/monthlysaju-opensource` 소스를 현재 워크스페이스로 가져왔다.
- npm lock 파일을 제거하고 pnpm 기반 프로젝트로 정리하기 시작했다.
- PRD/TRD/화면 명세/도메인 테스트를 먼저 추가하는 TDD 흐름으로 진행한다.
- 오늘피드, 기억 요약, 결제 유도, AI provider 선택 도메인 테스트를 먼저 작성했다.
- 테스트 실패를 확인한 뒤 `src/lib/monthly-saju` 도메인 모듈을 구현했다.
- `소연` 캐릭터를 `하나`로 리브랜딩하고, 생성 이미지 후보를 `public/characters/hana-concept.png`에 저장했다.
- 모바일 5탭 구조와 `/today`, `/reports` 화면을 추가했다.
- 홈과 상단 네비게이션을 밝고 세련된 월간사주 톤으로 조정했다.

## 결정 사항
- 서비스명은 `월간사주`로 한다.
- 제품 정체성은 사주 기반 라이프 에이전트와 캐릭터챗의 관계감을 결합한다.
- v1은 빠른 MVP 우선이지만 AI 모델 제공자, 장기기억, 고급 자동화 확장 포인트를 열어둔다.
- `소연` 캐릭터는 `하나`로 변경한다.

## 검증
- `pnpm test`: 통과
- `pnpm build`: 통과
- 변경 파일 대상 `eslint`: 통과
- 전체 `pnpm lint`: 원본 코드의 기존 lint 오류로 실패
