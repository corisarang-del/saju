# 첫채팅 응답품질 저장전 게이트 추가

## 날짜
2026-07-07

## 문제
- 첫 상담 응답이 5문단으로 저장되는 케이스가 있었다.
- 첫 상담 응답에 이모지가 들어가는 케이스가 있었다.
- 무료 첫 상담 응답이 질문으로 끝나지 않는 케이스가 있었다.
- Gemini live QA에서 썸/재회는 사주 근거, 번아웃은 구체 행동이 확인필요였다.

## 원인
- 첫 상담 프롬프트에는 1~3문단, 이모지 금지, 질문 종료 규칙이 있었지만 저장 전 서버 게이트가 없었다.
- 캐릭터 원본 프롬프트의 한자 예시에 이모지가 들어 있어 모델이 그대로 따라 쓸 여지가 있었다.
- live QA 프롬프트가 사주라는 단어와 번아웃 구체 행동을 충분히 강제하지 못했다.

## 수정
- `src/lib/ai/chat-completion-guard.ts`
  - 첫 상담 저장 전 품질 게이트를 추가했다.
  - 4문단 이상, 이모지 포함, 마지막 질문 누락, 사주 근거 누락, 오늘 구체 행동 누락, 금지 표현 포함 시 저장하지 않게 했다.
- `src/lib/saju/initial-analysis.ts`
  - 반드시 사주라는 단어를 넣게 했다.
  - 오늘 행동은 기록하기, 정리하기, 비교하기, 나누기, 짧게 말하기 같은 구체 행동으로 제한했다.
  - 번아웃 케이스는 몸 상태 기록/정리 행동으로 유도했다.
- `src/lib/saju/characters.ts`
  - 캐릭터 프롬프트의 이모지 예시를 한글 설명으로 교체했다.
- `scripts/qa-gemini-first-consultation.mjs`
  - 사주 근거, 번아웃 구체 행동, 금지 표현 기준을 강화했다.

## 검증
```bash
pnpm test src/lib/ai/chat-completion-guard.test.ts src/lib/saju/chat-stream-failure-regression.test.ts src/lib/saju/characters.test.ts src/lib/saju/first-consultation-quality.test.ts src/lib/saju/gemini-first-consultation-qa-runner.test.ts src/lib/saju/chat-generation.test.ts
pnpm exec tsc --noEmit
node scripts/qa-gemini-first-consultation.mjs
```

- 최신 리포트: `docs/qa/gemini-first-consultation-qa-2026-07-07.md`.
