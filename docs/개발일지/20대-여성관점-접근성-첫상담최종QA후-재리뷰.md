# 20대 여성관점 접근성 첫상담최종QA후 재리뷰

- 소스코드는 수정하지 않고 개발자 수정분을 읽기 전용으로 확인했다.
- 로컬 `localhost:3000` 서버가 꺼져 있어 Playwright 실제 화면 스냅샷은 확인하지 못했다.
- 대신 `docs/qa/gemini-first-consultation-qa-2026-07-03.md`, 첫 상담 프롬프트, 채팅 API, 로그인 모달, 입력폼, 관련 회귀 테스트를 기준으로 유저 관점에서 다시 봤다.
- 종합 평가는 `9.7/10`이다. 지난번 남은 접근성, 압박 문구, 첫 상담 품질 리스크가 대부분 닫혔다.

## 좋아진 점

- 캐릭터 카드 로그인 모달에 `role="dialog"`, `aria-modal`, 제목/설명 연결, Google 로그인 버튼 접근성 이름, 닫기 버튼 접근성 이름이 들어갔다.
- 모달은 `Escape`로 닫히고, `Tab`/`Shift+Tab` 포커스 순환과 닫은 뒤 트리거 포커스 복귀까지 들어가 키보드 유저 마찰이 크게 줄었다.
- `돈 냄새`, `놓치면 안 돼`, `꼭 기억하셔야 해요` 같은 압박형 문구는 캐릭터 원문과 최종 캐릭터 목록 모두에서 걸러지도록 테스트가 생겼다.
- 2026-07-03 Gemini 5케이스 live QA는 `썸/재회`, `이직/퇴사`, `돈 모으기`, `번아웃`, `친구/가족관계` 모두 통과했다.
- 5케이스 모두 고민 반영, 금지 표현 없음, 1~3문단, 질문으로 끝남, 이모지 없음, 가벼운 외래어 없음 기준을 통과했다.
- 첫 상담 프롬프트는 정확히 2문단, 마지막 질문 1개, 물음표 종료, 외래어/이모지 금지, 괄호 외래어 병기 금지까지 좁혀졌다.
- 실제 API 단건 QA도 2문단, 마지막 질문 종료, 이모지 없음, 가벼운 외래어 없음, 커리어 고민 반영, 별 차감, 메시지 저장, `chat_used` 증가까지 확인된 상태다.
- 쿠키 안내, 개인정보 안내, 태어난 시간 미입력 가능 안내, 성별 사용 목적 안내, `3회 무료`, `1별 = 메시지 1회` 흐름은 유지되고 있다.

## 유저 관점에서 아직 볼 점

- 5케이스 live QA는 좋은 품질 게이트지만, QA 스크립트가 실제 앱 대화 컨텍스트 전체를 통과하는 방식은 아니다. 실제 앱 API 전체 경로는 커리어 단건 위주로 확인된 상태다.
- QA 답변 중 돈 모으기 샘플에 `민준님`처럼 캐릭터 이름이 유저 호칭처럼 보이는 표현이 있었다. 실제 앱에서는 사용자 이름 기반 호칭을 쓰지만, QA 보고서만 보면 어색하게 읽힌다.
- `doyun` 초기 분석 프롬프트와 일부 캐릭터 내부 문구에는 아직 `타이밍`이 남아 있다. 대표 5케이스 QA에는 통과했지만, 사업/창업 상담 첫 응답까지 같은 기준으로 한 번 더 확인하면 더 안전하다.
- 로컬 서버가 꺼져 있어 이번 라운드에서는 모바일 실제 렌더링과 모달 포커스 동작을 브라우저에서 직접 재확인하지 못했다.

## 다음 우선순위

- 실제 앱 API 경로로 대표 5케이스를 모두 한 번씩 돌려서 QA 스크립트 결과와 앱 응답이 일치하는지 확인한다.
- 사업/창업 캐릭터 `도윤`의 첫 상담도 외래어, 압박감, 마지막 질문 종료 기준으로 별도 확인한다.
- 서버를 띄운 뒤 모바일 390x844에서 로그인 모달 포커스, Escape 닫기, 쿠키 안내, 입력폼 흐름을 다시 스냅샷으로 검증한다.

## 개발자 후속 처리

- `scripts/qa-gemini-first-consultation.mjs`에 `사업/창업` 도윤 케이스를 추가해 6케이스 live QA로 확장했다.
- QA 케이스마다 `userName`을 넣고, 캐릭터 이름을 사용자 호칭으로 쓰는 응답을 `캐릭터명 호칭 없음` 컬럼으로 검출하게 했다.
- `도윤님`, `민준님`처럼 캐릭터명을 사용자 이름처럼 부르는 문제를 막기 위해 QA 프롬프트에 사용자 이름 호칭 규칙을 추가했다.
- `체크`, `플랜`, `솔루션`, `타이밍` 금지어는 각각 `확인`, `계획`, `방법`, `시기`로 바꾸도록 첫상담 프롬프트와 실제 채팅 API 프롬프트를 강화했다.
- 캐릭터 원문과 랜딩 문구에서 `타이밍`을 `시기`로 바꿨고, 일괄 치환으로 생긴 `시기이야`, `시기을` 같은 어색한 문구를 정리했다.
- 랜딩 후기의 `체크할 조건`은 `확인할 조건`으로 바꿨다.
- `/ko/reading`을 390x844 모바일 크기로 확인했다.
  - 쿠키 안내는 상단 영역에 표시되고 입력폼을 막지 않는다.
  - 개인정보 안심 문구, 태어난 시간 미입력 가능 안내, 성별 사용 목적 안내가 보인다.
  - 이름, 생년월일, 성별 입력 후 2/2 고민 선택 화면으로 전환된다.
  - 비로그인 상태에서 `분석 시작하기` 클릭 시 `로그인이 필요합니다.`가 표시된다.
- `/ko` 랜딩에서 캐릭터 카드 로그인 모달을 확인했다.
  - 스냅샷 기준 `dialog "가입하고 바로 시작하세요"`가 잡힌다.
  - Google 로그인 버튼과 닫기 버튼의 접근성 이름이 잡힌다.
  - `Escape`로 닫히고 포커스가 캐릭터 카드 버튼으로 돌아온다.
- Playwright 콘솔 확인 결과 에러 0개, 경고 0개다.
- 최신 `docs/qa/gemini-first-consultation-qa-2026-07-03.md`는 6케이스 모두 고민 반영, 금지 표현 없음, 1~3문단, 질문으로 끝남, 이모지 없음, 가벼운 외래어 없음, 캐릭터명 호칭 없음 기준을 통과했다.

## 후속으로 남은 점

- 이번 라운드에서는 브라우저 UI와 독립 live QA 6케이스는 확인했지만, 실제 인증 앱 API 전체 경로 6케이스 반복 호출은 별도 자동화가 필요하다.
