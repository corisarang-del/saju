# 20대 여성고객 9점대 신뢰도 개선 구현

- 일시: 2026-07-01 00:04:40 KST

## 사용한 접근

- `design-taste-frontend` 기준으로 문구, 접근성, 첫인상 우선순위를 조정했다.
- `verification-before-completion` 기준으로 완료 전 fresh 검증을 실행했다.
- TDD 방식으로 디자인/프롬프트 회귀 테스트를 먼저 추가하고 실패를 확인한 뒤 구현했다.

## 변경 내용

- 쿠키 배너 문구를 `로그인과 서비스 이용에 필요한 쿠키만 사용해.`로 변경했다.
- 생년월일 입력, 채팅 입력, 로그인 사이드바 편집 폼의 성별 영역에 `성별은 사주 계산 기준에 필요해서만 사용해.` 안내를 추가했다.
- 현우 캐릭터 카드 문구와 프롬프트에서 위협적인 표현을 줄이고, `조심할 포인트`, `미리 알면 덜 흔들려`, `주의 포인트` 중심으로 바꿨다.
- API 채팅 프롬프트 예시의 `좀 위험한데` 표현을 `조심할 포인트가 있어`로 완화했다.
- 랜딩 가격표와 별 충전 상품 버튼에 접근성 라벨을 추가하고, 시각 배지는 `aria-hidden` 처리했다.
- 데스크톱 비로그인 사이드바 문구를 `무료 3회로 먼저 확인하기` 중심으로 바꿔 로그인 요구보다 가치 제안을 먼저 보이게 했다.
- 사이드바 사주 정보 편집 이름 입력을 단일 입력으로 정리했다.
