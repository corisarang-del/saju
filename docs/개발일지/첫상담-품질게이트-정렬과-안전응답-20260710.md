# 첫상담 품질게이트 정렬과 안전응답

## 배경
- QA 문서 `docs/개발일지/수정후-실사용-QA-첫상담-품질불안정-20260710.md`에서 실제 `/api/saju/chat` 첫 상담이 반복적으로 불안정하다고 보고됐다.
- 주요 증상은 첫 상담 3회 재생성 후 503, 내부 게이트 통과 후 외부 QA의 `오늘 할 구체 행동 없음` 실패, 무료 채팅 `chat_message:-1` 거래 로그 미확인이었다.

## 수정 내용
- `src/lib/ai/chat-completion-guard.ts`
  - 첫 상담 저장 전 게이트를 외부 QA 기준에 맞춰 강화했다.
  - 정확히 2문단, 마지막 질문 종료, `사주` 근거, 두 번째 문단의 오늘 구체 행동, 이모지 없음, 영문자 없음, 가벼운 외래어 금지를 같은 기준으로 검사한다.
  - `getInitialAnalysisQualityReport`를 추가해 실패 로그에 문단 수, 질문 종료, 사주 근거, 오늘 행동, 이모지, 영어, 금지 패턴 여부를 구조화해서 남길 수 있게 했다.
- `src/app/api/saju/chat/route.ts`
  - 첫 상담 모델 재생성이 3회 모두 실패해도 바로 503을 내지 않고, 저장 전 게이트를 통과하는 안전 답변 fallback을 생성해 저장/전송한다.
  - fallback은 캐릭터별 고민 축에 맞춰 관계, 재회, 돈, 일, 사업, 건강 키워드를 반영하고 두 번째 문단에 `오늘 ... 기록하고 ... 정리` 행동을 고정한다.
  - 품질 실패 로그에 `qualityReport`를 남기고, fallback 사용 여부를 `usedFallback`으로 기록한다.
- `src/lib/saju/initial-analysis.ts`
  - 첫 상담 지시문의 `1~3문단`과 `정확히 2문단` 충돌을 제거했다.
  - 영문자와 `루틴, 패턴, 플랜, 체크, 밸런스, 리스크, 포인트, 타이밍` 금지를 명시했다.
- `scripts/qa-gemini-first-consultation.mjs`
  - live Gemini QA 기준을 내부 게이트와 맞췄다.
  - 정확히 2문단, 두 번째 문단의 오늘 행동, 짧은 영문자 포함 여부, 추가 외래어 금지를 검사한다.
- `scripts/qa-live-api-check.mjs`
  - 실제 API QA의 `오늘 할 구체 행동` 판정을 두 번째 문단 기준으로 좁혔다.

## 남은 확인
- 무료 첫 상담 거래 로그 누락은 원격 DB의 `decrement_star` 함수가 이전 버전이어서 발생했다.
- 원격 Supabase 마이그레이션 히스토리는 비어 있었으므로 전체 `db push` 대신 `202607070900_user_memberships_and_chat_transaction_log.sql`만 `supabase db query --linked --file ...`로 적용했다.
- 적용 후 `chat_message:-1` 거래 로그가 무료/유료 채팅 모두에서 확인됐다.
