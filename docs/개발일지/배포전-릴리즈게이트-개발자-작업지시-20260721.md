# 배포전 릴리즈게이트 개발자 작업지시

## 작업 내용
- 배포 전 개발자가 처리해야 할 보안/운영 릴리즈게이트를 문서화했다.
- 문서 위치: `docs/pm/배포전-릴리즈게이트-개발자-작업지시-20260721.md`.

## 정리한 P0 항목
- Vercel `monthlysaju` production env 전체 등록.
- `REQUIRE_PADDLE_ENV=true` 기반 Paddle env 누락 차단.
- Supabase `user_stars`, `star_transactions` 직접 조작 정책 제거.
- 채팅 별 차감 원자성 재설계.
- in-memory rate limit을 공유 저장소 기반으로 교체.
- Paddle subscription webhook price/product id 검증.
- `pnpm audit --prod` 취약 패키지 업데이트와 재검증.

## 정리한 검증 기준
- 각 항목은 TDD로 실패 테스트를 먼저 작성한다.
- 최종 검증 명령은 `REQUIRE_PADDLE_ENV=true pnpm test:env`, `pnpm test`, `pnpm exec tsc --noEmit`, `pnpm lint`, `pnpm build`, `pnpm audit --prod`다.

## 주의
- 이 작업은 문서화 작업이라 앱 코드와 DB 정책을 직접 변경하지 않았다.
- 실제 env 값과 시크릿은 문서에 남기지 않았다.
