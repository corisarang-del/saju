# 채팅 초기분석 멈춤 복구

- 일시: 2026-07-01 00:13:24 KST

## 원인 분석

- 채팅 화면은 사주 정보 저장 후 빈 대화면 자동으로 초기 분석 프롬프트를 전송한다.
- 자동 전송 직전에 `sessionStorage`에 자동 시작 완료 키를 먼저 저장하고 있었다.
- API, 모델, 네트워크, DB 저장 중 하나가 실패하면 화면에는 에러가 보이지 않고, 저장된 자동 시작 키 때문에 새로고침 후에도 자동 분석이 다시 시작되지 않을 수 있었다.
- 서버 스트리밍 경계에도 `onError` 로그가 없어 실제 실패 원인을 운영 로그에서 찾기 어려웠다.

## 변경 내용

- `ChatRoom`에 채팅 생성 실패 상태를 추가했다.
- `useChat`의 `onError`에서 사용자에게 `분석을 시작하지 못했어. 잠시 후 다시 시도해줘.` 메시지를 보여주게 했다.
- 자동 초기 분석 실패 시 `sessionStorage` 자동 시작 키를 제거해서 재시도 가능하게 했다.
- 에러 UI에 `다시 분석하기` 버튼을 추가했다.
- 수동 메시지 전송 실패도 같은 복구 경로를 타도록 했다.
- `/api/saju/chat`의 `streamText`에 `onError` 로그를 추가했다.
