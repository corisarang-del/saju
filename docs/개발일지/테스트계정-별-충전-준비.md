# 테스트계정 별 충전 준비

- Supabase Auth에서 `corisarang@gmail.com` 유저 조회는 가능했으나 `public.user_stars` 테이블이 없어서 충전이 실패했다.
- Supabase REST OpenAPI 스키마에서도 공개 테이블이 잡히지 않아 앱 DB 스키마가 아직 생성되지 않은 상태로 판단했다.
- 실제 차감 로직이 쓰는 `user_stars`, `star_transactions`, `decrement_star` 기본 스키마 마이그레이션을 추가했다.
- 스키마 생성 후 같은 서비스 롤 방식으로 해당 계정에 별 10000개를 충전하면 기존 차감 로직과 동일하게 동작한다.
