# 캐릭터 인영 변경과 말투차별화 문서화 20260724

## 작업 내용

`하은`을 `인영`으로 변경하고 각 상담사 말투/화법을 더 특색 있게 만드는 방향을 개발자 전달 문서로 정리했다. 소스코드는 수정하지 않았다.

## 핵심 판단

- `하은`은 `인영`으로 바꾸는 방향이 좋다.
- 내부 id `haeun`은 유지하고, 유저에게 보이는 이름과 프롬프트 자기소개만 `인영`으로 바꾼다.
- 캐릭터 말투는 말끝 차이보다 답변 구조, 선호 표현, 금지 표현으로 차별화해야 한다.
- 공통 말투 안전장치는 유지하되 캐릭터별 개성을 덮어쓰지 않게 구조화가 필요하다.

## 개발자 전달 핵심

1. `CHARACTERS.haeun.name`과 prompt 내 자기소개를 `인영`으로 바꾼다.
2. `id: "haeun"`, `CharacterType`의 `"haeun"`, 기존 저장 데이터 호환은 유지한다.
3. analytics 표시명도 `인영 (운세분석)` 등으로 맞춘다.
4. 상담사별 tone profile을 구조화해 답변 구조와 금지 표현을 분리한다.
5. 대표 고민 5개와 `인영` 전용 시기 질문으로 실제 Gemini/API QA를 돌린다.

## PM 판단

`인영`은 시간의 흐름, 월운, 길일 상담사로 더 성숙하고 차분한 인상을 준다. 다만 하나/지안과 모두 여성 존댓말 상담사라서 하나는 감정 번역, 지안은 재회 조건, 인영은 시기와 준비 행동으로 명확히 분리해야 한다.
