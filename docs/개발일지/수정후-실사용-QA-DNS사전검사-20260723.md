  - `node scripts/qa-live-api-check.mjs`
- 결과:
  - 둘 다 `Supabase DNS lookup failed`로 종료.
- 오류 상세:
  - host: `sfpwgywcmhgilrqearsz.supabase.co`
  - code: `ENOTFOUND`
  - errno: `-3008`
  - syscall: `getaddrinfo`
- QA 스크립트가 제안한 다음 조치:
  - 현재 네트워크나 DNS에서 Supabase 프로젝트 호스트를 해석할 수 있는지 확인.
  - `NEXT_PUBLIC_SUPABASE_URL`의 project ref가 운영 Supabase ref와 같은지 확인.
  - DNS가 정상인 환경에서 QA 스크립트 재실행.

## 관찰한 개선점
- `scripts/qa-live-api-check.mjs`는 이전처럼 Supabase auth user 생성 실패만 보여주지 않고, DNS 사전검사 실패 원인과 다음 조치를 구조화해서 출력한다.
- Playwright 스냅샷 기준 `지안` 카드 문구가 `인연의 끈은 쉽게 끊어지지 않아요. 근데 문제는...`에서 `남은 마음이 있다면, 다시 잡아도 괜찮은지 차분히 봐드릴게요.`로 바뀌어 덜 자극적이다.

## 주의할 점
- Gemini 단독 QA는 6케이스 통과지만, `썸/재회` 케이스에서 관계 고민에 `오늘 몸 상태를 3줄로 기록해 보세요`가 나왔다.
- 자동 기준상 구체 행동으로 통과했지만, 사람 관점에서는 고민 도메인과 행동 제안의 정합성을 더 강하게 볼 필요가 있다.
