# 배포후 PC 폰 디자인개선 반영

- 일자: 2026-07-24
- PM 문서: `docs/pm/배포후-PC-폰-디자인개선-개발자전달-20260724.md`
- 사용 스킬: `design-taste-frontend`, `playwright`

## 수정 내용

- `src/lib/design/design-audit-regression.test.ts`에 배포 후 디자인 QA 회귀 테스트를 먼저 추가했다.
- `src/components/saju/landing/CharacterCards.tsx`에서 캐릭터 캐러셀을 `max-w-5xl` 본문 기준으로 맞추고 PC edge fade mask, scroll padding, desktop/mobile snap 기준을 분리했다.
- 모바일 카드 폭을 `auto-cols-[66vw]`, 최대 폭 `max-w-[250px]`로 낮추고 이미지 비율을 모바일 `aspect-[4/5]`로 줄여 첫 viewport 안에서 CTA가 보이게 했다.
- `src/app/[locale]/page.tsx`에서 H1 max-width와 desktop font-size를 조정하고, `오늘의 흐름부터`, `먼저 정리해줄게`를 의미 단위 `inline-block`으로 묶어 단어 중간 줄바꿈을 막았다.
- `src/components/saju/input/BirthDateForm.tsx`에서 placeholder/helper/보조 라벨 대비를 한 단계 올렸다.
- `src/app/api/analytics/track/route.ts`를 추가해 client analytics POST가 404가 아니라 no-op 성공 또는 정상 trackEvent 호출로 처리되게 했다.

- 통과 확인: 수정 후 `./node_modules/.bin/vitest run src/lib/design/design-audit-regression.test.ts` 통과.
- 통과 확인: `./node_modules/.bin/tsc --noEmit` 통과.
- 통과 확인: 대상 파일 `./node_modules/.bin/eslint ...` 통과.
- 통과 확인: sandbox 밖 `./node_modules/.bin/next build` 통과. `/api/analytics/track` 라우트가 빌드 결과에 포함됐다.
- Playwright 모바일 390x844 확인: 쿠키 배너가 있는 상태에서 첫 카드의 `대화하기` CTA가 viewport 안에 보인다.
- Playwright PC 1440x900 확인: 캐러셀은 사이드바와 직접 겹치지 않고 본문 영역 안에서 edge fade peek로 보인다. H1은 의미 단위로 자연스럽게 줄바꿈된다.
- Playwright console 확인: console error/warning 0건.

## 남은 참고사항

- `pnpm exec`, `pnpm build` 래퍼는 기존 관찰 포인트처럼 현재 환경에서 `[ERROR] fetch failed`가 재현됐다. 동일 명령의 로컬 바이너리 직접 실행은 통과했다.
- production 실제 analytics 수집 여부는 배포 후 운영 콘솔에서 한 번 더 확인해야 한다.
