- `/api/saju/update-status`에 `status: paid` 직접 요청:
  - 응답: 403 `Forbidden status transition`
  - 리딩 상태: `pending` 유지
  - 판정: 통과
- `/api/saju/deduct-stars`:
  - 응답: 200 `{ "balance": 5, "amount": -5 }`
  - 리딩 상태: `paid`
  - 거래 로그: `amount=-5`, `balance_after=5`, `type=report`
  - 판정: 통과
- `/api/monthly-saju/deduct-monthly-report`:
  - 첫 호출: 200 `{ "balance": 2, "amount": -3, "alreadyUnlocked": false }`
  - 두 번째 호출: 200 `{ "balance": 2, "amount": 0, "alreadyUnlocked": true }`

## Gemini live QA 결과

`node scripts/qa-gemini-first-consultation.mjs`를 실제 모델 호출로 실행했고, 리포트는 `docs/qa/gemini-first-consultation-qa-2026-07-07.md`에 갱신됐어.

- 6케이스 중 형식 대부분은 통과했어.
- `썸/재회`: 사주 근거 `확인필요`
- `번아웃`: 구체 행동 `확인필요`
- 이모지, 외래어, 영어 혼합, 캐릭터명 호칭 기준은 통과했어.

## 남은 문제

1. 유료 첫 채팅 응답이 아직 1~3문단 규칙을 깨고 5문단으로 저장돼.
2. 유료 첫 채팅 응답에 이모지가 들어가.
3. 무료 첫 채팅 응답이 질문으로 끝나지 않아.
4. Gemini live QA에서 `사주 근거`와 `구체 행동` 기준이 각각 1건씩 확인필요로 남아 있어.

## 개발자에게 전달할 포인트

- 단위 테스트는 통과하지만 실제 `/api/saju/chat` 라이브 응답 품질은 아직 닫히지 않았어.
- `getFirstConsultationInstructions`나 캐릭터 유료/무료 프롬프트만 볼 게 아니라, `advancedContext`의 한자/동물 설명이 모델에게 이모지를 유도하는지도 같이 봐야 해.
- 무료/유료 모두 첫 상담 마지막 문장을 강제로 질문으로 끝내는 후처리나 더 강한 검증 로직이 필요해 보여.
- 문단 수는 생성 지시만으로는 불안정하니 저장 전 검증 또는 재생성 기준을 고려해야 해.
