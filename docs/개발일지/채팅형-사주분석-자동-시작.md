# 채팅형 사주분석 자동 시작

## 배경

사주분석 버튼을 눌렀는데 분석 결과가 나오지 않는 문제가 있었다. 원격 Supabase에는 최신 `saju_readings`가 없었고, Gemini 최소 호출은 정상 동작했다. 코드 흐름상 채팅형 `사주 분석 시작`은 사주 정보를 저장한 뒤 첫 분석 질문을 자동 전송하지 않고 대기 메시지만 보여주고 있었다.

## 작업

- `src/lib/saju/initial-analysis.ts` 추가.
- 새 reading이고 메시지가 없을 때만 첫 분석을 자동 시작하는 순수 조건 함수를 만들었다.
- 캐릭터별 초기 분석 프롬프트를 분리했다.
- `ChatRoom`에서 새 reading 진입 시 `sessionStorage` 중복 방지 키를 사용해 첫 분석 메시지를 자동 전송하도록 연결했다.
- `src/lib/saju/initial-analysis.test.ts`로 자동 시작 조건과 캐릭터별 프롬프트를 검증했다.

## 검증

- `./node_modules/.bin/vitest run` 통과: 13 files / 33 tests.
- `./node_modules/.bin/eslint src/components/saju/chat/ChatRoom.tsx src/lib/saju/initial-analysis.ts src/lib/saju/initial-analysis.test.ts` 통과.
