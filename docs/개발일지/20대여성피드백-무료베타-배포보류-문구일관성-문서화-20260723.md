# 20대 여성 피드백 무료베타 배포보류 문구일관성 문서화 20260723

## 작업 내용

20대 여성 유저 피드백을 개발자에게 전달할 수 있게 PM 문서로 정리했다. 소스코드는 수정하지 않았다.

## 반영한 피드백

- Supabase DNS 문제 때문에 실제 무료 상담 API QA가 아직 미완료다.
- 무료 첫 상담, 별 지급/차감, 거래 로그, assistant 저장 품질은 실제 DB 기준으로 아직 확인되지 않았다.
- 따라서 무료 베타 배포 승인은 보류가 맞다.
- 무료 베타라고 안내하면서 푸터와 JSON-LD에는 결제/별 충전 문구가 남아 있어 메시지 일관성이 부족하다.

## 개발자 전달 핵심

1. `sfpwgywcmhgilrqearsz.supabase.co` DNS `ENOTFOUND`를 먼저 해결한다.
2. DNS 정상 환경에서 `env QA_SCENARIO=free-only node scripts/qa-live-api-check.mjs`와 `node scripts/qa-live-api-check.mjs`를 다시 실행한다.
3. 실제 무료 상담 API QA가 통과되기 전에는 배포 승인으로 보지 않는다.
4. `PAYMENTS_ENABLED=false` 상태에서 푸터, JSON-LD, FAQ, placeholder, paywall, 정책 문서의 결제/충전 문구를 무료 베타 문구로 맞춘다.

## PM 판단

이번 이슈는 기능 추가보다 출시 신뢰성 문제다. 결제 기능은 나중에 열어도 되지만, 무료 베타에서 실제 무료 상담 흐름이 검증되지 않았거나 결제 가능처럼 보이는 문구가 남으면 유저 신뢰가 떨어질 수 있다.
