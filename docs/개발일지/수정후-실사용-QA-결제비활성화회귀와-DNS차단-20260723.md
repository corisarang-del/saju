## 막힌 항목

### 실제 API QA

다음 두 명령이 모두 같은 이유로 중단됐다.

- `env QA_SCENARIO=free-only node scripts/qa-live-api-check.mjs`
- `node scripts/qa-live-api-check.mjs`

출력:

```json
{
  "ok": false,
  "message": "Supabase DNS lookup failed",
  "extra": {
    "host": "sfpwgywcmhgilrqearsz.supabase.co",
    "code": "ENOTFOUND",
    "errno": -3008,
    "syscall": "getaddrinfo"
  }
}
```

이 상태에서는 무료 첫 상담, 별 지급, 별 예약/차감/환불, 채팅 거래 로그, assistant 저장 품질을 실제 DB 기준으로 확인할 수 없다.

## 추가로 발견한 문제

### P1. Supabase DNS 차단으로 실사용 API QA 미완료

개발자에게 전달할 내용:

- 현재 환경에서 `sfpwgywcmhgilrqearsz.supabase.co`가 DNS 해석되지 않는다.
- `scripts/qa-live-api-check.mjs`의 DNS 사전검사와 구조화된 실패 보고는 정상 작동한다.
- 하지만 실제 사용자 흐름 검증은 DNS가 정상인 환경에서 다시 해야 한다.
- 재검증 명령:
  - `env QA_SCENARIO=free-only node scripts/qa-live-api-check.mjs`
  - `node scripts/qa-live-api-check.mjs`

### P2. 무인증 채팅 API 오류 본문이 JSON이 아님

`POST /api/saju/chat` 무인증 요청은 HTTP 401로 막히지만 본문은 `Unauthorized` 텍스트다.

클라이언트가 모든 API 오류를 JSON으로 처리한다면 파싱 예외가 날 수 있다. 일관성을 위해 `{ "error": "Unauthorized" }` 같은 JSON 응답을 고려하는 게 좋다.

### P3. 무료 베타 문구가 일부 남은 결제 문구와 충돌함

검색 기준으로 결제 비활성화 안내는 많이 반영됐지만 일부 문구는 아직 결제 가능 상태를 암시한다.

- `src/app/[locale]/page.tsx`: `이후에는 필요한 만큼 별을 충전`
- `src/app/[locale]/layout.tsx`: JSON-LD FAQ의 `이후에는 별을 충전하여 계속 상담`
- `src/components/saju/landing/SajuFooter.tsx`: `결제는 Paddle의 안전한 결제 시스템`
- `src/components/saju/chat/ChatInput.tsx`: disabled placeholder `별을 충전해주세요`
- `src/components/saju/chat/ChatPaywall.tsx`: 결제 가능 상태 문구 분기 확인 필요

### P3. 접근성 스냅샷에서 띄어쓰기 붙음

Playwright 스냅샷에 `로그인하면 모든 대화 기록을저장하고 다시 볼 수 있어요`처럼 `기록을저장하고`가 붙어서 읽힌다. 시각적으로도 붙어 보일 수 있으니 실제 화면/보조기술 기준으로 확인이 필요하다.

### 참고. pnpm test/lint 래퍼 재실행 불안정

이번 라운드에서 `pnpm test`, `pnpm lint`, `pnpm exec vitest run`, `pnpm exec eslint`가 한때 `fetch failed`를 반환했다. 같은 검사 도구를 `node_modules/.bin`에서 직접 실행하면 통과했다.

빌드와 env 검사는 pnpm으로 통과했으므로 코드 실패로 단정하지는 않는다. 다만 재현되면 pnpm/corepack/네트워크 상태를 별도로 확인해야 한다.
