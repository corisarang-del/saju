# 하나 궁합상담 상대방정보 미반영 수정

## 문제

- 사용자가 하나와 상담할 때 상대방 정보를 함께 입력했는데도 첫 상담 답변이 내 정보 중심으로만 나오는 것처럼 보였다.
- 실제 코드에는 `saju_compatibility` 조회와 상대방 컨텍스트 조립이 있었지만, 첫 자동 상담 사용자 문구가 `내 사주를 바탕으로`로 시작해 모델이 사용자 한 명 중심으로 답하기 쉬운 구조였다.
- 첫 상담 품질 재시도 실패 시 사용하는 안전 대체 답변도 상대방 이름과 궁합 맥락을 받지 않아, fallback 경로에서는 확실히 내 정보 중심 답변이 나올 수 있었다.
- 채팅 API의 궁합 row 조회가 `.single()`이라 같은 reading에 row가 복수로 생기면 상대방 컨텍스트가 비어질 수 있었다.

## 수정

- `getInitialAnalysisPrompt("charon_f")`를 두 사람의 관계와 궁합 흐름, 상대방 정보를 함께 보는 문구로 변경했다.
- `buildSafeInitialAnalysisFallback`을 `src/lib/saju/initial-analysis.ts`로 이동하고 테스트 가능한 순수 함수로 만들었다.
- 하나/지안 궁합 fallback은 `partnerName`이 있으면 사용자와 상대방 이름, 두 사람의 궁합, 오늘 기록 행동을 함께 포함한다.
- 채팅 API는 `compatibilityPartnerName`을 보관해 첫 상담 fallback에도 전달한다.
- `saju_compatibility` 조회는 최신 row 기준 `.order("created_at", { ascending: false }).limit(1).maybeSingle()`로 바꿔 복수 row에서도 상대방 컨텍스트를 최대한 유지한다.
- 상대방 만세력/오행 JSON이 비어 있는 오래된 궁합 row에서도 상대방 이름, 성별, 생년월일 기본 정보는 프롬프트에 남도록 기본 정보와 분석 블록을 분리했다.
- 시스템 프롬프트에 상대방 이름과 “첫 답변부터 두 사람을 함께 언급” 규칙을 추가했다.
