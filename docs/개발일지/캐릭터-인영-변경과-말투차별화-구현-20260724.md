# 캐릭터 인영 변경과 말투차별화 구현 20260724

## 배경

PM 전달 문서 기준으로 `하은`은 시간/월운/길일 상담사 포지션에 비해 이름 인상이 약해서 `인영`으로 변경하기로 했다. 단, 기존 저장 데이터와 URL 호환을 위해 내부 id `haeun`은 유지한다.

  - 최종 프롬프트에 `## 캐릭터별 답변 구조`가 주입되고 압박 표현이 남지 않는지 확인했다.

- 실패 확인 후 구현했다.

## 구현

- `src/lib/saju/characters.ts`
  - `CharacterToneProfile` 타입과 `toneProfile` 필드를 추가했다.
  - 현우, 하나, 민준, 인영, 지안, 서준, 도윤의 답변 순서, 선호 표현, 피할 표현, 마지막 질문 방식을 구조화했다.
  - `withSoftTone()`에서 캐릭터별 tone profile을 최종 prompt에 자동 주입하도록 했다.
  - `haeun` id는 유지하고 노출명만 `인영`으로 바꿨다.
  - `freePrompt`, `paidPrompt`의 자기소개를 `넌 '인영'이야`로 바꿨다.
  - 인영의 톤을 예언자식 단정이 아니라 월 단위 흐름과 준비 행동을 정리하는 상담사로 낮췄다.
- `src/components/saju/chat/ChatRoom.tsx`
  - 인영 첫 안내 문구를 `보이네요..` 반복 톤에서 차분한 월운/시기 안내 톤으로 바꿨다.
- `src/services/analytics/actions.ts`
  - 관리자 분석용 캐릭터 표시명을 `인영 (운세분석)`으로 변경했다.
- `scripts/qa-gemini-first-consultation.mjs`
  - 번아웃 QA 케이스 캐릭터명을 `인영`으로 변경했다.
