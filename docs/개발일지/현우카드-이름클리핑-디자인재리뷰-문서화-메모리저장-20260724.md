# 현우카드 이름클리핑 디자인재리뷰 문서화 메모리저장

## 작업 내용

- 사용자가 제보한 PC `현우` 카드 이름 짤림 현상을 production에서 재검증했다.
- `/ko` 홈을 1280x720, 1366x768, 모바일 390x844 기준으로 확인했다.
- `/ko/reading` 모바일 상단/하단을 다시 확인해 이전 개선점 유지 여부를 검증했다.
- 개발자 전달 문서와 프롬프트 기록을 작성했다.
- `.codex/memory/project.md`, `.codex/memory/learnings.md`에 최신 디자인 리뷰 결과를 저장했다.

## 확인 결과

- 1280x720 PC에서 `현우` 이름 영역이 `y=709`, `bottom=737`로 viewport height `720`을 넘어 실제로 잘린다.
- 1366x768에서는 같은 이름 영역이 viewport 안에 온전히 들어와서 높이 720px 전후의 낮은 PC 화면 문제가 핵심이다.
- 모바일 홈 H1/CTA 개선은 유지되고 있다.
- `/ko/reading` 시간 선택 placeholder와 하단 `다음` CTA도 안정적이다.
- 소스코드는 변경하지 않았다.
