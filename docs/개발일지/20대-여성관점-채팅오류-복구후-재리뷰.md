# 20대 여성관점 채팅오류 복구후 재리뷰

- 소스코드는 수정하지 않고 `/ko`, `/ko/reading` 화면과 관련 구현을 읽기 전용으로 확인했다.
- 이번 수정은 이전에 남았던 릴리즈 리스크였던 쿠키 배너 가림, 빈 AI 응답, 첫 상담 품질 쪽을 꽤 직접적으로 건드렸다.
- 종합 평가는 `9.5/10`이다. 이제 랜딩 설득력보다 실제 상담 안정성까지 어느 정도 방어된 상태로 보인다.

## 유저 입장에서 좋아진 점

- 쿠키 안내가 fixed overlay가 아니라 문서 흐름 안의 상단 `쿠키 안내` 영역으로 바뀌었다.
- `/ko`와 `/ko/reading` 모두 쿠키 안내가 CTA나 입력폼을 덮지 않는다.
- 개인정보처리방침 링크도 `/ko/privacy-policy`로 유지되어 한국어 흐름이 끊기지 않는다.
- 랜딩 첫 화면은 `무료 3회로 먼저 확인하기`, `가입하면 3회 무료`, `1별 = 메시지 1회`, 가격표까지 이어져서 가격을 숨기는 느낌이 거의 없다.
- 캐릭터 카드 클릭 후 로그인 모달에도 `가입하면 3회 무료 상담`, `1별 = 메시지 1회, 가입 후에도 가격을 확인할 수 있어요`가 보여서 가입 압박이 덜하다.
- `/ko/reading` 입력 화면은 개인정보 사용 목적, 태어난 시간 미입력 가능 안내, 성별 사용 목적이 잘 보인다.
- 채팅 실패 시 빈 화면으로 성공 처리되는 대신 `지금 AI 응답 한도가 잠시 막혔어. 잠시 후 다시 시도해줘.`와 `다시 분석하기`가 노출되는 구조가 생겼다.
- 서버도 빈 assistant 응답이면 저장과 별 차감을 건너뛰는 흐름이라, 유저 입장에서 “답변도 없는데 별이 차감됨” 리스크가 내려갔다.
- 첫 상담 답변 규칙에 `사용자가 고른 고민`, `첫 문장`, `1~3문단`, `다음 대화`, 태어난 시간 모를 때 안심 문구, 겁주는 표현 금지가 들어갔다.

## 아직 남은 아쉬움

- 로그인 모달은 시각적으로는 괜찮지만 접근성 기준으로 `role="dialog"`, `aria-modal`, 포커스 트랩이 확인되지 않는다. 일반 유저에게 큰 문제는 아니지만 키보드/스크린리더 유저에게는 마찰이 남을 수 있다.
- 일부 캐릭터 문구는 아직 살짝 자극적이다. 예를 들어 `돈 냄새`, `놓치면 안 돼`, `꼭 기억하셔야 해요`는 캐릭터성은 있지만 민감한 20대 여성 유저에게는 광고성/압박감으로 읽힐 수 있다.
- 실제 Gemini API를 붙인 로그인 후 첫 상담 라이브 품질은 아직 별도 QA가 필요하다. 회귀 테스트와 UI 방어는 좋아졌지만, 실제 답변이 충분히 차분한지는 샘플 대화로 확인해야 한다.

## 다음 우선순위

- 로그인 모달에 접근성 semantics와 닫기/포커스 흐름을 보강하면 9점대 후반으로 갈 수 있다.
- 자극적인 카드 문구는 캐릭터성을 유지하되 압박형 표현을 조금 더 생활언어로 낮추는 게 좋다.
