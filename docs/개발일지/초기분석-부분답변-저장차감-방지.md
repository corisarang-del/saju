# 초기분석 부분답변 저장차감 방지

## 문제

초기 분석에서 `현철 씨, 사`처럼 매우 짧은 partial 답변이 화면에 남는 문제가 있었다.

확인 결과 토큰 한도는 무료 4000, 유료 10000이라 정상 답변이 몇 글자에서 끊길 이유가 없었다. 더 가능성 높은 원인은 AI 스트림이 중간 실패했거나 비정상적으로 짧게 종료됐는데, 서버와 클라이언트가 이를 성공 응답처럼 처리하는 구조였다.

## 수정

- `src/lib/ai/chat-completion-guard.ts` 추가
  - `finishReason === "error"` 또는 `isError`면 실패로 판단
  - 초기분석 답변이 40자 미만이면 비정상 종료로 판단
  - 일반 후속 답변의 짧은 답은 허용
- `src/app/api/saju/chat/route.ts`
  - incomplete assistant response는 DB 저장하지 않음
  - 별 차감 RPC 호출하지 않음
  - 제목 생성도 진행하지 않음
- `src/components/saju/chat/ChatRoom.tsx`
  - AI SDK `onFinish`의 `isError`, `finishReason`, assistant 텍스트를 검사
  - 실패/비정상 짧은 초기분석이면 별 UI 차감 없이 재시도 안내 표시
- `src/lib/ai/chat-error-handling.ts`
  - 새 사용자용 에러 문구를 안전 문구로 보존

## 테스트

- `chat_completion_guard`
  - 스트림 에러 partial 답변 거부
  - 너무 짧은 초기분석 거부
  - 짧은 후속 답변 허용
  - 충분한 초기분석 허용
- 기존 채팅 스트림 회귀 테스트 갱신
