# 배포보안 20260707 OAuth와 리포트생성 복구 수정

## 날짜
2026-07-07

## 문서

## 수정
- 운영 OAuth redirect origin 하드닝
  - production에서는 `APP_ORIGIN` 또는 `NEXT_PUBLIC_APP_URL` 고정값을 헤더보다 우선 사용하게 했다.
  - 조작된 `Origin`, `x-forwarded-host`가 운영 callback/redirect origin을 바꾸지 못하게 했다.
  - 로컬/터널 개발환경에서는 기존 forwarded host 흐름을 유지했다.
- 종합 리포트 analyze 실패 복구
  - `readingId`와 `userId`를 함수 초기에 보관해 catch에서 `req.clone().json()`에 의존하지 않게 했다.
  - `paid -> generating` 전이를 `.eq('status', 'paid')` 조건부 update로 바꿔 동시 생성 요청을 막았다.
  - 중복 생성 또는 이미 전이된 상태는 `409`로 응답하게 했다.
  - AI 생성 실패 시 `generating` 상태인 reading만 `failed`로 복구하게 했다.

## 남은 외부 환경 조치
- `REQUIRE_PADDLE_ENV=true pnpm test:env`는 현재 로컬 기준 Paddle production env가 비어 실패한다.
- 운영 Vercel에 Paddle production 값 13개를 넣은 뒤 다시 실행해야 한다.
- 운영 Supabase migration/RPC/RLS 확인은 이미 직접 적용한 이력이 있지만, 배포 전 운영 프로젝트에서 최종 재확인이 필요하다.
