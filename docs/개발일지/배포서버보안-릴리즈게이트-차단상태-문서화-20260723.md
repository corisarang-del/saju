# 배포 서버보안 릴리즈게이트 차단상태 문서화 20260723

## 작업 내용

배포 및 서버보안 담당자 피드백을 개발자 전달 문서로 정리했다. 소스코드는 수정하지 않았다.

## 반영한 피드백

- `pnpm run release:gate:payments`가 Paddle 필수 env 누락으로 실패한다.
- `PADDLE_API_KEY`, `PADDLE_WEBHOOK_SECRET`, `NEXT_PUBLIC_PADDLE_CLIENT_TOKEN`, 별 상품/가격 ID, 월간멤버십 상품/가격 ID가 비어 있다.
- Vercel `todocori/monthlysaju` production env가 아직 하나도 없다.
- 기본 `pnpm run release:gate`도 `qa:live-api:free` 단계에서 Supabase DNS `ENOTFOUND`로 실패한다.
- `release:gate:code`는 통과했지만 실제 무료 상담 API QA를 대체하지 않는다.
- `pnpm audit --prod` 전체 기준 `@ai-sdk/provider-utils` low 1건이 남아 있다.
- 결제 포함 production 배포라면 `release:gate:payments`까지 반드시 통과해야 한다.

## 개발자 전달 핵심

1. 무료 베타 배포도 `release:gate`가 통과해야 승인 가능하다.
2. 현재 `release:gate`는 Supabase DNS `ENOTFOUND` 때문에 실패한다.
3. `release:gate:code`만 보고 배포하면 안 된다.
4. 결제 포함 배포는 `release:gate:payments`까지 통과해야 한다.
5. Vercel production env 없음은 즉시 정리해야 한다.
6. audit low 1건은 즉시 차단급은 아니지만 AI 비용/리소스 리스크로 추적해야 한다.

## PM 판단

현재 배포 보류 판단은 유지한다. 결제 제외 무료 베타라도 실제 무료 상담 API QA가 닫히지 않았고 production env가 비어 있기 때문이다. 결제 포함 배포는 Paddle env가 전부 누락돼 있으므로 별도 보류다.
