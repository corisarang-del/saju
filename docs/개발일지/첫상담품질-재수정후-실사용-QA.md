## 실제 화면 QA

- 기존 dev 서버 `http://localhost:3000` 사용.
- `/ko`:
  - HTTP 200.
  - 페이지 제목 `월간사주 - 먼저 챙겨주는 사주친구` 정상.
- `/ko/reading`:
  - 1/2 생년월일 입력 화면 정상.
  - 이름, 생년월일, 성별 순차 입력 정상.
  - `다음` 클릭 후 2/2 고민 선택 화면 전환 정상.
  - `이직/퇴사` 선택 후 비로그인 상태에서 `분석 시작하기` 클릭 시 `로그인이 필요합니다.` 정상 표시.
- 테스트 중 병렬 입력으로 이름 칸에 연도가 섞이는 조작 실수가 있었지만, 순차 입력으로 재검증했을 때 앱 동작은 정상이었다.

## 실제 인증 채팅 API QA

- 임시 Supabase QA 유저, reading, 별 3개를 생성했다.
- 인증 쿠키를 만들어 `/api/saju/chat`에 실제 POST 요청을 보냈다.
- 요청 고민: `이직을 고민 중인데 올해 움직여도 괜찮을지 현실적으로 봐줘.`
- 응답:
  - status `200`
  - content-type `text/event-stream`
  - body length `1226`
  - quota 메시지 없음.
  - 이직/퇴사/회사/직장/커리어 고민 반영 확인.
- DB/과금 확인:
  - `saju_chat_messages`에 user 1건, assistant 1건 저장.
  - assistant content length `516`.
  - `user_stars.balance` 3에서 2로 차감.
  - `saju_readings.chat_used` 1로 증가.
  - title `이직운세` 생성 확인.
- 품질 확인:
  - 문단 수 `2`: 통과.
  - 마지막 문장 질문 종료: 통과.
  - 이모지 없음: 통과.
  - 가벼운 외래어 없음: 통과.
  - 커리어 고민 반영: 통과.
- 정리:
  - 임시 QA 유저, reading, 채팅 메시지, 별 데이터는 스크립트 finally에서 삭제 완료.

## 판단

- 실제 상담 API는 이번 재검증에서 기능과 핵심 품질 모두 통과했다.
- 저장, 별 차감, `chat_used`, 제목 생성도 정상이다.
- 화면 입력 흐름과 비로그인 차단도 정상이다.
- 하지만 독립 5케이스 live QA에서 금지 외래어 `체크`가 1건 남아 완전 통과는 아니다.
- 첫 상담 품질 릴리즈 게이트는 실제 API 단건 성공만으로 닫지 말고, 5케이스 live QA의 외래어 금지까지 전부 통과해야 한다.
