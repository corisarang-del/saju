## 실제 API QA 결과
### 무료 첫 상담 단독 QA 1차
- 명령: `env QA_SCENARIO=free-only node scripts/qa-live-api-check.mjs`
- 결과: 실패.
- 서버 로그:
  - `[saju/chat] prevalidated initial analysis generated`
  - `attempts: 2`
  - `POST /api/saju/chat 200`
- QA 실패:
  - `Free chat transaction count mismatch`
  - `extra: []`
- 해석:
  - 무료 첫 상담 생성 자체는 성공했지만, 채팅 별 차감 거래 로그가 확인되지 않았다.
  - 이 부분은 DB 함수 배포 상태 또는 채팅 차감 거래 기록 정책을 다시 확인해야 한다.

### 무료 첫 상담 단독 QA 3차
- 명령: `env QA_SCENARIO=free-only node scripts/qa-live-api-check.mjs`

  - `[saju/chat] prevalidated initial analysis generated`
  - `attempts: 2`

- 저장된 답변 QA 실패:
  - `free saved assistant: 오늘 할 구체 행동이 없음`

    - 사주 근거: 통과.
    - 이모지 없음: 통과.

## 개발자에게 전달할 핵심 이슈
### P1: 첫 상담 선검증 경로가 아직 안정적으로 통과하지 않는다
- 무료 첫 상담은 반복 실행에서 200, 503, 품질 미달 저장이 섞여 나온다.
- 유료 첫 상담도 전체 동시성 QA에서 503/409로 실패했다.
- 단독 Gemini QA가 통과해도 실제 API에서는 프롬프트, 토큰 상한, 저장 전 게이트, 제목 생성, 별 차감이 함께 작동하므로 별도 검증이 필요하다.

### P1: 내부 저장 전 게이트와 외부 QA 기준이 어긋나 있다
- 서버는 `prevalidated initial analysis generated`로 통과 처리했지만, 저장된 무료 답변은 오늘 할 구체 행동이 없었다.
- `shouldPersistAssistantAnswer`가 외부 QA의 핵심 품질 조건과 같은 기준을 공유하도록 맞춰야 한다.

### P2: 채팅 별 차감 거래 로그 확인 필요
- 무료 첫 상담 200 성공 케이스에서 `star_transactions`에 `chat_message:-1` 로그가 확인되지 않았다.
- 잔액 차감과 거래 기록을 함께 확인하도록 서버 RPC 배포 상태와 DB 함수 정의를 점검해야 한다.

## 권장 수정 방향
- 첫 상담 저장 전 게이트를 Gemini QA 평가기와 같은 기준으로 정렬.
- 품질 게이트 실패 사유를 구조화해서 로그에 남김.
  - 예: `paragraphCount`, `endsWithQuestion`, `hasConcreteTodayAction`, `hasLightForeignWord`.
- 무료/유료 모두 첫 상담 재생성 실패 시 정책 결정.
  - 503으로 실패할지.
  - 저장/차감 없이 안전 fallback을 줄지.
  - 추가 재시도나 모델/토큰 정책을 바꿀지.
- `decrement_star` 배포 함수가 채팅 거래 로그를 남기는지 운영 DB에서 확인.

- “제대로 작동되고 제대로 응답한다”고 말할 수 없다.
