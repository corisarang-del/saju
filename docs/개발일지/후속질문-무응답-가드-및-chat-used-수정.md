# 후속질문 무응답 가드 및 chat_used 수정

## 문제

첫 사주풀이 이후 사용자가 `응 알려줘`처럼 짧은 후속 답변을 보냈을 때 두 번째 AI 응답이 표시되지 않는 문제가 있었다.

조사 결과 두 가지 문제가 있었다.

1. 빈 assistant 응답이 후속 답변에서는 실패로 분류되지 않았다.
   - 서버는 빈 응답을 저장/차감하지 않지만, 클라이언트는 빈 후속 응답을 성공처럼 보고 별 UI를 차감할 수 있었다.
   - 이 경우 사용자는 답변이 비어 있는데도 아무 안내 없이 멈춘 것처럼 느낀다.

2. `saju_readings.chat_used`가 성공 답변 이후 증가하지 않았다.
   - 서버는 매 요청마다 `reading.chat_used === 0`이면 첫 상담 규칙을 붙인다.
   - 값이 증가하지 않으니 후속 질문도 계속 첫 상담으로 취급됐다.
   - `응 알려줘`처럼 앞 문맥을 이어받는 짧은 응답에서 대화 흐름이 불안정해질 수 있다.

## 수정

- `chat-completion-guard`
  - assistant 텍스트가 빈 문자열이면 초기/후속 상관없이 실패로 처리한다.
  - 실패 문구: `응답을 받지 못했어. 별은 차감하지 않았으니 다시 시도해줘.`
- `chat-error-handling`
  - 새 실패 문구를 사용자용 안전 문구로 보존한다.
- `api/saju/chat`
  - 성공한 assistant 응답 저장 뒤 `saju_readings.chat_used`를 `reading.chat_used + 1`로 업데이트한다.
  - incomplete 응답은 기존처럼 DB 저장/별 차감/제목 생성으로 가지 않는다.

## 검증

- 빈 후속 assistant 응답은 실패로 판정하는 테스트 추가.
- 성공 답변 후 `chat_used` 증가를 요구하는 회귀 테스트 추가.
- 관련 채팅 회귀 테스트 통과 확인.
