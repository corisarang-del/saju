# 결제모듈 보존형 비활성화 배포 반영

## 배경
- 이번 배포 목표는 Paddle 결제 모듈을 삭제하지 않고 보존하면서, production에서는 결제 진입점만 닫아 무료 상담형 베타를 먼저 배포하는 것이다.
- 기존 `release:gate`는 Paddle env를 필수로 요구해 비결제 베타 배포와 결제 재오픈 게이트가 섞여 있었다.

## TDD
- `src/lib/payments/feature-flag.test.ts`
  - env 누락 시 결제 기본값 false
  - 서버 결제 route는 `PAYMENTS_ENABLED=true`에서만 enable
  - 브라우저 checkout은 `NEXT_PUBLIC_PAYMENTS_ENABLED=true`에서만 enable
  - public flag off 상태에서 checkout 진입 전 에러 발생
- `src/lib/security/payment-disabled-beta-regression.test.ts`
  - 비결제 베타 release gate와 결제 재오픈 gate 분리
  - 코인샵 직접 접근 차단
  - Paddle checkout fail-closed
  - 상단/사이드바/채팅 paywall 충전 링크 숨김
  - 월간 리포트 상세판과 종합 백서 유료 CTA 숨김
- `src/lib/monthly-saju/billing.test.ts`
  - 결제 flag off 기본 상태에서는 무료 quota가 없어도 결제 prompt를 띄우지 않도록 변경

## 구현
- `src/lib/payments/feature-flag.ts`
  - `arePaymentsEnabled`: 서버용 `PAYMENTS_ENABLED` 판정
  - `areClientPaymentsEnabled`: 브라우저용 `NEXT_PUBLIC_PAYMENTS_ENABLED` 판정
  - `assertPaymentsEnabled`: checkout 진입 전 fail-closed
- `package.json`
  - `release:gate`: 비결제 베타용. Paddle env 없이 `test:env`, test, tsc, lint, build, high audit 실행
  - `release:gate:payments`: 결제 재오픈용. `REQUIRE_PADDLE_ENV=true` 포함
- `.env.example`
  - `PAYMENTS_ENABLED=false`
  - `NEXT_PUBLIC_PAYMENTS_ENABLED=false`
  - `REQUIRE_PADDLE_ENV=false`
  - Paddle star10/monthlyMembership product/price env 예시 추가
- 결제 진입점 차단
  - `/ko/coin-shop`: flag off면 준비 중 화면 노출
  - `openCheckout`, `PaddleCheckout`: public flag off면 checkout 차단
  - `SajuNavbar`, `LoginSidebar`: 충전 링크 숨김
  - `ChatInput`, `ChatPaywall`: 충전 링크 대신 무료 베타 안내
  - `MonthlyReportUnlockClient`, `/ko/reports`: 상세판 unlock/결제 옵션 CTA 숨김
  - `/ko/saju-report`, `SajuReportClient`: 종합 백서 결제 CTA 차단
  - `ReadingPreview`, `PaywallOverlay`: 전체 분석 결제 CTA 차단
- 랜딩/FAQ
  - 결제 off 상태에서는 `유료 충전과 멤버십은 준비 중` 톤으로 낮춤
  - 가격 정책 상수와 Paddle 상품 정책은 삭제하지 않고 유지

## 검증
- 타깃 테스트:
  - `pnpm exec vitest run src/lib/payments/feature-flag.test.ts src/lib/security/payment-disabled-beta-regression.test.ts src/lib/monthly-saju/billing.test.ts`
  - 3개 파일 / 12개 테스트 통과
- 릴리즈/디자인 관련 타깃:
  - `pnpm exec vitest run src/lib/security/release-gate-regression.test.ts src/lib/security/payment-disabled-beta-regression.test.ts src/lib/payments/feature-flag.test.ts src/lib/monthly-saju/billing.test.ts`
  - 4개 파일 / 21개 테스트 통과
- 비결제 베타 코드 gate:
  - `pnpm release:gate:code`
  - `pnpm test:env`, `pnpm test`, `pnpm exec tsc --noEmit`, `pnpm lint`, `pnpm build`, `pnpm audit --prod --audit-level high` 통과
  - 전체 테스트 52개 파일 / 212개 테스트 통과
  - audit은 high 이상 없음, low 1건만 남음
- production 승인 release gate:
  - `pnpm release:gate`
  - 코드 gate 후 `pnpm qa:live-api:free`, `pnpm qa:live-api`를 실행한다.
  - 현재는 Supabase DNS `ENOTFOUND`로 실패하므로 production 배포 승인은 아직 불가다.
- 결제 재오픈 gate:
  - `pnpm release:gate:payments`
  - 현재 Paddle env 미등록으로 실패. 비결제 베타에서는 정상적인 fail-closed 상태다.

## 남은 운영 작업
- Vercel production env에는 `PAYMENTS_ENABLED=false`, `NEXT_PUBLIC_PAYMENTS_ENABLED=false`를 명시한다.
- Supabase DNS 정상 환경에서 실제 API QA는 다시 실행해야 한다.
- 결제 재오픈 전 Paddle production 상품/가격/webhook env 등록 후 `pnpm release:gate:payments`와 실제 결제 QA를 별도로 통과해야 한다.
