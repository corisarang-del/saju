# 현우카드 이름클리핑 낮은높이 PC 보정 20260724

## 배경

디자인 재리뷰에서 production `/ko` PC 1280x720 기준 `현우` 카드 이름 라인이 `y=709`, `bottom=737`로 viewport 720 아래에 걸려 이름이 잘린 것처럼 보인다는 제보가 재현됐다. 텍스트 자체 overflow가 아니라 첫 화면 높이 부족으로 카드 하단 오버레이가 fold 아래로 밀린 문제였다.

## TDD

- `src/lib/design/design-audit-regression.test.ts`에 낮은 높이 PC 전용 회귀 테스트를 먼저 추가했다.
- 테스트 기준:
  - `CharacterCards.tsx`에 `@media (min-width:1024px) and (max-height:760px)` 전용 상하 여백 보정이 있어야 한다.
  - 같은 조건에서 카드 이미지가 `aspect-[3/4]`로 낮아져야 한다.
  - 카드 이름 오버레이 padding이 줄어야 한다.
  - 홈 hero 상단 여백, H1 크기, 문단 간격도 같은 조건에서 압축되어야 한다.
- 실패 확인 후 구현했다.

  - 설명 문단의 margin/line-height를 줄여 카드 시작 위치를 위로 당겼다.

## 브라우저 검증

- 로컬 dev 서버 `http://localhost:3000/ko`에서 Playwright로 1280x720 측정했다.
- `현우` 이름 요소 측정값:
  - `top`: 563.25
  - `bottom`: 591.25
  - `height`: 28
  - `viewportHeight`: 720
  - `fullyVisible`: true
- 기존 보고서의 `bottom=737`에서 fold 안쪽으로 충분히 올라왔다.
- 로컬 dev console의 1개 error는 production CSP에서 `unsafe-eval`을 제거한 상태와 React development debugging 기능이 충돌하는 개발모드 메시지였다. 카드 수정과는 별개다.
