# 디자인 감사 우선순위 수정

- 일시: 2026-06-30 23:59:37 KST

## 사용한 접근

- `design-taste-frontend` 기준으로 P0/P1/P2 디자인 감사 항목을 코드에 반영했다.
- TDD 방식으로 디자인 회귀 테스트를 먼저 추가하고 실패를 확인한 뒤 구현했다.

## 변경 내용

- 쿠키 배너를 모바일 하단 탭/CTA 위로 올리고, 닫기 상태도 저장하도록 수정했다.
- 랜딩/입력/결제/오늘피드/리포트/관리자 일부의 CTA 색상 체계를 퍼플 중심으로 통일했다.
- 토스 블루 하드코딩 색상(`#3182F6`, `#1B64DA` 등)을 주요 화면에서 제거했다.
- 모바일 랜딩에서 H1 바로 아래 캐릭터 카드가 먼저 보이도록 순서를 조정했다.
- 캐릭터 캐러셀 dot에 접근성 이름과 현재 상태를 추가했다.
- `prefers-reduced-motion`과 사용자 터치/포커스 이후 자동 캐러셀을 멈추도록 수정했다.
- 비로그인 캐릭터 카드를 `div onClick` 대신 button semantics로 바꿨다.
- 채팅 안 생년월일 입력카드를 `/reading` 흐름과 맞춰 단일 이름 입력으로 바꾸고 년/월/일 aria-label을 추가했다.
- 후기 카피는 기존 신뢰도 개선 카피를 유지하고, 이번 변경에서는 과장 표현을 추가하지 않았다.
