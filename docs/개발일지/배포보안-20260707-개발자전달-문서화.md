# 배포보안 20260707 개발자전달 문서화

## 요청
- 배포 전 보안이나 배포 시 문제점을 파악한 내용을 개발자에게 전달할 수 있도록 문서화해달라는 요청을 받았어.
- 소스코드는 수정하지 않았고 문서만 추가했어.

## 점검 요약
- `pnpm test`: 45개 파일 / 156개 테스트 통과.
- `pnpm lint`: 통과.
- `pnpm build`: 권한 상승 환경에서 통과.
- 기본 `pnpm test:env`: 통과.
- `REQUIRE_PADDLE_ENV=true pnpm test:env`: Paddle production 필수 환경변수 누락으로 실패.
- `pnpm audit --prod`: dependency metadata 외부 전송 정책 때문에 실행 거절됨.

## 개발자 전달 핵심
- 이전 P0였던 `/api/saju/update-status` 결제 상태 우회는 정책 함수로 차단된 상태다.
- 운영 Supabase에는 2026-07-07 신규 migration 4개 적용 확인이 필요하다.
- chat/suggestions rate limit과 chat generation lock은 메모리 기반이라 Vercel 분산 배포에서 전역 보장이 안 된다.
- 종합 리포트는 5별 차감 후 AI 생성 실패 시 status/환불/재시도 정책이 부족하다.
- OAuth redirect base URL이 요청 헤더를 우선 신뢰해 production에서는 고정 origin 또는 allowlist가 필요하다.
- 관리자 별 수동 조정은 이메일 allowlist 외에 IP 제한, 2차 인증, actor/reason 감사로그 보강이 필요하다.
