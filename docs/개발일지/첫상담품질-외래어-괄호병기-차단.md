# 첫상담품질 외래어 괄호병기 차단

## 문제

- 최신 Gemini 첫 상담 QA에서 5케이스 중 1개가 확인필요로 남았다.
- `썸/재회` 답변에 `시기(타이밍)`이 생성됐다.
- 기존 규칙은 `타이밍`을 `시기`로 바꾸라고 했지만, 모델이 한국어와 외래어를 괄호로 함께 쓰는 방식으로 우회했다.

## 처리 방향

- 첫 상담 프롬프트에 괄호로 외래어를 병기하지 말라는 규칙을 추가한다.
- QA 스크립트에 괄호 병기 외래어 패턴을 추가한다.
- 회귀 테스트로 해당 규칙 존재를 고정한다.

## 수정

- `getFirstConsultationInstructions`와 `/api/saju/chat` 첫 상담 규칙에 `괄호로 외래어를 병기하지 마`를 추가했다.
- QA 스크립트에 `parenthesizedForeignWordPattern`을 추가해서 `시기(타이밍)` 같은 괄호 병기를 `가벼운 외래어 없음` 실패로 잡게 했다.
- 첫 live QA 재실행에서 외래어 실패는 사라졌지만 `썸/재회`가 질문으로 끝나지 않는 변동 실패가 나와, 마지막 문장은 실제 질문 1문장이고 설명문이나 조언문으로 끝내지 말라는 규칙을 추가했다.
