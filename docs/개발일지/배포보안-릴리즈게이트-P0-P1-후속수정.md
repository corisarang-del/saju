## 수정 내용

- `/api/saju/update-status`에서 클라이언트가 `paid`, `generating`, `completed` 같은 특권 상태로 직접 변경하지 못하게 막았다.
- 리포트 생성 흐름에서 클라이언트의 별도 `paid` 상태 변경 호출을 제거했다.
- `/api/saju/deduct-stars`는 body의 `userId`를 신뢰하지 않고 세션 사용자만 기준으로 처리하게 바꿨다.
- 새 Supabase migration `202607030010_report_payment_status_hardening.sql`을 추가해 별 차감, 거래 기록, `saju_readings.status = 'paid'` 승격을 하나의 RPC 트랜잭션 안에서 처리하게 했다.
- 이미 `paid/generating/completed` 상태인 리포트는 `REPORT_ALREADY_PAID`로 중복 차감을 막게 했다.
- `/api/saju/chat`에 사용자 단위 in-memory 생성 잠금과 분당/일일 rate limit을 추가해 AI 호출 전 동시 생성과 과도한 요청을 차단했다.
- 종합 리포트 화면의 필요 별 개수 표시를 서버 정책 상수 `REPORT_STAR_COST`와 맞춰 실제 차감과 UI 안내가 갈라지지 않게 했다.
