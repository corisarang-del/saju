# 배포보안 관리자 별조정 감사로그 하드닝

- 일시: 2026-07-10 15:21:50 KST

## 작업 내용

- 배포보안 문서의 P2 관리자 수동 조정 하드닝을 코드로 보완했다.
- `admin_audit_logs` 테이블과 `admin_adjust_user_stars` RPC를 추가했다.
- 관리자 별 충전/차감은 이제 서버 액션에서 직접 `user_stars`와 `star_transactions`를 따로 수정하지 않고, service role 전용 RPC 하나로 처리한다.
- RPC 안에서 `user_stars` row lock, 잔액 변경, `star_transactions` 기록, `admin_audit_logs` 기록을 같은 DB 트랜잭션으로 묶었다.
- 관리자 화면에 `조정 사유` 입력을 추가했고, 서버에서 4자 이상 500자 이하로 검증한다.
- 감사로그에는 actor user id, actor email, target user id, action, amount, before/after balance, reason, request IP, user-agent가 남는다.
- `.next/types` 중복 생성물 exclude를 `* *.ts` 패턴으로 정리해 `2/3/4`처럼 늘어나는 복제 파일 충돌을 막았다.
