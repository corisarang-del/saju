## 실제 인증 API QA

### 상태 전이 차단

- 임시 Supabase QA 유저와 pending reading을 생성했다.
- `/api/saju/update-status`에 `status: paid` 직접 승격 요청을 보냈다.
- 결과:
  - status `403`
  - body `{"error":"Forbidden status transition"}`
  - reading status는 `pending` 유지.
- 판단: 클라이언트의 유료 상태 직접 승격 차단은 정상이다.

### 리포트 별 차감

- 같은 pending reading으로 `/api/saju/deduct-stars`를 호출했다.
- 결과:
  - 첫 호출 status `500`
  - body `{"error":"Failed to deduct stars"}`
  - reading status는 `pending` 유지.
  - 두 번째 호출도 status `500`.
- 직접 RPC 확인:
  - `deduct_stars_for_report` 호출이 `PGRST202`로 실패.
  - 오류: `Could not find the function public.deduct_stars_for_report(p_reading_id, p_user_id) in the schema cache`
- 판단: 코드와 마이그레이션은 추가됐지만 실제 Supabase DB에는 RPC가 적용되지 않았거나 PostgREST schema cache가 아직 반영되지 않았다. 리포트 별 차감은 실제 환경에서 실패한다.
