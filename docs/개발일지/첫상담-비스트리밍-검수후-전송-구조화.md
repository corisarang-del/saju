# 첫상담 비스트리밍 검수 후 전송 구조화

## 배경

- 기존 `/api/saju/chat`은 첫 상담도 `streamText`로 바로 브라우저에 보내고, `onFinish`에서 저장 전 게이트를 검사했다.
- 이 구조에서는 저장과 별 차감은 막을 수 있지만, 품질 미달 문장이 사용자 화면에 이미 노출될 수 있었다.

## 수정

- `src/app/api/saju/chat/route.ts`
  - 첫 상담 요청을 `isInitialAnalysisRequest`로 분리했다.
  - 첫 상담은 `generateValidatedInitialAnalysis`에서 `generateText`로 먼저 생성한다.
  - `shouldPersistAssistantAnswer`를 통과하지 못하면 `buildInitialAnalysisRetryPrompt`로 최대 3회 재작성한다.
  - 통과한 답변만 DB 저장, 코칭 스냅샷 생성, 별 차감, 제목 생성 후 `createUIMessageStreamResponse`로 UI 메시지 스트림 형식으로 내려보낸다.
  - 후속 상담은 기존 `streamText` 경로를 유지한다.
  - 저장/스냅샷/차감/제목 생성 흐름을 `persistSuccessfulAssistantAnswer`로 묶어 첫 상담과 후속 상담이 같은 성공 처리 규칙을 쓰게 했다.
- `src/lib/ai/chat-completion-guard.ts`
  - 이름이 빠진 님 호칭을 저장 전 게이트에서 차단하되, 정상적인 `다은님의 사주` 같은 표현은 과차단하지 않도록 정규식으로 조정했다.
- `scripts/qa-gemini-first-consultation.mjs`
  - 이름 빠진 님 호칭 검사를 추가하고 과차단을 방지했다.

## 테스트

- `src/lib/saju/chat-stream-failure-regression.test.ts`
  - 첫 상담이 `streamText` 전에 비스트리밍 생성/검수를 거치고, 통과본만 UI 메시지 스트림으로 반환하는 회귀 테스트를 추가했다.
